# 배송준비중 주문

<Callout type="info">
**운영자가 상품을 준비한 다음 배송을 준비하고 있는 단계인 배송준비 상태의 주문 건을 조회합니다.**

**배송준비중 주문 건은 배송번호와 주문번호 기준으로 주문정보를 확인하고 관리할 수 있으며, '결제완료' 상태부터 '구매확정' 상태까지 모든 단계의 변경(이동)이가능합니다.**\ <mark style="color:red;">**단, 에스크로 정책에 의해, 배송준비중 상태의 에스크로 결제 주문 건은 '결제완료\~배송중' 상태로 수동 변경(이동)은 가능하고, '배송완료'와 '구매확정' 상태로 변경이 불가한 점을 참고해 주세요**</mark><mark style="color:red;">.</mark>

**엑셀파일 업로드로 배송준비중 주문 건에 택배사와 송장번호 정보를 일괄 적용할 수 있습니다.**
</Callout>

## 배송준비중 주문리스트 (배송번호/주문번호)

<figure><img src="/images/order/delivery-prepare/spaces%2F5MtRLrF338WdIfIJP9jq%2Fuploads%2FupBibqB97a7x4IWOtapW%2F%EB%B0%B0%EC%86%A1%EC%A4%80%EB%B9%84%EC%A4%91%EC%A3%BC%EB%AC%B8%EB%A6%AC%EC%8A%A4%ED%8A%B8%20%EB%B3%B5%EC%82%AC.png" alt=""/><figcaption><p>&#x3C;배송준비중 주문리스트></p></figcaption></figure>

**① 주문상태 선택** \
주문상태 업데이트(변경)를 위해 주문상태 선택에서 변경을 희망하는 주문상태를 선택합니다.

**② 일괄처리** \
복수 주문 건을 선택한 주문상태로 일괄변경 처리 할 수 있습니다.

* 배송번호 기준리스트 : 배송번호 단위 주문상태 변경&#x20;
* 주문번호 기준리스트 : 상품번호 단위 주문상태 변경\
  (예 : 1명의 주문자가 2개 이상의 상품을 주문한 경우 상품별 주문상태 변경 가능)&#x20;

**③ 배송중** \
\[배송중]을 클릭하면 배송준비중 다음 단계인 배송중 상태로 변경(이동)되며, 배송중 주문리스트에서 해당 주문 건을 조회합니다.

**④ 송장번호 저장**\
주문 건에 택배사를 선택하고 송장번호를 입력한 다음 \[송장번호 저장]버튼을 클릭하면 해당 주문 건에 정보가 바로 적용됩니다.

**⑤** [**스캐닝 출고 처리**](#undefined-2)\
송장번호 또는 상품 옵션관리코드를 선택하고 스캐닝 방식으로 저장하여 자동 출고처리를 진행할 수 있습니다.&#x20;

**⑥** [**배송보류 처리**](#undefined-3)\
출고가 어려운 경우 등 배송중 상태로 변경(이동)이 불가한 주문 건을 수동으로 '배송보류 처리' 합니다.&#x20;

**⑦** [**묶음배송 일괄처리**](#undefined-4)\
출고가 함께가능한 주문 건을 한 번에 묶어서  출고 가능한 배송준비중 상태로 변경하는 기능입니다.&#x20;

**⑧ 묶음배송 해제** \
묶음배송으로 처리된 주문 건을 해제합니다.&#x20;

* 묶음배송 주문 건을 해제하면 각각의 주문 건에 원배송번호가 다시 부여되고, 묶음배송 주문 건에 입력한 택배사 및 송장번호 정보는 삭제됩니다.

**⑨** [**양식설정**](#undefined-5)\
엑셀파일로 제공되는 택배사 출고양식의 세부 항목을 설정합니다.&#x20;

**⑩** [**택배사 출고양식 다운**](#undefined-6)\
\[택배사 출고양식 다운] 버튼을 클릭하면 설정한 출고양식 엑셀파일을 다운로드 받을 수 있습니다.&#x20;

***

#### <mark style="color:blue;">■  스</mark><mark style="color:blue;">**캐닝 출고 처리**</mark>

* 송장 스캐닝 방식 : 주문 단위로 배송처리 가능하며 송장번호를 입력해 스캐닝 처리를 진행합니다.
* 옵션 스캐닝 방식 : 상품 단위로 배송처리 가능하며 옵션바코드를 입력해 스캐닝 처리를 진행합니다.&#x20;
* 주문 건은 배송번호로만 조회 가능하며 스캐닝 출고 처리 시 해당 주문 건은 즉시 배송중 상태로 변경(이동)되어, 배송중 메뉴에서 해당 주문 건을 조회합니다.

#### <mark style="color:blue;">■</mark>  <mark style="color:blue;"></mark><mark style="color:blue;">**배송보류 처리**</mark>

* 배송보류 처리 창에서 출고예정일, 배송보류처리 사유, 배송보류 안내 알림 등을 설정하고 배송보류 처리 합니다.&#x20;
* 배송보류 처리 주문 건은 '배송준비중\_배송보류' 상태로 변경되어, [배송보류 관리](https://nhn-commerce.gitbook.io/shopby_enterprise_manual/order/hold-delivery) 메뉴에서 별도 관리됩니다.&#x20;
* 출고가 가능해진 배송보류 처리 주문 건은 배송보류 해제 처리할 수 있습니다. 해당 주문 건은 다시 배송준비중 상태로 변경(이동)됩니다.

#### <mark style="color:blue;">■  묶</mark><mark style="color:blue;">**음배송 일괄처리**</mark>

* 배송번호와 주문번호 주문 건 모두 각각 묶음배송 할 수 있고, 묶음배송 주문 건에는 신규 배송번호가 부여되며 묶음 아이콘이 표시됩니다.&#x20;
* 묶음배송은 배송비 템플릿의 묶음배송 그룹이 같은 주문 건과 수령자명/수령자 휴대폰번호/배송지주소등의 정보가 모두 동일한 주문 건에 한해서 가능합니다.&#x20;

<Callout type="warning">
배송지의 실제 주소지가 동일하더라도 지번과 도로명으로 각각 다르게 입력된 경우 묶음배송이 불가한 점을 참고해 주세요.&#x20;
</Callout>
