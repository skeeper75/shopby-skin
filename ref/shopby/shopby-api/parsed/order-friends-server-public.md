# order-friends-server

**버전**: unspecified
**서버**: https://server-api.e-ncp.com

주문과 관련된 기능(통계, 정산, CS 등)을 수행하는 server API입니다.

## OrderCs

### GET /cs

**요약**: CS 처리내역 조회

**설명**:
CS 처리내역 조회

**파라미터**:

| 이름      | 위치   | 타입   | 필수 | 설명                                                  |
| --------- | ------ | ------ | ---- | ----------------------------------------------------- | ---------- |
| orderNo   | query  | number | ❌   | 주문번호^                                             | order-no   |
| csNos     | query  | string | ❌   | CS 번호^                                              | [1, 2, 3]  |
| startYmd  | query  | string | ❌   | 조회 시작 일자 (기본값: 3개월 전)^                    | 2025-10-28 |
| endYmd    | query  | string | ❌   | 조회 종료 일자 (기본값: 오늘)^                        | 2025-10-31 |
| page      | query  | number | ❌   | 페이지 번호 (기본값: 1)^                              | 1          |
| size      | query  | number | ❌   | 페이지 크기 (기본값: 30)^                             | 30         |
| systemKey | header | string | ✅   | 시스템 키 (외부시스템 연동을 위한 server API 호출 키) |

- 발급경로: 워크스페이스> 셀러어드민> 신규 앱(App) 등록 시 수정페이지에서 확인가능
- 앱 기준으로 systemKey 발급됨
  ^|test-system-key |
  | Authorization | header | string | ❌ | 인증토큰 (Bearer 발급받은 엑세스 토큰)

- 형식: Bearer {access_token} (중간에 띄어쓰기 필수입니다.)
- 발급경로: POST /auth/token 또는 POST /auth/token/long-lived 의 응답 값으로 내려온 access_token
  ^|Bearer ABCDERAdjflaskjdfosiaetlkfs |
  | mallKey | header | string | ❌ | 외부 API 연동키 (외부시스템연동 코드)

※ server API 호출하는 기존 방식으로, 향후 fade out될 수 있습니다.<br />
Authorization을 입력을 통한 serverAPI 호출방식 권장드립니다.<br />
(Authorization 입력했을 경우, mallKey는 null로 보내시길 바랍니다.)<br />
^|각 어드민에서 복사된 값 |
| Version | header | string | ✅ | API 버전^|1.0 |

**응답**:

- **200**: 200

**설명**:

## 부가설명 및 특이사항

매출 데이터 조회를 위한 API입니다.

### v1.0

페이징을 지원하지 않습니다.
request 의 page와 size 는 사용하지 않습니다.
응답값은 v1.1 응답의 contents[] 필드와 동일합니다.

```
[
  {
    "companyType": "회사이름",
    "mallName": "쇼핑몰명",
    ...
  }
]
```

### v1.1

페이징 처리를 위한 totalCount 를 제공합니다.

request 의 page와 size 를 입력해야 합니다.

응답값 중, contents[] 하위 필드는 기존 v1.0 API 의 응답과 일치합니다.
