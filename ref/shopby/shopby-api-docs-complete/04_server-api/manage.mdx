등록된 적립금 변동 요청 내역을 검색합니다.
         
## Request

- page: 페\
        이지 번호
- pageSize: 페이지 크기
- searchType: 검색 유형 enum(option). 기본값은 **ALL**.
\
        - keyword: 검색어(option)
- periodType: 검색 기간 유형 enum(option). 기본값은 **REGISTER**.
\
        - startDateTime: 검색 기간 시작일(option). yyyy-MM-dd HH:mm:ss. 기본값은 오늘 기준 3개월 전
\
        - endDateTime: 검색 기간 종료일(option). yyyy-MM-dd HH:mm:ss. 기본값은 오늘
- immediately:\
        \ 즉시 지급 여부(option)
- requestGroupType : 전체 / 지급 / 차감
- requestTypes: 즉시지\
        급 / 즉시차감 
- statuses: 지급 요청 상태 enum 리스트(option)

## Response

- no: 적\
        립금 변동 요청 번호
- mallNo: 적립금 지급을 요청한 쇼핑몰 번호
- mallName: 쇼핑몰 이름
- requestType\
        \ : 적립금 변동 요청 타입 (즉시지급, 즉시차감, 예약지급, 주문예약지급)
- status: 지급 요청 상태 enum
- requestDateTime:\
        \ 적립금이 지급될 시각.
- immediately: 즉시 지급 여부
- amount: 지급할 적립금 금액
- reason: 적\
        립금 변동 사유
- reasonDetail: 적립금 지급 사유(주문 지급 시 사용)
- validPeriodType: 유효기간 설\
        정에 따른 타입 - NONE: 별도 유효기간 지정(체크안함)
- PERIOD: 기간지정
- DATETIME: 만료일지정
- NO_LIMIT:\
        \ 제한없음
- validPeriod: 지급된 적립금의 유효 개월 수. 0 일 경우 쇼핑몰 적립금 설정의 유효기한을 따름.- 샵바이\
        \ Pro의 경우 0인 경우 유효기간 제한 없음
- expireDateTime : validPeriod 0이고 값이 있을 때 넣어줘\
        야 반영 됩니다. validPeriod보다 우선순위가 높기에 0이라도 값이 있다면 지정만료일자로 반영됩니다.
- orderRequest:\
        \ 구매확정 시 지급되는 주문정보 (orderNo, orderProductOptionNo)
- reviewNo: 구매후기 작성 일련\
        번호
- productDisplayName: 상품 노출 명
- registerAdminNo: 적립급 지급 요청을 등록한 운영자 번\
        호
- registerAdminName: 적립급 지급 요청을 등록한 운영자 이름
- registerAdminStatus: 적립급\
        \ 지급 요청을 등록한 운영자 상태
- registerDateTime: 적립급 지급 요청 등록일시
- updateAdminNo:\
        \ 마지막으로 적립급 지급 요청을 수정한 운영자 번호. 수정되지 않은 경우 **null**
- updateAdminName: 마지막\
        으로 적립급 지급 요청을 수정한 운영자 이름. 수정되지 않은 경우 **null**
- updateAdminStatus: 마지막으로\
        \ 적립급 지급 요청을 수정한 운영자 상태. 수정되지 않은 경우 **null**
- updateDateTime: 마지막 적립급 지급\
        \ 요청 수정일시. 수정되지 않은 경우 **null**
- paymentDateTime: 실제 적립금 지급이 수행된 시각. 아직 처\
        리되지 않았다면 **null**
- paymentCount: 적립금을 지급받은 회원 수. 아직 처리되지 않았다면 **0**
- tryCount\
        \ : 적립금지급 시작 시 지급대기 상태의 지급 수. 아직 처리 되지 않았앋면 **0**

## Enums

### searchTypes
\
        - ALL: 전체
- REASON: 지급사유
- REGISTER: 지급등록자
- UPDATER: 최종수정자
- REQUEST_NO:\
        \ 적립금번호

### periodType
- REGISTER: 등록일시
- PAYMENT: 지급일시
- REQUEST: 예\
        약일시
- UPDATE: 최종수정일

### requestGroupTypes
- ALL: 전체
- ADD: 지급
- SUB:\
        \ 차감

### requestTypes
- DIRECT_ADD: 즉시지급
- DIRECT_SUB: 즉시차감
- RESERVE_ADD:\
        \ 예약지급
- ORDER_RESERVE_ADD: 주문예약지급
### validPeriodType
- NONE: 별도 유효기간\
        \ 지정(체크안함)
- PERIOD: 기간지정
- DATETIME: 만료일지정
- NO_LIMIT: 제한없음
### status
\
        - WAITING: 지급대기
- COMPLETED: 지급완료
- CANCELED: 지급취소
- PROCESSING: 지급처리중
