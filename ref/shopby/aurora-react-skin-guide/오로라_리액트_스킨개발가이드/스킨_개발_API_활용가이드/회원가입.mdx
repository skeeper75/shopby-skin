### b. 입력 정보 중복여부 확인




회원 정보 입력란 항목 중 아이디, 이메일의 경우 중복 여부를 확인해야 합니다.

기본 스킨에서는 해당 쇼핑몰 내 중복된 회원 정보가 없을 경우 '(ex) 사용 가능합니다'라는 안내 메시지를 노출하며
중복된 회원 정보가 존재할 경우 '(ex) 이미 사용 중인 아이디입니다. 다른 아이디를 입력하여 주세요' 라는 안내 메시지를 노출합니다.




> [GET /profile/id/exist](https://docs.shopby.co.kr/?url.primaryName=member/#/Profile/get-profile-ci-exists)

```
▶ CI 중복 확인하기
해당 쇼핑몰에 이미 해당 아이디로 가입한 회원이 존재하는지 확인합니다
```




> [GET /profile/email/exist](https://docs.shopby.co.kr/?url.primaryName=member/#/Profile/get-profile-email-exist)

```
▶ 해당 쇼핑에 이메일 중복여부 체크하기
해당 쇼핑몰에 이미 해당 이메일로 가입한 회원이 존재하는지 확인합니다
```




### c. 본인 인증 버튼




:heavy\_check\_mark: **인증 버튼 노출 여부**

이메일 인증버튼, SMS 인증 버튼 또는 휴대폰 인증 버튼의 경우 [샵바이 API 호출가이드](https://nhnent.dooray.com/share/pages/WoOk8q6KT1K3MZcLSuyZsw/3520338751321947577) 문서에서 소개한 `GET /malls` 을 통해
어드민의 이메일/SMS/휴대폰 인증 사용함 여부를 확인하여, 인증버튼 노출 여부를 결정합니다.

어드민 설정경로는 아래와 같습니다.
shop by pro [설정> 기본정책 >쇼핑몰관리> 회원 인증 설정]
shop by premium [서비스 관리 > 쇼핑몰 관리 > 회원 인증 설정]

* 이메일 인증 선택 시: 이메일 인증버튼
* SMS 인증 선택 시: SMS 인증 버튼이 노출
* 휴대폰 인증 선택 시: 휴대폰 인증 버튼이 노출




> [GET /malls](https://docs.shopby.co.kr/?url.primaryName=admin/#/Mall/get-malls) 

```
▶ 몰 정보 조회하기
쇼핑몰 전반에 대한 기본 정보와 설정 데이터를 조회할 수 있습니다
```




해당 API는 쇼핑몰 어느 화면에서든 공통적으로 필요한 기본 API입니다.
기본 스킨에서는 API 호출 횟수를 줄이고자 응답 값을 localStorage 에 캐싱 주기를 5분으로 설정하였습니다.

`mallJoinConfig` 내 `AuthenticationType` 에 따라 버튼 노출 여부를 결정합니다.
아래 코드를 만족했을 때 SMS 또는 이메일 인증이 가능하며, 각각의 인증 버튼이 노출됩니다.




```
1
2    const {
3        mallJoinConfig: { authenticationTimeType, authenticationType },
4    } = shopby.cache.getMall();
5
6    const needsAuthType = {
7        needsEmailAuth: authenticationType === 'AUTHENTICATION_BY_EMAIL',
8        needsSmsAuth: authenticationType === 'SMS_AUTHENTICATION',
9    };
10    const authType = authenticationTimeType === 'JOIN_TIME' ? needsAuthType : {};
11
```




`needsEmailAuth` 와 `needsSmsAuth`은 프론트 화면에서 관리하는 변수 값이며,
`AUTHENTICATION_BY_EMAIL`이면 `needsEmailAuth = true`
`SMS_AUTHENTICATION`이면 `needsSmsAuth = true`로 각 ENUM과 매핑됩니다.

:heavy\_check\_mark: **인증번호 발송과 확인**
SMS와 이메일, 그리고 휴대폰 인증 모두 아래 API가 동일하게 적용됩니다.




> [POST /authentications](https://docs.shopby.co.kr/?url.primaryName=auth/#/Authentication/SendAuthenticationNumber)

```
▶ 인증번호 발송하기
인증 번호를 발송합니다
```




회원의 연락처 또는 입력한 연락처로 인증번호를 발송합니다.
해당 API는 회원가입 화면 뿐 아니라, 아이디/비밀번호 찾기 및 회원정보 수정 등의 화면에서 사용됩니다.
Request body 내 `type`에서 SNS인증의 경우 `SNS`, 이메일 인증의 경우 `EMAIL`을 입력합니다.

유효기간(인증번호 만료까지 남은 시간) `remainTime`이 만료되었을 경우 아래와 같은 안내 문구를 노출합니다. 




* SMS: 5분
* 이메일: 10분

![Inline-image-2026-01-28 11.04.46.240.png](https://nhnent.dooray.com/share/tree/WoOk8q6KT1K3MZcLSuyZsw/pages/3519671841187655806/attach-files/4255221121227105873)







> [GET /authentications](https://docs.shopby.co.kr/?url.primaryName=auth/#/Authentication/get-authentications)

```
▶ 인증번호 확인하기
입력 받은 인증번호를 확인합니다
```

사용자로부터 입력 받은 인증 번호의 유효성을 체크하며, 실패 시 에러코드 메시지를 전달합니다.

### a. 약관 데이터 호출




 어드민 설정경로는 아래와 같습니다.

> [GET /terms](https://docs.shopby.co.kr/?url.primaryName=manage/#/Terms/search-used-terms)

```
▶ 적용 중인 몰 약관 조회하기

```




기본 스킨에서는 API 호출 횟수를 줄이기 위해 localStorage에 5분 동안 캐시합니다.
해당 API는 회원 가입 화면뿐 아니라, 주문서 구매 동의 화면, 회원 탈퇴 및 쇼핑몰 푸터 영역에서도 활용됩니다.




* [필수] 이용약관 `USE`
* [필수] 개인정보 수집 및 이용동의 `PI_COLLECTION_AND_USE_REQUIRED`
