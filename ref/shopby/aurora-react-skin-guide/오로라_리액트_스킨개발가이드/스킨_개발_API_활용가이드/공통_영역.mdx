# > 공통 영역

## 이해하기

> 쇼핑몰 내에서 화면이 바뀌어도, 계속 유지되는 요소를 공통 화면으로서 소개합니다.

* 🅐 상단 바
    * 로고 배너
    * 장바구니
* 🅑 하단 바
    * 홈 버튼, 검색영역, 마이페이지
* 🅒 좌측 슬라이드 메뉴
    * 로그인
    * 카테고리
    * 게시판 (공지사항, 고객센터>FAQ )
* 🅓 푸터 (footer)영역
    * 회사소개
    * 이용약관
    * 개인정보 처리방침
    * 이용안내

***

## 🅐 상단 바

![Group 2 (1).png](https://nhnent.dooray.com/share/tree/WoOk8q6KT1K3MZcLSuyZsw/pages/3520339276669442643/attach-files/3523465464322598708)
상단 바 영역에서 API 를 호출하여 화면을 구성해야 하는 요소는 아래 2가지입니다. 

* ⓐ 로고 배너
* ⓑ 장바구니 개수

### :heavy_check_mark:ⓐ 로고 배너

[메인페이지_배너영역](https://nhnent.dooray.com/share/pages/WoOk8q6KT1K3MZcLSuyZsw/3520339109559697129) 문서에서 소개하는 API입니다.
해당 배너 리스트 데이터에 쇼핑몰 로고(Logo)가 포함되어 있습니다.
로고의 경우 화면이 바뀌어도 대부분의 스킨에서 고정된 자리에 위치해있기 때문에, 사실상 모든 화면에서 지속적으로 사용되는 API입니다.
따라서 기본 스킨에서는 API 호출 횟수를 줄이고자 localStorage 에 5분 주기로 캐싱합니다.

### :heavy_check_mark:ⓑ 장바구니 개수

> 장바구니 기능은 쇼핑몰 회원 여부와 상관없이 제공합니다.
> 단, 회원인 경우에만 API를 호출하며 비회원인 경우 API 호출이 아닌 localStorage 에 자체적으로 저장합니다.

* 회원 (member)
    * 기본 스킨에서는 API 호출 횟수를 줄이기 위해 localStorage 에 캐싱 주기 5분으로 사용하고 있습니다.

> [GET /cart](https://docs.shopby.co.kr/?url.primaryName=order/#/Cart/get-cart)

```
▶ 장바구니 개수 조회하기
로그인 회원의 장바구니 개수를 조회합니다.
```

* 비회원(guest)
    * 비회원의 경우에도 장바구니 버튼을 제공합니다. 단, API 호출이 아닌 localStorage 내 비회원 장바구니 데이터를 자체적으로 관리해야 합니다.

## 🅒 좌측 슬라이드 메뉴

* ⓐ 로그인
* ⓑ 카테고리
* ⓒ 게시판 (공지사항, 고객센터>FAQ )

### :heavy_check_mark:ⓐ 로그인

[로그인 가이드문서](https://nhnent.dooray.com/share/pages/WoOk8q6KT1K3MZcLSuyZsw/3519671880330221332)를 확인하시길 바랍니다.

### :heavy_check_mark:ⓑ 카테고리

> [GET /categories](https://docs.shopby.co.kr/?url.primaryName=display/#/Category/get-categories-by-keyword)

```
▶ 전체 카테고리 조회하기
쇼핑몰의 모든 배너 정보를 조회합니다.
```

쇼핑몰 화면에서 카테고리란, 화면이 바뀌어도 대부분의 경우 화면에 상시 노출되는 요소입니다.
따라서   `localStorage`에 캐싱 주기 5분으로 사용하고 있습니다.

아래 2가지 형태의 카테고리 데이터를 제공합니다.

* `flatCategories` 모든 카테고리를 하나의 배열에 쭉 나열한 형태
* `multiLevelCategories` 계층을 가지는 카테고리 형태 (최대 5depth)

오로라 리액트 기본 스킨에서는 '좌측 슬라이드 메뉴' 및 '메인화면 상단bar 하단영역'에서 1depth만 리스팅되어 출력됩니다.
카테고리 상품 리스트에서는  5depth까지 표기됩니다.

쇼핑몰 어드민에서 아래 경로에서 관리할 수 있습니다.

* shop by pro: 상품> 상품 분류관리> 전시 카테고리 관리
* shop by premium: 전시관리 > 전시 카테고리 관리

### :heavy_check_mark:ⓒ 게시판

일반 게시판에 대한 게시글 리스트는 (1) 공지사항 (2) FAQ 2가지만 제공합니다.

> [GET /boards/{boardNo}/articles](https://docs.shopby.co.kr/?url.primaryName=manage/#/Board/search-posts)

```
▶ 게시글 리스트 조회하기 
특정 게시판의 게시글 리스트를 조회합니다
```

게시판 번호 *`boardNO`*기준으로 특정 게시판의 게시글 리스트를 조회하는 API입니다. 
기본 스킨에서는, API 호출 횟수를 줄이기 위해 localStorage 에 5분 주기로 캐시합니다.

## 🅓 푸터 (footer)영역

* ⓐ 회사소개
* ⓑ 이용약관/ 개인정보 처리방침/  이용안내

> 쇼핑몰 하단 푸터 영역에 필수적으로 기재해야 할 법적 필수 항목은 다음과 같습니다.

* 상호 및 대표자 이름
* 사업장 주소(소재지)
* 연락처 (대표번호, 이메일)
* 개인정보관리책임자
* 사업자등록번호
* 통신판매업신고번호
* 이용약관
* 개인정보취급방침
* 호스팅제공자: 전자상거래법 규정에 따라 호스팅서비스 제공자 상호를 반드시 표시하도록 의무화 되어 있습니다. 예시는 다음과 같습니다.
    * 호스팅제공자: 엔에이치엔커머스(주)
    * Hosting by 엔에이치엔커머스(주)
* 에스크로 인증마크

### :heavy_check_mark:ⓐ 회사소개

앞서 소개한 `GET/ malls`를 통해 이미 호출한 쇼핑몰 기본 정보 내 포함되어 있는 데이터입니다.
해당 설정 값들은 실시간 변동성이 크지 않은 데이터로, API 호출 횟수를 줄이고자 응답 값을 localStorage 에 캐싱 주기 5분으로 사용하고 있습니다.

### :heavy_check_mark:ⓑ 이용약관/ 개인정보 처리방침/  이용안내

> [GET /terms](https://docs.shopby.co.kr/?url.primaryName=manage/#/Terms/search-used-terms)

```
▶ 몰 약관 조회
쇼핑몰에 적용 중인 몰 약관을 조회합니다.
```

어드민 [설정 > 기본정책 > 약관/개인정보 처리방침 관리]에서 생성한 약관 내용을 조회할 수 있습니다.

`termsTypes` 으로 각 약관 내용을 구분합니다.

* 쇼핑몰/회사 소개 : `MALL_INTRODUCTION`
* 이용약관 : `USE`
* 개인정보처리방침 : `PI_PROCESS`
* 이용 안내 : `ACCESS_GUIDE`

해당 API는 푸터 영역 뿐 아니라, 회원가입, 회원탈퇴 및 주문서 구매 동의 화면에서도 활용됩니다.
기본 스킨에서는, API 호출 횟수를 줄이기 위해 localStorage 에 5분 주기로 캐싱합니다
