# @shopby/shared

## 이해하기

> NHN커머스에서 제공하는 **@shopby 패키지**를 활용하여 보다 손 쉬운 개발이 가능합니다.
> @shopby 패키지 중 '@shopby/shared'에 대해 안내합니다.

### `@shopby/shared`

* 서버상태 관리와 복잡한 비즈니스 로직을 처리하는 타입스크립트 기반 패키지를 제공합니다
* [패키지 릴리즈 문서](https://skins.shopby.co.kr/shopby/shared/-/releases)

***

## ShopbyQuery

* [fetch](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch) API 를 사용해 `cache`, `abort` 등의 기능을 쉽게 사용할 수 있도록 맵핑한 기능을 제공합니다. .
* `ShopbyQuery` 는 `executeQuery` 와 `executeMutation` 두가지 api 를 제공합니다.
    * 추후 동기/비동기 구분하여 개선한 인터페이스를 제공할 예정입니다.
* `ShopbyQuery` 는 `ShopbyContext`를 반환합니다.

###  :heavy_check_mark:`executeQuery(queryOption)`

* `GET` api 호출 시 사용합니다.
* 인자로 queryOption 을 넘겨야 합니다.

| 이름 | 필수여부 | 타입 | 기본 값 | 설명 |
| --- | ---- | --- | ---- | --- |
| queryFn | o | function | - | api 호출 함수.
ㄴ response 를 리턴해야함. |
| queryKey | x | any |  | 고유한 키.
queryKey 를 기준으로 캐싱을 관리함. |
| payload | x | any |  | queryFn 함수 인자로 넘겨짐. |
| enabled | x | boolean \| function | true | queryFn 함수 호출 여부
ㄴ function 인 경우 반환 결과에 따라 queryFn 을 호출함 |
| cacheTimeByMinutes | x | number | 0 | queryKey 를 기준으로 캐싱되는 데이터의 유효한 시간(분 단위) |
| usesLocalStorage | x | boolean | false | 로컬스토리지 사용여부 |
| onReady | x | (context: ShopbyContext) => void | - | queryFn 이 호출 되기 전에 호출함. |
| onSuccess | x | (context: ShopbyContext) => void | - | queryFn 이 성공적으로 호출된 후에 호출함. |
| onError | x | (context: ShopbyContext) => void | - | queryFn 호출이 실패했을 때 호출함. |
| onSettled | x | (context: ShopbyContext) => void | - | queryFn 호출 후 성공/실패에 관련없이 호출함. |

* `@shopby/shared > apis` 에서 제공하는 `apiCreator` 를 사용한 예시

```typescript
await executeQuery({
  queryKey: QUERY_KEY.QUERY_PRODUCT_INQUIRIES,
  queryFn: api.display.getProductsProductNoInquires,
  payload: {
    pathVariable: {
      productNo,
    },
    queryString: {
      isMyInquiries: false,
      hasTotalCount: true,
      ...restParam,
    },
    useAccessToken: isSignedIn(),
  },
  cacheTimeByMinutes: 5,
  usesLocalStorage: true
});
```

* 외부 api 사용한 예시

```typescript
await executeQuery({
    queryFn: async ({ id }) => {
      const result = await fetch(`https://jsonplaceholder.typicode.com/todos/${id}`);
      const response = await result.json();

      return response;
    },
    onSuccess: ({ data }) => {
      console.log(data);
    },
    payload: {
      id: '1',
    },
  });
```

###  :heavy_check_mark:`executeMutation(mutationOption)`

* `POST` `PUT` `DELETE` api 호출 시 사용합니다.
* `executeMutation` 은 `mutate` 를 반환합니다. 반환된 `mutate` 를 호출해 `mutation` 을 실행시킬 수 있습니다.
    * `mutate` 함수는 `mutationFn` 와 `mutationKey` 를 제외한 `mutationOption` 을 넘길 수 있습니다.
    * 콜백 실행 순서는 `executeMutation` 함수 인자로 넘긴 `option` 의 콜백이 `mutate` 함수 인자로 넘긴 `option` 콜백보다 먼저 호출됩니다.
* 인자로 `mutationOption` 을 넘겨야 합니다.

| 이름 | 필수여부 | 타입 | 기본 값 | 설명 |
| --- | ---- | --- | ---- | --- |
| mutationFn | o | function | - | api 호출 함수 |
| mutationKey | o | any |  | 고유한 키 |
| payload | x | any |  | mutationFn 함수 인자로 넘겨짐. |
| enabled | x | boolean \| function | true | mutationFn 함수 호출 여부.
ㄴ function 인 경우 반환 결과에 따라 mutationFn 을 호출함 |
| onReady | x | (context: ShopbyContext) => void | - | mutationFn 이 호출 되기 전에 호출함. |
| onSuccess | x | (context: ShopbyContext) => void | - | mutationFn 이 성공적으로 호출된 후에 호출함. |
| onError | x | (context: ShopbyContext) => void | - | mutationFn 호출이 실패했을 때 호출함. |
| onSettled | x | (context: ShopbyContext) => void | - | mutationFn 호출 후 성공/실패에 관련없이 호출함. |

* `@shopby/shared > apis` 에서 제공하는 `apiCreator` 를 사용한 예시

```typescript
const Example = () => {
  const mutate = executeMutation({
    mutationKey: MUTATION_KEY.PUT_PRODUCT_INQUIRY_BY_INQUIRY_NO,
    mutationFn: ({ inquiryNo, isSecreted, ...rest }) =>
      api!.display.putProductsInquiresInquiryNo({
        pathVariable: {
          inquiryNo,
        },
        requestBody: {
          secreted: isSecreted,
          ...rest,
        },
      }),
  });

  return (
     {
      mutate({
        onReady: (context) => {
          if (!context?.payload?.title) {
            ClientError.of({
              code: CLIENT_ERROR.PRODUCT_INQUIRY_FAIL_HAS_NO_TITLE,
              description: CLIENT_ERROR_MESSAGE[CLIENT_ERROR.PRODUCT_INQUIRY_FAIL_HAS_NO_TITLE],
            });
          }

          if (!context?.payload.content) {
            ClientError.of({
              code: CLIENT_ERROR.PRODUCT_INQUIRY_FAIL_HAS_NO_CONTENT,
              description: CLIENT_ERROR_MESSAGE[CLIENT_ERROR.PRODUCT_INQUIRY_FAIL_HAS_NO_CONTENT],
            });
          }
        },

      })
    }} >click me!
  )
}
```

###  :heavy_check_mark:ShopbyContext

* ShopbyQuery api 호출 시 반환되는 공통 타입입니다.

| 이름 | 필수여부 | 타입 | 기본 값 | 설명 |
| --- | ---- | --- | ---- | --- |
| data | o | any | - | ShopbyQuery api 의 반환 값 |
| error | o | null \| ShopbyError | null | ShopbyQuery api 호출
ㄴ 성공 시 : null 반환
ㄴ 실패 시 : ShopbyError 객체 반환 |
| isSuccess | o | boolean | true | ShopbyQuery api 호출 성공여부 |
| isError | o | boolean | false | ShopbyQuery api 호출 실패여부 |
| status | o | null \| http status code | null | api 응답 status |
| payload | x | any | - | ShopbyQuery api option 의 payload |

* shopbyError

| 이름 | 필수여부 | 타입 | 기본 값 |  |
| --- | ---- | --- | ---- | --- |
| code | x | string | - | 오류 코드
ㄴ 'CE' 프리픽스가 붙는 오류코드는 ShopbyQuery 에서 핸들링하는 오류임. 즉, 프론트오류
ㄴ api 발생 오류코드가 담김. |
| description | x | string | - | 실패 메시지 |
| statusLabel | x | string | - | HTTP\_STATUS\_CODE 의 값 |
| serverError | x | any | - | api 호출 실패 시 해당 api 의 오류가 할당됨. |
