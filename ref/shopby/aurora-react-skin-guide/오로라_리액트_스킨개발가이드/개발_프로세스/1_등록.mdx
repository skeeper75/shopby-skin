# 1️⃣등록

## 이해하기

> 본 문서는 셀러어드민 내 디자인 상품 등록에 대해 안내합니다.
>  :exclamation:스킨 저장소(GitLab) 내 소스코드를 저장할 프로젝트를 발급받기 위해서는 셀러어드민에 상품을 등록해야 합니다.

:heavy_check_mark:**GitLab이란?**
공동 소스코드 작업을 위한 변경사항 및 버전 등을 로컬에서 추적하고 관리할 수 있는 소스코드 버전 관리 시스템입니다.
디자인 스킨 소스코드를 GitLab 프로젝트에 저장해두고 디자인 스킨을 쇼핑몰에 설치하여 배포할 수 있습니다.

:heavy_check_mark:**한국어 설정방법**
언어설정 기능을 통해 GitLab을 한국어로 쉽게 이용할 수 있습니다.
[로그인 > 우측상단 아이콘 > settings > preferences > localization > 한국어 > 저장]

### 등록 경로

워크스페이스 > 셀러어드민 > 상품 > 디자인

**① 개발방식**
스킨을 개발하기 전에 먼저 특정 스킨을 기반으로 일부를 수정하여 개발할지 또는 처음부터 개발할지, 개발방식을 선택하는 항목입니다.
선택된 값에 따라 스킨 저장소에 비어있는 프로젝트가 생성되거나 선택된 스킨 소스코드가 저장된 프로젝트가 생성됩니다.
특정 스킨을 기반으로 개발하는 경우 스킨의 구분(통합형, 개별형), 빌드여부, 빌드타입 선택에 제한이 있습니다.

aurora 스킨은 통합형으로 개발되어 있고, yarn build 명령어로 실행되기 때문에 aurora 스킨을 기반으로 개발하는 경우 구분은 통합형, 빌드여부는 Y, 빌드타입은 yarn 으로 고정하여 개발해야 합니다.
마찬가지로 another 스킨을 기반으로 개발하는 경우에도 해당 스킨이 개발된 형태에 맞춰 구분은 개별형, 빌드여부는 N 으로 고정하여 개발해야 합니다.
aurora 스킨 기반의 개별형 스킨을 개발하고자 하거나 another 스킨 기반의 통합형 스킨을 개발하고자 한다면 처음부터 개발하기 방식을 선택해야 합니다.

**② 프로젝트명**
스킨 저장소에 프로젝트가 생성될 때 사용되는 항목으로 입력한 프로젝트명으로 스킨 저장소에 프로젝트가 생성됩니다.
생성된 프로젝트명은 수정할 수 없습니다.

**③ 상품명**
셀러어드민에 노출되는 항목입니다.

**④ 구분**
스킨 소스코드 저장구분을 선택하는 항목입니다.
위(①)에서 설명한 내용과 같이 개발방식에 따라 구분 선택에 제한적입니다.

**⑤ 배너정보**
스킨에서 사용하는 배너그룹을 정의한 메타정보와 배너 샘플 이미지가 포함된 정보를 등록하는 항목입니다.
스킨에는 어느 위치에서 어떠한 배너를 호출하겠다는 내용이 포함되어 있기 때문에 배너정보는 반드시 필요합니다.
배너정보 관련하여 자세한 내용은 개발하기 문서를 참고해 주시기 바랍니다.
다만 상품을 등록하는 시점에는 스킨 개발을 시작하는 단계일 수 있어 배너정보는 상품 등록 후 개발이 완료된 이후 상품 수정 화면에서 등록하면 됩니다.
특정 스킨을 기반으로 개발하는 경우 각 스킨에서 사용된 메타정보와 샘플 이미지가 자동 등록되나, 상품 수정 화면에서 등록된 파일을 삭제 후 재등록 가능합니다.

:bangbang:등록된 상품이 설치된 쇼핑몰이 존재하는 경우 배너정보 수정이 불가합니다.

**⑥ 빌드여부**
프론트엔드 프레임워크(리액트, 뷰, 앵귤러 등) 기반으로 소스코드를 작성한 경우 빌드가 필요합니다.
위(①)에서 설명한 내용과 같이 개발방식에 따라 빌드여부 선택에 제한적입니다.

**⑦ 빌드타입**
빌드가 필요한 경우에만 선택이 필요한 항목으로, 프로젝트를 빌드할 패키지 매니저에 따라 yarn build 또는 npm run build 명령어로 실행됩니다.
