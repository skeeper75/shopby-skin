# assets 경로 변경 가이드

# assets

**빌드 도구를 활용하여 최적화 되어 생성되는 파일은 필요할 경우 경로를 수정해야할 수도 있습니다.**

## package.json

**애플리케이션의 배포 URL을 지정할 수 있습니다.**

```javascript
// package.json
{
//   ...
    "homepage": "https://mypage.com/my-path"
}
```

`path`만 변경되고 도메인 설정이 안될 경우 설정에 영향을 미치는 다른 요소가 있을 수 있습니다.  이 경우 환경변수 `PUBLIC_URL`에 값을 설정하면 됩니다.

```javascript
$ PUBLIC_URL=https://mypage.com/my-path yarn build
```

## webpack

**웹팩을 사용할 경우 웹팩 옵션에서 경로를 지정할 수 있습니다.**

```javascript
// webpack.config.js
import webpack from 'webpack';

// 환경 변수를 사용하고 존재하지 않는다면 루트를 사용하세요.
const ASSET_PATH = process.env.ASSET_PATH || '/';

export default {
  output: {
    publicPath: ASSET_PATH,
  },

  plugins: [
    // 코드에서 환경 변수를 안전하게 사용할 수 있습니다.
    new webpack.DefinePlugin({
      'process.env.ASSET_PATH': JSON.stringify(ASSET_PATH),
    }),
  ],
};
```

* [webpack 참고](https://webpack.kr/guides/public-path/)

## vite

**vite를 사용할 경우 vite 옵션에서 경로를 지정할 수 있습니다.**

```javascript
// vite.config.js
// 환경 변수를 사용하고 존재하지 않는다면 루트를 사용하세요.
const ASSET_PATH = process.env.ASSET_PATH || '/';

export default {
    base: ASSET_PATH,
}
```

* [vite 참고](https://ko.vite.dev/guide/build.html#public-base-path)
