## 1. 정책 요약

본 문서는 인쇄 주문의 전체 워크플로우를 정의한다. 고객이 상품을 선택한 후 파일 업로드부터 결제, 파일검수, 인쇄, 배송까지의 전 과정을 다룬다.

**핵심 정책 방향**:
- 회원/비회원 모두 주문 가능
- 선결제 후 제작 원칙 (B2B 후불결제는 별도 정책)
- 파일검수 완료 후 제작 착수
- 주문 취소는 파일검수 완료 전까지 가능
- 재주문 파일 보관 기간: 최소 1개월

## 4. UserFlow

### 4.1 회원 주문 UserFlow

```mermaid
flowchart TD
    A[상품 선택] --> B[인쇄 옵션 선택<br/>용지/수량/후가공]
    B --> C[파일 업로드<br/>PDF/AI/PSD]
    C --> D[미리보기 확인]
    D --> E{장바구니 담기?}
    E -->|예| F[장바구니]
    E -->|아니오| G[바로 주문]
    F --> G
    G --> H[배송정보 입력<br/>저장된 배송지 선택]
    H --> I[결제수단 선택]
    I --> J[결제 완료]
    J --> K[주문완료 화면<br/>주문번호 안내]
    K --> L[알림톡 발송<br/>주문확인 + 예상납기]
    L --> M[파일 검수 대기]
```

### 4.2 비회원 주문 UserFlow

```mermaid
flowchart TD
    A[상품 선택] --> B[인쇄 옵션 선택]
    B --> C[파일 업로드]
    C --> D[미리보기 확인]
    D --> E[바로 주문<br/>장바구니 세션 유지]
    E --> F[배송정보 직접 입력]
    F --> G[주문자 정보 입력<br/>이름/전화번호/이메일]
    G --> H[결제수단 선택]
    H --> I[결제 완료]
    I --> J[주문완료 화면<br/>주문번호 + 비밀번호 안내]
    J --> K[알림톡 발송]

    L[비회원 주문조회] --> M[주문번호 + 휴대전화 입력]
    M --> N[주문 상세 확인]
```

### 4.3 파일검수 UserFlow (관리자)

```mermaid
flowchart TD
    A[신규 주문 접수] --> B[파일 자동검수<br/>해상도/색상모드]
    B --> C{자동검수 통과?}
    C -->|통과| D[담당자 수동검수<br/>재단선/여백/서체]
    C -->|불통과| E[재업로드 요청<br/>알림톡 + 이메일]
    E --> F{재업로드 횟수?}
    F -->|3회 미만| G[고객 파일 재업로드]
    G --> B
    F -->|3회 이상| H[고객 상담 안내<br/>1:1 문의 유도]
    D --> I{수동검수 통과?}
    I -->|통과| J[파일 승인]
    I -->|불통과| E
    J --> K[작업지시서 생성<br/>PDF 출력]
    K --> L[제작 착수]
```

## 8. MES 기반 주문 상태 정의

### 8.1 MES 주문 상태 전환 흐름

MES(Manufacturing Execution System) 기반의 주문 상태는 아래 흐름을 따른다.

```
미입금 → 결제완료 → 제작대기 → 제작중 → 제작완료 → 출고완료
                                                    ↘ 주문취소
```

```mermaid
stateDiagram-v2
    [*] --> 미입금
    미입금 --> 결제완료 : 입금확인/카드결제
    미입금 --> 주문취소 : 미입금 자동취소

    결제완료 --> 제작대기 : 파일검수 완료
    결제완료 --> 주문취소 : 고객 취소 요청

    제작대기 --> 제작중 : 작업 착수
    제작중 --> 제작완료 : 공정 완료
    제작완료 --> 출고완료 : 택배발송/방문수령

    주문취소 --> [*]
    출고완료 --> [*]
```

### 8.2 파일번호 상태 관리

각 주문의 파일번호는 입고 경로에 따라 3가지 Case로 상태가 전환된다.

| Case | 경로 | 상태 전환 |
|------|------|-----------|
| Case 1 | 배송 (파일 없음) | 준비중 → 포장완료 |
| Case 2 | 파일업로드 | 확인전 → 파일확인 → 재업로드 → 다운로드완료 → 포장완료 |
| Case 3 | 편집기 | 렌더링전 → 렌더링완료 → 수정 → 다운로드 → 포장완료 |

### 8.3 파일명 규칙

생산 현장에서 사용하는 파일명은 아래 규칙을 따른다:

```
{품목}_{출력사이즈}_{양단면}_{소재명}_{거래처명}_{고객명}_{파일고유번호}_{수량}
```

**예시**: `명함_90x50_양면_아트지250g_후니프린팅_홍길동_F20260315001_200매`

## 10. 클레임 처리 정책

### 10.1 클레임 유형 및 처리 방법

| 유형 | 세부 구분 | 처리 방법 | 비고 |
|------|-----------|-----------|------|
| **재제작** | 일부 재제작 | 불량 파일번호만 재제작 | 원본 파일 기준 |
| **재제작** | 전체 재제작 | 전체 주문 재제작 | 제작 오류 시 |
| **주문취소** | 부분환불 | 일부 상품만 취소/환불 | 미제작 분만 |
| **주문취소** | 전체환불 | 전체 주문 취소/환불 | 결제수단별 처리 |
| **반송** | Nfocus 반송 | 불량품 Nfocus 택배로 회수 | 회수 후 검수 |
| **재출고** | 누락분 출고 | 누락된 상품 추가 출고 | 검수 확인 후 |
| **재출고** | 교환 출고 | 불량 교환 상품 출고 | 반송 확인 후 |

### 10.2 정산 처리

- 클레임 발생 시 관리자 **메모 필드**에 클레임 사유 및 처리 내역 기록
- 환불 금액, 재제작 비용, 반송 비용 등은 정산 시 반영
- 클레임 이력은 주문 상세에서 조회 가능

### 10.3 클레임 처리 흐름도

```mermaid
flowchart TD
    A[클레임 접수] --> B{클레임 유형?}
    B -->|인쇄 불량| C{불량 범위?}
    B -->|배송 누락| D[누락분 확인]
    B -->|파손/훼손| E[반송 처리]
    B -->|고객 변심| F{제작 착수?}

    C -->|일부 불량| G[일부 재제작]
    C -->|전체 불량| H[전체 재제작]

    D --> I[재출고 - 누락분]

    E --> J[Nfocus 반송 접수]
    J --> K[불량 검수]
    K --> L{검수 결과?}
    L -->|재제작| H
    L -->|교환| M[재출고 - 교환]

    I --> Q
    M --> Q
    N --> Q
    O --> Q
    P --> Q

    Q --> R[클레임 종결]
```

## 11. Pain Points (현행 운영 이슈)

### 개발 우선순위 (CUSTOM 항목)

1. **파일 업로드 모듈**: 인쇄 파일 업로드/저장/미리보기 (핵심 기능)
2. **파일 검수 워크플로우**: 자동검수 + 수동검수 + 재업로드 요청
3. **작업지시서 PDF**: 주문정보 + 인쇄옵션 + 파일정보 포함
4. **커스텀 주문 상태**: 인쇄 공정별 상태 추가
5. **후불결제/정산**: B2B 고객용 (2차 개발)
6. **디자인의뢰**: 워크플로우 설계 후 개발 (2차 이후)

### 주요 커스텀 개발 영역

- **파일 스토리지**: AWS S3 또는 NHN Cloud Object Storage (인쇄 파일 전용)
- **파일 검수 엔진**: 해상도 확인, 색상모드(CMYK) 확인, 재단선 확인
- **작업지시서 생성**: PDF 라이브러리 활용 (jsPDF 또는 서버사이드 생성)
- **인쇄 상태 관리**: shopby 주문 상태 확장 또는 별도 상태 테이블
- **알림 연동**: 카카오 알림톡 API 연동 (주요 상태 변경 시 자동 발송)
