## 1. 배송비 정책

### 핵심 정책 (What)

> 기본적으로 10만원 이상 주문 시 무료배송이며, 대형·박스 단위 상품(배너, 쿠션, 머그컵)은 별도 배송비를 부과한다.

### 정책 상세 (Detail)

**기본 규칙**

- 주문 금액 10만원 이상: 무료배송 ✅
- 주문 금액 10만원 미만: 기본 배송비 부과

**상품별 예외 배송비**

| 상품 | 박스 기준 | 1박스 | 2박스 | 3박스 이상 |
|------|-----------|-------|-------|------------|
| 배너/실외거치대 | 개당 1박스 | 15,000원 | 12,000원/개 (6~10개) | 10,000원/개 (11개+) |
| 머그컵 | 36개 = 1박스 | 5,000원 | 7,500원 | 10만원 이상 시 무료 |
| 쿠션 | 10개 = 1박스 | 3,000원 | 5,500원 | 10만원 이상 시 무료 |

> ⚠️ 배너는 주문 금액과 관계없이 무료배송 미적용 (대형 물품 특성상)

**지역 추가 배송비**

| 지역 | 추가 금액 |
|------|-----------|
| 제주도 | +5,000원 |
| 도서산간 | +3,000원 ~ +10,000원 |

**혼합 주문 규칙**

- 여러 상품을 함께 주문할 경우: 가장 높은 배송비 1건만 적용

**고객 안내 문구 (예시)**

> "배너·실외거치대는 대형 물품 특성상 무료배송 혜택이 적용되지 않으며, 수량에 따라 배송비가 부과됩니다. 제주·도서산간 지역은 추가 배송비가 발생합니다."

### 의사결정 필요사항 (Decision Required)

- [ ] 배너/실외거치대 무료배송 완전 제외 여부 최종 확정
- [ ] 혼합 주문 시 배송비 계산 방식 확인 (가장 높은 1건 적용 방식 동의 여부)
- [ ] 도서산간 지역별 세부 금액 확정 (3,000원~10,000원 범위 내)

### shopby 관리자 설정 (How - 관리자)

1. **기본 배송비**: 관리자 > 배송설정 > 배송비 정책 > 기본배송비 설정
2. **상품별 배송비**: 상품 관리 > 해당 상품 선택 > 배송옵션 > 개별배송비 설정
3. **지역 추가 배송비**: 관리자 > 배송설정 > 지역별 추가배송비 > 지역 추가

### 개발 필요사항 (How - 개발자)

- 배너 상품 카테고리 식별 로직 (무료배송 제외 처리)
- 박스 단위 자동 계산 함수 (수량 입력 → 박스 수 → 배송비 산출)
- 혼합 주문 배송비 비교 로직 (최고값 적용)
- 상세 구현은 POLICY-SHIPPING-PAYMENT-REVIEW-001 참조

## 2. 결제방법 정책

### 핵심 정책 (What)

> 기존 결제수단에 간편결제 3사(네이버페이, 카카오페이, 토스페이)를 순차적으로 추가한다.

### 정책 상세 (Detail)

**추가 결제수단 및 수수료**

| 결제수단 | 예상 수수료 | 도입 순서 |
|----------|-------------|-----------|
| 네이버페이 | 약 2.9% | 1순위 |
| 카카오페이 | 약 2.8% | 2순위 (1주 후) |
| 토스페이 | 약 2.5% | 3순위 (2주 후) |

> 📌 수수료는 계약 조건에 따라 변동될 수 있으며, 실제 계약 시 협상 가능합니다.

**도입 일정**

- 네이버페이 먼저 추가 → 1주일 운영 확인 → 카카오페이 추가 → 1주일 후 토스페이 추가

**고객 안내 문구 (예시)**

> "이제 네이버페이, 카카오페이, 토스페이로도 간편하게 결제하실 수 있습니다."

### 의사결정 필요사항 (Decision Required)

- [ ] 각 PG사 사업자 등록 및 계약 진행 여부 확인
- [ ] 도입 순서 및 일정 최종 확정
- [ ] 각 PG사 수수료 협상 후 최종 요율 확정
- [ ] 결제 실패 시 고객 안내 방식 결정

### shopby 관리자 설정 (How - 관리자)

1. **결제수단 추가**: 관리자 > 결제관리 > 결제수단 설정 > 간편결제 항목에서 각 PG 활성화
2. **PG 키 입력**: 각 PG사에서 발급받은 API 키를 해당 설정 항목에 입력
3. **테스트 결제 확인**: 설정 완료 후 테스트 결제로 정상 작동 확인

### 개발 필요사항 (How - 개발자)

- 각 PG사 SDK 연동 (네이버페이 JS SDK, 카카오페이 API, 토스페이 SDK)
- 결제 성공/실패 웹훅(Webhook) 처리 로직
- 결제 내역 주문 시스템 연동
- 상세 구현은 POLICY-SHIPPING-PAYMENT-REVIEW-001 참조

## 3. 할인쿠폰 정책

### 핵심 정책 (What)

> 회원 행동(신규 가입, 리뷰 미작성, 재구매, VIP 달성) 조건 충족 시 쿠폰을 자동으로 발행한다.

### 정책 상세 (Detail)

**쿠폰 종류 및 조건**

| 쿠폰명 | 금액 | 최소 주문금액 | 발행 조건 | 유효기간 |
|--------|------|---------------|-----------|----------|
| 신규회원 환영 | 10,000원 | 50,000원 | 회원가입 완료 시 | 30일 |
| 리뷰 유도 | 5,000원 | 30,000원 | 배송완료 14일 후 미작성 시 | 30일 |
| 재구매 장려 | 20,000원 | 100,000원 | 해당 월 200,000원 이상 구매 시 | 30일 |
| VIP 보너스 | 30,000원 | 150,000원 | 연간 누적 1,000,000원 이상 시 | 30일 |

**사용 제한 규칙**

- 회원당 동일 쿠폰 유형: 월 1회만 발행
- 한 주문에 동시 사용: 최대 3개
- 같은 카테고리 쿠폰: 중복 사용 불가

**고객 안내 문구 (예시)**

> "회원 혜택 쿠폰이 발행되었습니다. 마이페이지 > 쿠폰함에서 확인하세요. 유효기간: 발행일로부터 30일"

### 의사결정 필요사항 (Decision Required)

- [ ] 각 쿠폰 금액 최종 확정 (현행 안 동의 여부)
- [ ] 최소 주문금액 조건 최종 확정
- [ ] 리뷰 유도 쿠폰 발행 시점 결정 (현행: 배송완료 14일 후)
- [ ] 쿠폰 유효기간 결정 (현행: 30일)
- [ ] VIP 기준 금액 결정 (현행: 연간 100만원)

### shopby 관리자 설정 (How - 관리자)

1. **쿠폰 생성**: 관리자 > 프로모션 > 쿠폰 관리 > 쿠폰 추가
2. **각 쿠폰 설정**: 할인 금액, 최소 주문금액, 유효기간, 발급 조건 입력
3. **발급 방식**: 자동발급 설정 항목에서 조건 지정

> ⚠️ 자동 발행(배치 처리)은 shopby 관리자 화면만으로는 불가. 개발 지원 필요.

### 개발 필요사항 (How - 개발자)

- 쿠폰 자동 발행 배치 스케줄러 개발
  - 회원가입 이벤트 훅 연동 (신규회원 쿠폰)
  - 일별 배치: 배송완료 14일 경과 + 미작성 회원 조회 (리뷰 유도 쿠폰)
  - 월말 배치: 월 구매 200,000원 이상 회원 조회 (재구매 쿠폰)
  - 연말 배치: 연간 1,000,000원 이상 회원 조회 (VIP 쿠폰)
- shopby 쿠폰 발급 API 호출 로직
- 상세 구현은 POLICY-SHIPPING-PAYMENT-REVIEW-001 참조

## 4. 구매후기(리뷰) 정책

### 핵심 정책 (What)

> 배송완료 주문에 대해 고객이 리뷰 작성 시 즉시 등록·노출되며, 자동으로 5,000원 쿠폰이 발행된다. 관리자는 부적절한 리뷰를 사후에 삭제한다.

### 정책 상세 (Detail)

**리뷰 운영 흐름**

1. 배송완료 주문에만 리뷰 작성 버튼 표시
2. 고객이 리뷰 작성 → 관리자 승인 없이 즉시 등록 및 노출
3. 리뷰 등록 즉시 → 5,000원 쿠폰 자동 발행
4. 관리자가 정기적으로 리뷰 점검 → 부적절한 리뷰 삭제

**리뷰 작성 가능 조건**

| 조건 | 내용 |
|------|------|
| 주문 상태 | 배송완료 상태여야 함 |
| 작성 가능 기간 | 배송완료 후 60일 이내 |
| 작성 횟수 | 동일 상품 1회만 가능 |

**관리자 삭제 기준**

| 삭제 사유 | 처리 |
|-----------|------|
| 욕설·비방·혐오 표현 | 즉시 삭제 |
| 개인정보 노출 (전화번호, 주소 등) | 즉시 삭제 |
| 스팸·광고성 내용 | 즉시 삭제 |
| 상품과 무관한 내용 | 삭제 |

**쿠폰 회수 규칙**

| 상황 | 처리 방식 |
|------|-----------|
| 리뷰 삭제 + 쿠폰 미사용 | 쿠폰 즉시 회수 |
| 리뷰 삭제 + 쿠폰 이미 사용 | ⚠️ 환불 처리 여부 결정 필요 |

**고객 안내 문구 (예시)**

> "리뷰 작성 시 5,000원 쿠폰이 즉시 지급됩니다. 단, 부적절한 리뷰(욕설, 광고, 개인정보 포함 등)는 삭제될 수 있으며 쿠폰이 회수됩니다."

### 의사결정 필요사항 (Decision Required)

- [ ] 리뷰 즉시 등록·노출 방식 최종 확인 (사전 승인 없음에 동의하는지)
- [ ] 리뷰 작성 가능 기간 확정 (현행: 배송완료 후 60일)
- [ ] 리뷰 보상 쿠폰 금액 확정 (현행: 5,000원)
- [ ] **쿠폰 사용 후 리뷰 삭제 시 환불 처리 여부 결정** (주요 결정사항)
- [ ] 관리자 리뷰 점검 주기 결정 (일별·주별 등)

### shopby 관리자 설정 (How - 관리자)

1. **리뷰 관리 메뉴**: 관리자 > 마켓플레이스 > 상품문의/후기 > 후기관리
2. **리뷰 노출 설정**: 후기관리에서 노출 여부 설정 (즉시 노출 또는 승인 후 노출 선택)
3. **부적절 리뷰 삭제**: 해당 리뷰 선택 > 삭제 처리

### 개발 필요사항 (How - 개발자)

- 리뷰 등록 시 쿠폰 자동 발행 이벤트 연동
- 관리자 리뷰 삭제 시 미사용 쿠폰 회수 처리
- 배송완료 상태 확인 후 리뷰 버튼 노출 로직
- 60일 기간 체크 로직
- 상세 구현은 POLICY-SHIPPING-PAYMENT-REVIEW-001 참조
