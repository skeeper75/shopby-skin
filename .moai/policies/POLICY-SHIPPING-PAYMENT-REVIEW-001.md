## 1. 배송비 정책

### 1.1 정책 개요

**정책명**: 차등 배송비 정책
**목표**: 상품 특성에 맞는 합리적인 배송비 정책으로 고객만족도 향상
**원칙**:
- 기본 배송비: 10만원 이상 무료배송
- 상품 특성별 예외 배송비 부과
- 상품별 박스 단위 배송비 관리

### 1.2 shopby 기능

#### 1.2.1 지역별 추가배송비 설정
**API**: `POST /areafees` (delivery-server)
**기능**: 지역별 추가배송비 설정 생성

```
POST https://server-api.e-ncp.com/areafees
Header: systemKey, Authorization, Version

Request Body:
{
  "areaNo": 198,      // 지역 코드 (제주, 도서산간 등)
  "deliveryFee": 5000 // 추가 배송비
}
```

#### 1.2.2 상품별 배송비 설정
**API**: 상품 정보 수정 시 배송 옵션 설정
**기능**: 상품별 배송비 개별 설정 가능

### 1.3 실제 구현 방식

#### 1.3.1 기본 배송비 정책 (10만원 이상 무료배송)
```
최소주문금액: 100,000원
배송비: 0원 (무료)
조건: 상품 정가 기준 100,000원 이상
```

#### 1.3.2 배너/실외거치대 배송비
**분류**: 대형 물품
**박스 단위**: 개당 1박스
**배송비 체계**:

| 수량 범위 | 배송비 | 비고 |
|----------|--------|------|
| 1 ~ 5 박스 | 개당 15,000원 | 소량 배송 추가비 |
| 6 ~ 10 박스 | 개당 12,000원 | 중량 배송 할인 |
| 11 박스 이상 | 개당 10,000원 | 대량 배송 할인 |

**구현**: 상품 정보 > 배송설정 > 상품별 배송비 개별 설정

#### 1.3.3 머그컵 배송비
**분류**: 일반 상품 (박스 단위)
**박스 환산**: 1 박스 = 36개 머그컵 (표준 박스 사양)
**배송비 체계**:

| 박스 수량 | 배송비 | 비고 |
|----------|--------|------|
| 1 박스 | 5,000원 | |
| 2 박스 | 7,500원 | |
| 3 박스 이상 | 무료 (100,000원 이상) | |

**구현**:
- 상품 옵션에 "박스 단위" 표시
- 장바구니에서 박스 수량 기반 배송비 자동 계산

#### 1.3.4 쿠션 배송비
**분류**: 일반 상품 (박스 단위)
**박스 환산**: 1 박스 = 10개 쿠션 (표준 박스 사양)
**배송비 체계**:

| 박스 수량 | 배송비 | 비고 |
|----------|--------|------|
| 1 박스 | 3,000원 | |
| 2 박스 | 5,500원 | |
| 3 박스 이상 | 무료 (100,000원 이상) | |

**구현**:
- 상품 옵션에 "박스 단위" 표시
- 수량별 자동 배송비 계산

### 1.4 예외 처리

#### 1.4.1 혼합 주문 배송비
**규칙**: 배송비가 서로 다른 상품 주문 시
- 가장 높은 배송비만 부과
- 각 상품의 박스 단위 수량 합산 후 계산

**예시**:
```
주문 내역:
- 배너 1개 (15,000원) + 머그컵 2박스 (7,500원) = 22,500원
-> 배송비: 15,000원 (가장 높은 배송비만 적용)
```

#### 1.4.2 도서산간/제주 추가배송비
**규칙**: 기본 배송비 + 지역별 추가 배송비
- 제주도: +5,000원
- 도서산간: +3,000 ~ 10,000원 (지역별 상이)
- 강화/옹진군 등 특수지역: +5,000원

**구현**: `POST /areafees` API로 지역별 추가비 사전 설정

#### 1.4.3 100,000원 이상 무료배송 조건
**기준금액**: 상품 정가 기준 (할인 미적용)
**상품군별 예외**:
- 배너: 기본 배송비 적용 (무료배송 제외)
- 머그컵/쿠션: 100,000원 이상 무료배송 적용

**규칙 재검토 필요**: 배너의 무료배송 제외 여부는 비즈니스 정책에 따라 조정 가능

### 1.5 관리자 운영 프로세스

#### 1.5.1 배송비 설정 (초기)
1. shopby 관리자 > 배송설정 > 기본배송비 설정
   - 기본 배송료: 0원 (10만원 이상 무료)
   - 최소주문금액: 100,000원

2. 상품별 배송비 설정
   - 배너 상품: 상품 정보 > 배송옵션 > "배너/실외거치대" 설정
   - 머그컵: 상품 정보 > 배송옵션 > "머그컵" 설정
   - 쿠션: 상품 정보 > 배송옵션 > "쿠션" 설정

3. 지역별 추가배송비 설정
   - `POST /areafees` API 호출 또는 관리자 UI에서 설정
   - 제주도, 도서산간 등 특수지역 등록

#### 1.5.2 배송비 수정 프로세스
**변경 권한**: 마케팅/배송 담당자
**절차**:
1. 변경 사유 문서화
2. shopby 관리자 시스템에서 배송비 수정
3. 고객 안내 공지사항 작성
4. 변경 로그 기록

#### 1.5.3 배송비 관리 체크리스트
- 월 1회 배송비 정책 검토
- 계절별 배송비 조정 필요성 검토
- 신상품 추가 시 배송비 카테고리 할당
- 지역별 추가배송비 최신성 유지

### 1.6 정책 결정 체크리스트

- [ ] 배너/실외거치대 배송비 정책 최종 승인
  - 1 ~ 5 박스: 개당 15,000원
  - 6 ~ 10 박스: 개당 12,000원
  - 11 박스 이상: 개당 10,000원

- [ ] 머그컵 배송비 정책 최종 승인
  - 1 박스(36개): 5,000원
  - 2 박스: 7,500원
  - 3 박스 이상: 무료배송 (100,000원 이상)

- [ ] 쿠션 배송비 정책 최종 승인
  - 1 박스(10개): 3,000원
  - 2 박스: 5,500원
  - 3 박스 이상: 무료배송 (100,000원 이상)

- [ ] 도서산간/제주 추가배송비 금액 최종 확정

## 2. 결제방법 정책

### 2.1 정책 개요

**정책명**: 다양한 간편결제 수단 제공
**목표**: 고객 결제 편의성 향상으로 전환율 증대
**원칙**:
- 주요 간편결제 3사 결제 수단 제공
- 결제 수단별 수수료 관리
- 결제 오류 최소화

### 2.2 shopby 기능

#### 2.2.1 결제 수단 관리
**관리 위치**: shopby 관리자 > 결제관리 > 결제수단 설정
**현재 지원 수단** (기본 설정):
- 신용카드
- 무통장입금 (계좌이체)
- 휴대폰 결제

#### 2.2.2 PG 연동 (Payment Gateway)
**기능**: 외부 결제사(KG이니시스, NHN KCP 등)와 연동
**지원 범위**: 신용카드, 체크카드, 계좌이체

### 2.3 실제 구현 방식

#### 2.3.1 네이버페이 통합
**추가 수단**: 네이버페이 (네이버 계정 기반)

**필수 사항**:
- 네이버 상점 등록 및 API 키 발급
- shopby PG 설정에서 네이버페이 연동 설정
- 결제 테스트 후 운영환경 적용

**shopby 설정**:
```
관리자 > 결제관리 > 결제수단 설정
- 결제수단: 네이버페이
- PG: 네이버페이 API 선택
- 수수료율: 2.9% (기본값, 계약 조건 따라 변동)
- 활성화: ON
```

#### 2.3.2 카카오페이 통합
**추가 수단**: 카카오페이 (카카오톡 기반)

**필수 사항**:
- 카카오 비즈니스 계정 등록
- 카카오 결제 API 키 발급
- shopby PG 설정에서 카카오페이 연동 설정

**shopby 설정**:
```
관리자 > 결제관리 > 결제수단 설정
- 결제수단: 카카오페이
- PG: 카카오페이 API 선택
- 수수료율: 2.8% (기본값, 계약 조건 따라 변동)
- 활성화: ON
```

#### 2.3.3 토스페이 통합
**추가 수단**: 토스페이 (토스 앱 기반)

**필수 사항**:
- 토스 비즈니스 계정 등록
- 토스 결제 API 키 발급
- shopby PG 설정에서 토스페이 연동 설정

**shopby 설정**:
```
관리자 > 결제관리 > 결제수단 설정
- 결제수단: 토스페이
- PG: 토스 API 선택
- 수수료율: 2.5% (기본값, 계약 조건 따라 변동)
- 활성화: ON
```

#### 2.3.4 결제 수단 활성화 순서

**Phase 1 (1주차)**: 네이버페이
- 개발 환경 테스트
- 운영 환경 배포 및 모니터링
- 고객 공지

**Phase 2 (2주차)**: 카카오페이
- Phase 1 안정성 확인 후 진행
- 동일 프로세스 진행

**Phase 3 (3주차)**: 토스페이
- Phase 1, 2 안정성 확인 후 진행
- 동일 프로세스 진행

### 2.4 예외 처리

#### 2.4.1 결제 오류 처리
**규칙**: 결제 실패 시 자동 재시도

| 결제 수단 | 재시도 횟수 | 재시도 간격 |
|---------|-----------|----------|
| 네이버페이 | 3회 | 5초 |
| 카카오페이 | 3회 | 5초 |
| 토스페이 | 3회 | 5초 |

**고객 안내**: "결제가 실패했습니다. 다른 결제수단을 선택하거나 다시 시도해주세요."

#### 2.4.2 결제 수단 장애
**규칙**: 특정 결제 수단 장애 시

- 자동 비활성화 (관리자 수동 개입 없이)
- 고객 결제 화면에서 해당 수단 제거
- 대체 결제 수단 제시
- 운영팀에 알림 발송

#### 2.4.3 테스트 카드/계정 관리
**규칙**: 운영 환경에서 테스트 카드 결제 차단

- 개발/테스트 환경만 테스트 카드 허용
- 운영 환경: 실카드만 결제 가능

### 2.5 관리자 운영 프로세스

#### 2.5.1 결제 수단 추가 설정
1. shopby 관리자 로그인
2. 결제관리 > 결제수단 설정 메뉴
3. 새 결제수단 추가 버튼 클릭
4. 해당 PG 선택 및 API 키 입력
5. 수수료율 설정
6. 테스트 결제 수행
7. 운영환경 활성화

#### 2.5.2 결제 수단 모니터링
**주기**: 일일 1회 이상
**확인 항목**:
- 각 결제 수단별 정상 작동 여부
- 결제 성공률 (목표: 98% 이상)
- 결제 실패 원인 분석
- 고객 결제 이슈 대응

#### 2.5.3 결제 통계 분석
**주기**: 주 1회
**분석 항목**:
- 결제 수단별 거래액 및 거래건수
- 결제 수단별 성공/실패율
- 고객 선호 결제 수단 추이

### 2.6 정책 결정 체크리스트

- [ ] 네이버페이 결제 통합 승인 및 API 키 확보
- [ ] 카카오페이 결제 통합 승인 및 API 키 확보

- [ ] 개발 환경 테스트 완료

## 3. 할인쿠폰 정책

### 3.1 정책 개요

**정책명**: 조건부 자동발행 쿠폰 정책
**목표**: 고객 재구매율 향상 및 마케팅 효율성 강화
**원칙**:
- 특정 조건 충족 시 자동 발행
- 사용 기한 명시
- 중복 발행 방지

### 3.2 shopby 기능

#### 3.2.1 쿠폰 생성 API
**API**: `POST /coupons` (promotion-server)
**기능**: 쿠폰 신규 생성

```
POST https://server-api.e-ncp.com/coupons
Header: systemKey, Authorization, Version

쿠폰 타입:
- PRODUCT: 상품적용 쿠폰
- PRODUCT_PLUS: 상품 금액 할인(플러스)
- CART: 주문적용 쿠폰
- CART_DELIVERY: 장바구니 배송비 할인
- GIFT: 기프트 쿠폰

발급 타입:
- DOWNLOAD: 다운로드 발급
- CODE_DESIGNATE: 지정코드 발급
- CODE_RANDOM: 난수코드 발급
- INSERT: 인서트 발급 (회원별 자동 발행)
```

#### 3.2.2 쿠폰 조회 API
**API**: `GET /coupons` (promotion-server)
**기능**: 검색 조건 기반 쿠폰 조회

```
검색 조건:
- 쿠폰번호, 쿠폰명
- 등록일, 발행시작일, 발행종료일
- 쿠폰 타입, 발급 타입
- 쿠폰 상태 (ISSUE_READY, ISSUE_ING, ISSUE_STOP, ISSUE_END)
```

### 3.3 실제 구현 방식

#### 3.3.1 자동발행 쿠폰 조건 정의

**조건 1: 첫 구매 신규회원**
```
조건: 회원가입 후 첫 주문 완료
쿠폰: 10,000원 할인 쿠폰 (최소주문금액 50,000원)
발행 방식: INSERT (자동발행)
기한: 30일
```

**조건 2: 배송완료 후 리뷰 미작성**
```
조건: 주문 배송완료 후 14일 경과 && 리뷰 미작성
쿠폰: 5,000원 할인 쿠폰 (최소주문금액 30,000원)
발행 방식: INSERT (자동발행)
기한: 14일
```

**조건 3: 월별 재구매 장려**
```
조건: 월 내 누적 구매금액 200,000원 이상
쿠폰: 20,000원 할인 쿠폰 (최소주문금액 100,000원)
발행 방식: INSERT (자동발행)
기한: 다음 월말
```

**조건 4: 회원등급별 보너스**
```
조건: VIP 회원 등급 (연간 구매금액 1,000,000원 이상)
쿠폰: 30,000원 할인 쿠폰 (최소주문금액 150,000원)
발행 방식: INSERT (자동발행)
기한: 3개월
매월 자동 재발행
```

#### 3.3.2 자동발행 구현 방식

**호출 방식**: 백엔드 배치 프로세스

```
일일 자동발행 배치 (자정)
1. 오늘 신규 주문 조회
2. 조건별 자동발행 대상 회원 필터링
3. POST /coupons API로 쿠폰 생성
4. 회원에게 발행 알림 메일 발송
5. 실패한 발행 로그 기록
```

**호출 예시**:
```javascript
POST /coupons
{
  "couponName": "신규회원 환영 쿠폰",
  "couponType": "CART",
  "issueType": "INSERT",
  "discountType": "FIXED_AMOUNT",
  "discountValue": 10000,
  "minOrderPrice": 50000,
  "issueStartYmd": "2026-03-15",
  "issueEndYmd": "2026-04-14",
  "memberNos": [123, 456, 789],  // 대상 회원
  "issuableCount": 1000,
  "issueablePerMember": 1
}
```

#### 3.3.3 중복 발행 방지
**규칙**: 회원별 월 1회만 발행

- 같은 유형 쿠폰 발행 기록 확인

## 4. 구매후기(리뷰) 정책

### 4.1 정책 개요

**정책명**: 즉시등록 + 자동쿠폰발행 리뷰 정책
**목표**: 리뷰를 통한 사용자 신뢰도 향상 및 구매 전환율 증대
**원칙**:
- 배송완료 상태 주문만 리뷰 가능
- 리뷰 업로드 시 즉시 노출 (관리자 사후 승인 필수)
- 리뷰 작성 시 자동 쿠폰 발행
- 관리자 삭제 기능 및 쿠폰 회수

### 4.2 shopby 기능

#### 4.2.1 리뷰 관리 API
**서버**: display-server 또는 product-server
**기능**: 리뷰 생성, 조회, 삭제

```
리뷰 생성: POST /products/{productNo}/notices
리뷰 조회: GET /products/{productNo}/notices
리뷰 삭제: DELETE /products/{productNo}/notices/{noticeNo}
```

#### 4.2.2 주문 상태 조회 API
**API**: `GET /orders` (order-server)
**기능**: 주문 상태별 조회

```
주문 상태:
- CREATED: 주문 생성
- PAYMENT_COMPLETED: 결제 완료
- PREPARING: 배송 준비 중
- SHIPPED: 배송 중
- DELIVERED: 배송 완료
- CANCELLED: 취소
- REFUNDED: 환불
```

### 4.3 실제 구현 방식

#### 4.3.1 리뷰 노출 프로세스

**Phase 1: 리뷰 업로드**
```
사용자 > 주문 상세보기 > "리뷰 작성" 버튼
-> 리뷰 입력 폼 (제목, 내용, 이미지, 평점)
-> 제출
```

**Phase 2: 즉시 등록 (임시 상태)**
```
리뷰 상태: "PENDING_REVIEW" (임시 등록)
노출 상태: 비공개 (관리자만 조회 가능)
쿠폰 발행: 즉시 발행 (임시 상태에서)
```

**Phase 3: 관리자 승인**
```
관리자 > 콘텐츠 관리 > 리뷰 관리
-> 신규 리뷰 목록에서 검토
-> "승인" 또는 "거절" 선택
-> 승인 시: 상태를 "APPROVED"로 변경, 공개 노출
-> 거절 시: 상태를 "REJECTED"로 변경, 쿠폰 회수
```

**Phase 4: 공개 노출**
```
리뷰 상태: "APPROVED"
노출 대상: 전체 고객 (상품 상세 페이지에 표시)
쿠폰 상태: 사용 가능
```

#### 4.3.2 리뷰 작성 조건
**규칙**: 다음 조건을 모두 만족해야 리뷰 작성 가능

```
✓ 주문 상태 = "DELIVERED" (배송 완료)
✓ 배송 완료 후 60일 이내
✓ 해당 상품별 1회만 리뷰 가능 (중복 방지)
✓ 리뷰 미작성 상태의 주문
✗ 취소/환불/반품 상태 주문은 리뷰 불가
```

#### 4.3.3 리뷰 버튼 노출 조건
**규칙**: 배송완료 상태인 주문만 리뷰 버튼 표시

```
Frontend 로직:
if (orderStatus === "DELIVERED") {
  showReviewButton = true;
} else {
  showReviewButton = false;
  showReviewButton_Tooltip = "배송 완료 후 리뷰를 작성할 수 있습니다";
}
```

#### 4.3.4 자동쿠폰발행
**규칙**: 리뷰 작성 시점에 자동 발행

```
리뷰 작성 API 호출
  ↓
리뷰 저장 (PENDING_REVIEW 상태)
  ↓
쿠폰 발행 API 호출: POST /coupons
  {
    "couponType": "CART",
    "discountValue": 5000,
    "minOrderPrice": 30000,
    "issueType": "INSERT",
    "memberNo": {리뷰 작성자},
    "issueStartYmd": "{today}",
    "issueEndYmd": "{today + 30days}"
  }
  ↓
쿠폰 발행 완료
  ↓
사용자에게 안내 메시지 표시:
"리뷰 감사합니다! 5,000원 할인 쿠폰이 발급되었습니다."
```

### 4.4 예외 처리

#### 4.4.1 리뷰 승인/거절 기준

**승인 기준 (자동 통과)**:
```
✓ 상품명/주문번호 포함
✓ 이미지 포함 (3장 이상 추천)
✓ 텍스트 길이 50자 이상
✓ 욕설/비방 없음
✓ 개인정보 미노출
```

**거절 기준 (삭제 필요)**:
```
✗ 상품과 무관한 내용
✗ 욕설, 비방, 혐오 표현
✗ 개인정보, 전화번호, 주소 포함
✗ 스팸/광고성 내용
✗ 부적절한 이미지 (음란물, 폭력 등)
✗ 경쟁사 상품/서비스 언급
```

#### 4.4.2 리뷰 삭제 및 쿠폰 회수

**삭제 권한**: 관리자만
**삭제 절차**:

```
관리자 > 리뷰 관리 > 삭제 버튼
  ↓
삭제 확인 팝업
  ↓
리뷰 삭제 (DELETE /products/{productNo}/notices/{noticeNo})
  ↓
쿠폰 회수 API 호출
  {
    "couponNo": {발급된 쿠폰},
    "memberNo": {리뷰 작성자},
    "action": "REVOKE"
  }
  ↓
회수 완료 알림 (관리자에게)
```

**쿠폰 회수 규칙**:
```
쿠폰 상태: "REVOKED" (회수됨)
사용자 알림: "리뷰가 관리 정책상 삭제되어 발급된 쿠폰도 회수되었습니다."
사용 전 회수: 즉시 사용 불가
사용 후 회수: 환불 처리 (주문 취소)
```

#### 4.4.3 쿠폰 발행 실패 처리

**규칙**: 리뷰 승인 후 쿠폰 미발행 시

```
리뷰 승인: OK
쿠폰 발행: FAIL
  ↓
관리자 알림 발송 (슬랙/이메일)
  ↓
수동 쿠폰 발행 대시보드
  ↓
누락된 쿠폰 일괄 발행
  ↓
사용자 안내 메일 발송
```

#### 4.4.4 중복 리뷰 방지

**규칙**: 회원당 상품별 1회만 리뷰 가능

```
리뷰 작성 전 조회:
GET /products/{productNo}/notices?memberNo={memberNo}
  ↓
기존 리뷰 확인
  ↓
기존 리뷰 있으면: 버튼 비활성화
"이미 리뷰를 작성했습니다. 수정 또는 삭제할 수 있습니다."
```

### 4.5 관리자 운영 프로세스

#### 4.5.1 리뷰 승인 프로세스

**일일 운영**:
```
1. 관리자 로그인 > 콘텐츠 관리 > 리뷰 관리
2. "대기 중인 리뷰" 탭에서 신규 리뷰 확인
3. 각 리뷰 클릭하여 내용 검토
4. 승인 기준 확인:
   - 상품 관련 내용 O
   - 욕설/비방 X
   - 개인정보 X
   - 이미지 3장 이상 O (권장)
   - 글자수 50자 이상 O
5. "승인" 또는 "거절" 버튼 클릭
6. 거절 사유 입력 (선택)
7. 저장
```

**승인된 리뷰**:
- 상태: "APPROVED"
- 노출: 상품 상세 페이지에 즉시 표시
- 쿠폰: 사용자가 사용 가능

**거절된 리뷰**:
- 상태: "REJECTED"
- 노출: 숨김 (관리자에서만 조회 가능)
- 쿠폰: 회수 처리
- 사용자 알림: 거절 사유 포함 메일 발송

#### 4.5.2 리뷰 삭제 기준 및 운영

**모니터링 주기**: 일 2회 (오전/오후)
**삭제 사유별 처리**:

| 삭제 사유 | 빈도 | 처리 방법 | 사용자 안내 |
|---------|------|---------|----------|
| 욕설/혐오 | 주 1~2회 | 즉시 삭제 + 쿠폰 회수 | 정책 위반 안내 |
| 스팸/광고 | 월 1~2회 | 즉시 삭제 + 쿠폰 회수 | 정책 위반 안내 |
| 개인정보 노출 | 월 2~3회 | 즉시 삭제 + 쿠폰 회수 | 개인정보 보호 안내 |
| 상품 무관 | 월 1회 | 거절 또는 삭제 | 상품 관련 리뷰 요청 |

#### 4.5.3 리뷰 통계 분석

**주기**: 주 1회
**분석 항목**:
- 리뷰 작성 현황 (총 리뷰수, 승인율)
- 평점 분포 (5점, 4점, 3점 등)
- 제품별 리뷰 수 및 평균 평점
- 쿠폰 발행/회수 현황
- 거절/삭제 리뷰 현황 및 사유

#### 4.5.4 리뷰 정책 개선

**분기별 검토**:
- 승인 기준 개선 필요성 검토
- 쿠폰 금액 조정 검토 (리뷰 작성 유도 효율성)
- 삭제 기준 업데이트 (새로운 위반 사항 추가)
- 고객 만족도 피드백 수집

### 4.6 정책 결정 체크리스트

#### 4.6.1 리뷰 노출 방식 (중요)
- [ ] 리뷰 즉시 노출 방식 확정
  - 선택1: 임시 노출 후 관리자 승인 (현재 방안)
  - 선택2: 관리자 승인 후 노출
  - 선택3: 자동 승인 (모든 리뷰 공개)

**선택1 (권장)**: 고객 만족도 높음, 관리자 워크로드 증가
**선택2**: 보수적, 관리자 승인 지연 시 고객 불만
**선택3**: 초기 운영 리소스 절감, 품질 관리 어려움

#### 4.6.2 리뷰 쿠폰 정책
- [ ] 리뷰 작성 시 발급 쿠폰 금액 확정
  - 기본안: 5,000원
  - 최소주문금액: 30,000원
  - 유효기간: 30일

- [ ] 리뷰 삭제 시 쿠폰 회수 방식 확정
  - 쿠폰 미사용 시: 즉시 회수
  - 쿠폰 사용 후 회수: 환불 처리 또는 불가

#### 4.6.3 리뷰 관리 기준
- [ ] 리뷰 승인 기준 최종 확정
  - 최소 글자수: 50자 (또는 변경)
  - 이미지 권장 수: 3장 (또는 필수)
  - 기타 요건 추가

- [ ] 리뷰 거절/삭제 기준 최종 확정
  - 욕설/혐오 표현: 즉시 삭제
  - 개인정보 노출: 즉시 삭제
  - 스팸/광고: 즉시 삭제
  - 상품 무관 내용: 거절 또는 삭제

#### 4.6.4 리뷰 가능 기간

## 부록: 미결정 사항 및 Q&A

### Q1. 구매후기 즉시노출 가능 여부
**질문**: 구매후기 즉시노출이 가능한가? 아니면 관리자 승인이 필요한가?

**답변 (권장 방안)**:
```
즉시 노출 + 사후 승인 (이중 검증)

방식:
1. 사용자가 리뷰 작성 제출
2. 시스템: 임시 상태로 등록, 자동 쿠폰 발행
3. 프론트엔드: "리뷰가 등록되었습니다" 메시지 표시 (쿠폰 발급 안내)
4. 관리자: 신규 리뷰 목록에서 검토
5. 관리자 승인 시: 상태 변경, 공개 노출 (기존 임시 등록 유지)
6. 관리자 거절 시: 상태 변경, 비공개 처리, 쿠폰 회수

장점:
- 고객: 즉시 피드백 받음, 쿠폰 획득
- 비즈니스: 사용자 신뢰도 향상
- 관리: 사후 검증으로 품질 관리 가능

단점:
- 관리자 승인 지연 시 부적절한 리뷰 공개 가능성
- 관리 리소스 필요
```

### Q2. shopby에서 지원하는 방식
**질문**: shopby에서 지원하는 리뷰 정책 방식은?

**답변**:
```
shopby 기본 제공 기능:

1. 리뷰 생성 API: POST /products/{productNo}/notices
   - 리뷰 즉시 등록 가능
   - 승인 상태 관리 필드 제공

2. 리뷰 상태 관리:
   - PENDING_REVIEW: 임시 등록
   - APPROVED: 공개 노출
   - REJECTED: 거절 (비공개)
   - DELETED: 삭제

3. 리뷰 관리 UI (어드민):
   - 신규 리뷰 목록 조회
   - 승인/거절 처리
   - 삭제 기능

4. 쿠폰 연동:
   - POST /coupons API로 쿠폰 발행 가능
   - 리뷰 이벤트 시 쿠폰 자동 발행 가능
   - 쿠폰 회수 API 미지원 (수동 처리 필요)

추천 구현 방식:
- shopby의 리뷰 상태 관리 활용
- 리뷰 승인/거절 기능 활용
- 별도 백엔드에서 쿠폰 발행/회수 로직 구현
```
