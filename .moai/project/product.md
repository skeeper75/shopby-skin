# 프로젝트 개요 - @shopby/react-skin (Aurora Skin)

> 버전: v1.16.5 | 라이선스: MIT

---

## 프로젝트 설명

**Aurora Skin**은 NHN Commerce의 쇼핑몰 솔루션 Shopby를 위한 React 기반 커스텀 스킨(테마)입니다. 쇼핑몰 운영자가 고객에게 제공하는 프론트엔드 UI를 구성하며, 상품 탐색부터 결제 완료까지 전체 구매 여정을 담당합니다.

이 스킨은 SPA(Single Page Application) 구조로 구현되었으며, Shopby API와 연동하여 완전한 e-커머스 경험을 제공합니다. 커스터마이징이 가능한 CSS 변수 기반 테마 시스템을 갖추고 있어 브랜드에 맞는 디자인 적용이 용이합니다.

## 대상 사용자

### 쇼핑몰 이용 고객 (End Users)
- 상품을 탐색하고 구매하는 일반 소비자
- 모바일 및 PC 환경에서 접속하는 사용자
- 회원 및 비회원 구매자

### 쇼핑몰 운영자 (Shop Owners)
- Shopby 플랫폼을 이용하는 온라인 쇼핑몰 사업자
- Aurora Skin을 기반으로 자사몰을 운영하는 담당자
- 어드민에서 디자인 팝업, 배너 등 콘텐츠를 관리하는 운영팀

### 개발자 (Developers)
- 스킨을 커스터마이징하거나 신규 기능을 개발하는 프론트엔드 개발자
- CI/CD 파이프라인을 관리하는 DevOps 엔지니어

## 핵심 기능 목록

### 상품 관련
- **상품 목록**: 카테고리별 상품 탐색, 필터링, 정렬
- **상품 상세**: 옵션 선택, 상세 이미지, 상품 정보 탭
- **상품 검색**: 키워드 검색, 검색 결과 페이지
- **카테고리 네비게이션**: 계층형 카테고리 메뉴 탐색

### 구매 관련
- **장바구니**: 상품 담기, 수량 변경, 삭제
- **주문서 작성**: 배송지 입력, 결제 수단 선택
- **주문 완료**: 주문 확인 및 완료 안내

### 회원 관련
- **회원가입 (SignUp)**: 일반 회원가입
- **로그인 (SignIn)**: 이메일/SNS 로그인
- **마이페이지 (MyPage)**:
  - 주문 내역 조회 및 상세
  - 1:1 문의 내역
  - 상품 리뷰 관리
  - 쿠폰 목록
  - 적립금 내역
  - 찜 목록 (좋아요)
  - 배송지 관리

### 클레임 처리
- **환불/반품 신청**: 취소, 반품, 교환 신청 흐름
- **클레임 내역 확인**: 처리 상태 추적

### 고객 지원
- **고객센터**: 공지사항, FAQ, 문의하기
- **FAQ**: 자주 묻는 질문 목록 및 상세
- **공지사항**: 쇼핑몰 공지 열람

### 콘텐츠/UI 시스템
- **디자인 팝업**: 어드민에서 설정한 팝업 표시
- **배너 시스템**: 어드민 배너 및 커스텀 배너
- **메인 페이지**: 기획전, 추천 상품 등 메인 콘텐츠

## 비즈니스 로직 요약

### 인증 및 토큰 관리
- JWT 기반 인증 사용
- `src/utils/api.js`의 `fetchHttpRequest()`가 모든 API 호출을 담당
- HTTP 401 응답 시 토큰 자동 갱신 후 재요청
- 비로그인 상태에서도 상품 탐색, 검색 가능

### Provider 체인 (의존성 순서)
앱 초기화 시 다음 순서로 컨텍스트를 구성합니다:
1. **ModalProvider**: 모달 UI 상태 관리
2. **MallProvider**: 쇼핑몰 기본 정보 (몰 설정, 메뉴 등)
3. **OrderConfigProvider**: 주문 설정 정보
4. **BoardConfigurationProvider**: 게시판 설정
5. **AuthProvider**: 인증 상태 (로그인 여부, 회원 정보)
6. **TermsProvider**: 약관 정보
7. **CurrencyWrap**: 통화 처리
8. **NaverShoppingConfigurationProvider**: 네이버 쇼핑 연동 설정
9. **DesignPopupProvider**: 디자인 팝업 관리
10. **EventProviderV2**: 이벤트 트래킹

### 환경 설정
- `public/environment.json`을 통해 클라이언트 ID, 프로파일, TC 서버 정보를 관리
- 개발/스테이징/운영 환경별 분리 설정 지원

### 국제화 (i18n)
- i18next 기반, 기본 언어 한국어
- 번역 키는 `src/constants/` 하위에 정의

## 주요 사용 사례

### 사례 1: 상품 구매 여정
1. 고객이 메인 페이지 접속 → 기획전 또는 카테고리 탐색
2. 상품 목록에서 원하는 상품 선택
3. 상품 상세 페이지에서 옵션 선택 후 장바구니 담기
4. 장바구니에서 수량 확인 후 주문서로 이동
5. 배송지 및 결제 수단 입력 후 결제 완료

### 사례 2: 회원 가입 및 로그인
1. 비회원이 구매 시도 시 로그인 유도
2. 신규 사용자는 회원가입 페이지에서 정보 입력
3. 이메일 인증 또는 SNS 계정으로 가입 완료
4. 로그인 후 마이페이지 및 장바구니 동기화

### 사례 3: 클레임(환불/반품) 신청
1. 마이페이지 > 주문 내역에서 대상 주문 선택
2. 취소/반품/교환 유형 선택
3. 사유 입력 및 환불 계좌 정보 입력
4. 클레임 처리 상태 추적 (접수 → 확인 → 완료)

### 사례 4: 운영자의 디자인 팝업 관리
1. 어드민에서 팝업 콘텐츠 및 노출 조건 설정
2. DesignPopupProvider가 앱 초기화 시 팝업 정보 로드
3. 설정된 조건(페이지, 시간)에 맞춰 고객에게 팝업 노출

### 사례 5: 모바일/PC 반응형 대응
- CSS 변수 및 미디어 쿼리를 통해 모바일/PC 레이아웃 자동 전환
- 터치 인터랙션 최적화된 모바일 UI 제공
