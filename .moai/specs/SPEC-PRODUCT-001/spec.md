## 1. 개요

### 1.1 목적

후니프린팅 shopby Enterprise 마이그레이션에서 상품관리(A10B4-PRODUCT) 도메인의 전체 기능을 정의한다. 관리자 상품등록/마스터 관리(모듈 1~5)와 쇼핑몰 프론트 주문페이지 옵션 엔진(모듈 6)을 포괄한다. 인쇄 도메인 특화 CUSTOM 모듈 중심의 Tier 2 구현 전략을 따르며, Figma option_NEW 분석 기반 11개 상품 카테고리별 옵션 구성을 정의한다.

### 1.2 범위

- **포함**: 관리자 상품등록/마스터 관리, 종속옵션 엔진, 가격 매트릭스 엔진, **쇼핑몰 주문페이지 옵션 엔진 (11개 상품 카테고리)**, 가격 계산 4유형, 파일 업로드 5패턴 정의
- **제외**: 파일 업로드/검수 구현(SPEC-ORDER-001), 옵션보관함(SPEC-ORDER-001), 주문/결제 프로세스(SPEC-ORDER-001), 에디터 구현(별도 SPEC)

### 1.3 SPEC-PLAN-001과의 관계

본 SPEC은 SPEC-PLAN-001 v1.1.0 마스터 기획서의 A10B4-PRODUCT 도메인 정책을 구현 수준으로 구체화한 문서이다. features-data.json의 B-4-1 ~ B-4-10 항목과 custom-dev-data.json의 #2(종속옵션+가격엔진 XL) 모듈을 포괄한다.

### 1.4 도메인 특성

- **가장 CUSTOM-heavy 도메인**: 14개 기능 중 9개가 CUSTOM (64.3%)
- **관리자 + 쇼핑몰 이중 시스템**: 관리자(admin) 화면 + 쇼핑몰 프론트 주문페이지
- **PC 우선 설계**: 인쇄 업종 관리자 화면은 PC에서만 운영, 쇼핑몰은 PC 우선 반응형
- **핵심 엔진**: 종속옵션 체인(33개 규칙) + 가격 계산 4유형이 전체 상품 시스템의 근간
- **11개 상품 카테고리**: 복잡도 XL(2) / L(2) / M(4) / S(3)으로 분류

## 3. EARS 요구사항

### 3.1 모듈 1: 옵션 마스터 관리 (Option Master) - 4개 기능

#### REQ-PRD-001 [Ubiquitous] 사이즈 마스터 데이터 관리

시스템은 항상 상품유형별 사이즈 옵션을 독립 마스터 테이블에서 CRUD 관리해야 한다.

#### REQ-PRD-002 [Ubiquitous] 소재 마스터 데이터 관리

시스템은 항상 상품유형별 소재 옵션(실사/굿즈용)을 독립 마스터 테이블에서 CRUD 관리해야 한다.

#### REQ-PRD-003 [Ubiquitous] 용지 마스터 데이터 관리

시스템은 항상 상품유형별 용지 옵션(디지털/제본/패키지용)을 독립 마스터 테이블에서 CRUD 관리해야 한다.

#### REQ-PRD-004 [Ubiquitous] 마스터 데이터 일관성 보장

시스템은 항상 마스터 데이터 변경 시 해당 옵션을 참조하는 상품의 일관성을 유지해야 한다.

#### REQ-PRD-005 [Event-Driven] 사이즈 마스터 등록

WHEN 관리자가 새 사이즈를 등록하면 THEN 시스템은 상품유형, 사이즈명, 가로/세로 치수(mm), 표시순서, 활성/비활성 상태를 저장해야 한다.

#### REQ-PRD-006 [Event-Driven] 소재 마스터 등록

WHEN 관리자가 새 소재를 등록하면 THEN 시스템은 상품유형, 소재명, 소재 설명, 적용 가능 사이즈 목록, 활성/비활성 상태를 저장해야 한다.

#### REQ-PRD-007 [Event-Driven] 용지 마스터 등록

WHEN 관리자가 새 용지를 등록하면 THEN 시스템은 상품유형, 용지명, 평량(g), 코팅 종류, 적용 가능 사이즈 목록, 활성/비활성 상태를 저장해야 한다.

#### REQ-PRD-008 [Event-Driven] 마스터 데이터 비활성화

WHEN 관리자가 마스터 데이터를 비활성화하면 THEN 시스템은 해당 옵션을 신규 상품등록에서 숨기되 기존 상품의 참조는 유지해야 한다.

#### REQ-PRD-009 [Unwanted] 참조 중인 마스터 데이터 삭제 금지

시스템은 활성 상품에서 참조 중인 마스터 데이터를 물리적으로 삭제하지 않아야 한다.

#### REQ-PRD-010 [Event-Driven] 마스터 데이터 검색

WHEN 관리자가 마스터 목록에서 검색어를 입력하면 THEN 시스템은 이름/코드 기준으로 필터링된 결과를 표시해야 한다.

### 3.2 모듈 2: 종속옵션 엔진 (Cascading Option Engine) - 3개 기능

#### REQ-PRD-011 [Ubiquitous] 종속옵션 체인 유지

시스템은 항상 상품유형->사이즈->소재/용지->후가공->수량 순서의 종속옵션 체인을 유지해야 한다.

#### REQ-PRD-012 [Event-Driven] 사이즈 선택 팝업

WHEN 관리자가 인쇄/제본 상품등록에서 "사이즈 선택" 버튼을 클릭하면 THEN 시스템은 해당 상품유형에 맞는 사이즈 마스터 목록을 팝업으로 표시하고 다중선택을 허용해야 한다.

#### REQ-PRD-013 [Event-Driven] 소재 선택 팝업

WHEN 관리자가 "소재 선택" 버튼을 클릭하면 THEN 시스템은 선택된 사이즈에 적용 가능한 소재 마스터 목록만 필터링하여 팝업으로 표시해야 한다.

#### REQ-PRD-014 [Event-Driven] 용지 선택 팝업

WHEN 관리자가 "용지 선택" 버튼을 클릭하면 THEN 시스템은 선택된 사이즈에 적용 가능한 용지 마스터 목록만 필터링하여 팝업으로 표시해야 한다.

#### REQ-PRD-015 [Event-Driven] 상위 옵션 변경 시 하위 리셋

WHEN 관리자가 상위 옵션(상품유형 또는 사이즈)을 변경하면 THEN 시스템은 하위 옵션 선택을 모두 초기화하고 "상위 옵션 변경으로 하위 옵션이 초기화됩니다" 확인 다이얼로그를 표시해야 한다.

#### REQ-PRD-016 [State-Driven] 하위 옵션 비활성화

IF 상위 옵션이 선택되지 않은 상태 THEN 시스템은 하위 옵션 선택 버튼/영역을 비활성화(disabled) 상태로 표시해야 한다.

#### REQ-PRD-017 [Ubiquitous] 옵션 조합 유효성 검증

시스템은 항상 선택된 옵션 조합의 유효성(사이즈-소재/용지 호환성)을 검증해야 한다.

#### REQ-PRD-018 [Event-Driven] 옵션 조합 충돌 알림

WHEN 선택된 옵션 조합이 유효하지 않으면 THEN 시스템은 충돌 원인과 변경 제안을 인라인 경고로 표시해야 한다.

### 3.3 모듈 3: 가격 매트릭스 엔진 (Price Matrix Engine) - 2개 기능

#### REQ-PRD-019 [Ubiquitous] 8종 가격 코드 체계

시스템은 항상 8종 가격 코드(DP02, DP04, DP06, GD01, GD02, PK01, PR01, PR02)별로 독립된 가격 매트릭스를 관리해야 한다.

#### REQ-PRD-020 [Ubiquitous] 가격 매트릭스 구성 요소

시스템은 항상 가격 매트릭스를 기본 출력비 + 용지비 + 코팅비 + 후가공비 + 제본비 요소의 조합으로 구성해야 한다.

#### REQ-PRD-021 [Event-Driven] 가격관리 팝업 호출

WHEN 관리자가 상품등록에서 "가격관리" 버튼을 클릭하면 THEN 시스템은 해당 상품의 가격 코드에 맞는 가격관리 팝업을 표시해야 한다.

#### REQ-PRD-022 [Event-Driven] 가격 매트릭스 편집

WHEN 관리자가 가격관리 팝업에서 수량구간별 가격을 입력/수정하면 THEN 시스템은 수량(100/500/1000/5000/10000매), 인쇄면(편면/양면), 코팅 종류별 가격 조합을 저장해야 한다.

#### REQ-PRD-023 [Event-Driven] 후가공 가격 설정

WHEN 관리자가 후가공 가격을 설정하면 THEN 시스템은 후가공 종류(박/형압/도무송/UV/에폭시/오시/미싱/타공/라운딩/넘버링)별 단가와 수량 할인율을 저장해야 한다.

#### REQ-PRD-024 [Event-Driven] 제본 가격 설정

WHEN 관리자가 제본 가격을 설정하면 THEN 시스템은 제본 방식(무선/중철/스프링/양장), 페이지 수 구간별 가격을 저장해야 한다.

#### REQ-PRD-025 [Event-Driven] 가격 시뮬레이터

WHEN 관리자가 가격 시뮬레이터에서 옵션 조합을 선택하면 THEN 시스템은 최종 가격을 실시간으로 계산하여 가격 구성 내역(출력비/용지비/코팅비/후가공비/제본비)과 함께 표시해야 한다.

#### REQ-PRD-026 [Ubiquitous] 가격 계산 서버사이드

시스템은 항상 최종 가격 계산을 서버사이드에서 수행하여 클라이언트 위변조를 방지해야 한다.

#### REQ-PRD-027 [Event-Driven] 수량별 단가 체감 적용

WHEN 주문 수량이 증가하면 THEN 시스템은 수량 구간별 단가 체감표에 따라 단가를 자동 적용해야 한다.

#### REQ-PRD-028 [Event-Driven] 당일출고 할증 적용

WHEN 관리자가 상품에 당일출고 옵션을 활성화하면 THEN 시스템은 기본 가격 대비 30~50% 할증 가격을 별도 저장해야 한다.

#### REQ-PRD-029 [Unwanted] 가격 0원 등록 금지

시스템은 어떤 수량 구간에서도 가격이 0원인 가격 매트릭스를 등록하지 않아야 한다.

### 3.4 모듈 4: 인쇄/제본 상품등록 (Print Product Registration) - 1개 기능

#### REQ-PRD-030 [Ubiquitous] 인쇄/제본 상품등록 필수 정보

시스템은 항상 인쇄/제본 상품등록 시 다음 필수 정보를 요구해야 한다: 상품명, 상품유형, 사이즈(종속), 소재/용지(종속), 가격 코드, 기본 가격 매트릭스, 인쇄 가이드라인.

#### REQ-PRD-031 [Event-Driven] 인쇄/제본 상품등록 저장

WHEN 관리자가 인쇄/제본 상품등록 폼을 완성하고 저장 버튼을 클릭하면 THEN 시스템은 shopby 상품 API에 기본 상품 정보를 등록하고, CUSTOM DB에 종속옵션과 가격 매트릭스를 연결 저장해야 한다.

#### REQ-PRD-032 [Event-Driven] 기본값 설정

WHEN 관리자가 인쇄/제본 상품의 기본값을 설정하면 THEN 시스템은 각 옵션 단계별 기본 선택값(기본 사이즈, 기본 소재/용지, 기본 수량)을 저장하여 쇼핑몰 상품상세에서 초기값으로 표시해야 한다.

#### REQ-PRD-033 [Event-Driven] 상품 미리보기

WHEN 관리자가 "미리보기" 버튼을 클릭하면 THEN 시스템은 쇼핑몰 상품상세 페이지 형태로 옵션 캐스케이딩과 가격 계산을 포함한 미리보기를 표시해야 한다.

#### REQ-PRD-034 [State-Driven] 상품 상태 관리

IF 상품이 "판매중지" 상태 THEN 시스템은 쇼핑몰 프론트에서 해당 상품을 노출하지 않아야 한다.

#### REQ-PRD-035 [Event-Driven] 상품 복제

WHEN 관리자가 기존 상품의 "복제" 버튼을 클릭하면 THEN 시스템은 종속옵션, 가격 매트릭스, 기본값 설정을 포함한 상품 전체를 복제하여 새 상품으로 생성해야 한다.

#### REQ-PRD-036 [Ubiquitous] 상품등록 폼 자동저장

시스템은 항상 상품등록 폼 작성 중 30초마다 임시저장하여 데이터 유실을 방지해야 한다.

#### REQ-PRD-037 [Event-Driven] PDF 가이드라인 템플릿 관리

WHEN 관리자가 인쇄 상품에 PDF 가이드라인 템플릿을 등록하면 THEN 시스템은 해당 상품의 상세 페이지에서 고객이 다운로드할 수 있도록 연결해야 한다.

### 3.5 모듈 5: 일반 상품등록 (General Product Registration) - 4개 기능

#### REQ-PRD-038 [Ubiquitous] 굿즈 카테고리 관리

시스템은 항상 shopby 카테고리 API를 활용하여 굿즈 카테고리를 관리해야 한다.

#### REQ-PRD-039 [Event-Driven] 굿즈 상품등록

WHEN 관리자가 굿즈 상품을 등록하면 THEN 시스템은 shopby 기본 상품등록 + 옵션 커스텀(색상/사이즈 등)을 통해 저장해야 한다.

#### REQ-PRD-040 [Event-Driven] 수작 상품등록

WHEN 관리자가 수작(수작업 인쇄) 상품을 등록하면 THEN 시스템은 shopby 기본 상품등록 + 수작업 옵션(소재, 크기, 작업 방식)을 통해 저장해야 한다.

#### REQ-PRD-041 [Event-Driven] 포장재 상품등록

WHEN 관리자가 포장재 상품을 등록하면 THEN 시스템은 shopby 기본 상품등록 + 포장재 옵션(재질, 크기, 수량 단위)을 통해 저장해야 한다.

#### REQ-PRD-042 [Event-Driven] 디자인 상품등록

WHEN 관리자가 디자인 서비스 상품을 등록하면 THEN 시스템은 shopby 기본 상품등록 + 디자인 옵션(작업 범위, 수정 횟수, 납기일)을 통해 저장해야 한다.

#### REQ-PRD-043 [Ubiquitous] shopby 상품 API 연동

시스템은 항상 굿즈/수작/포장재/디자인 상품의 기본 정보(상품명, 가격, 이미지, 카테고리)를 shopby 상품 API를 통해 관리해야 한다.

### 3.6 모듈 6: 쇼핑몰 주문페이지 옵션 엔진 (Storefront Option Engine) - 11개 상품 카테고리

> Figma option_NEW 분석 + 인쇄 도메인 심층 분석 결과 반영
> 참조: `product-order-pages.md`, `option-dependency-map.md`, `figma-screen-design-reference.md`

#### 3.6.1 상품 카테고리 정의

#### REQ-PRD-051 [Ubiquitous] 11개 상품 카테고리 구분

시스템은 항상 11개 상품 카테고리(디지털인쇄, 책자/제본, 문구, 포토북, 캘린더, 디자인캘린더, 액세서리, 아크릴, 실사/사인, 스티커, 굿즈/파우치)를 구분하고 각 카테고리별 독립된 옵션 세트를 제공해야 한다.

#### REQ-PRD-052 [Ubiquitous] 복잡도 등급별 옵션 구성

시스템은 항상 상품 복잡도 등급(XL: 12+그룹, L: 8-11그룹, M: 5-7그룹, S: 2-4그룹)에 따라 옵션 그룹 수와 종속 깊이를 관리해야 한다.

#### REQ-PRD-053 [Event-Driven] 상품 카테고리 전환 시 옵션 세트 교체

WHEN 사용자가 상품 카테고리를 전환하면 THEN 시스템은 해당 카테고리의 옵션 설정 JSON을 로드하고 전체 옵션 UI를 동적으로 재구성해야 한다.

#### 3.6.2 종속옵션 체인 (33개 규칙)

#### REQ-PRD-054 [Ubiquitous] FILTER 종속 규칙 (7개)

시스템은 항상 다음 7개 FILTER 종속 규칙을 유지해야 한다:
- 상품유형 -> 전체 옵션 세트 (옵션 그룹 설정 JSON 로드)
- 인쇄: 사이즈 -> 종이 (사이즈에 적합한 용지 목록만 표시)
- 인쇄: 인쇄방식(단면/양면) -> 코팅 (단면: 단면 코팅만, 양면: 양면 코팅 포함)
- 제본: 사이즈 -> 내지종이/표지종이 (판형에 적합한 용지)
- 제본: 책갈 -> 면수 (제본방식에 따른 면수 제한)
- 캘린더: 사이즈 -> 용지
- 스티커: 사이즈 -> 종이(유포)

#### REQ-PRD-055 [Ubiquitous] SHOW_HIDE 종속 규칙 (21개)

시스템은 항상 다음 21개 SHOW_HIDE 종속 규칙을 유지해야 한다:
- 인쇄: 인쇄방식 -> 별색인쇄 5종 활성화 (2개)
- 인쇄: 박(앞면) 있음/없음 -> 박(앞면) 칼라/크기입력 표시/숨김 (4개)
- 인쇄: 박(뒷면) 있음/없음 -> 박(뒷면) 칼라/크기입력 표시/숨김 (4개)
- 인쇄: 형압 양각/음각/없음 -> 형압 크기입력 표시/숨김 (2개)
- 제본: 박(앞면) -> 칼라/크기 (1개)
- 제본: 책갈 무선/중철 -> 띠걸이/띠사별 표시/숨김 (2개)
- 캘린더: 캘린더가공 고리형 -> 고리 색상 표시/숨김 (2개)
- 스티커: 인쇄 단면 -> 별색-화이트 표시 (1개)
- 실사/사인: 사이즈 직접입력/기본 -> 크기 직접입력 필드 표시/숨김 (2개)
- 아크릴: 크기 직접입력 항상 표시 (1개)

#### REQ-PRD-056 [Ubiquitous] RESET 종속 규칙 (5개)

시스템은 항상 다음 5개 RESET 종속 규칙을 유지해야 한다:
- 전체: 상품유형 변경 -> 전체 하위 옵션 리셋 + 확인 다이얼로그
- 인쇄: 사이즈 변경 -> 종이, 코팅, 후가공, 박/형압 리셋 + 확인 다이얼로그
- 제본: 사이즈 변경 -> 내지종이, 표지종이 리셋 + 확인 다이얼로그
- 제본: 책갈 변경 -> 띠걸이, 띠사별, 면수 리셋 + 확인 다이얼로그
- 캘린더: 사이즈 변경 -> 용지 리셋 + 확인 다이얼로그

#### 3.6.3 가격 계산 4유형

#### REQ-PRD-057 [Ubiquitous] 매트릭스형 가격 계산

시스템은 항상 매트릭스형 상품(디지털인쇄, 책자/제본, 스티커, 캘린더)에 대해 사이즈 x 용지 x 인쇄방식 x 수량 조합 테이블 기반 가격을 계산해야 한다.

#### REQ-PRD-058 [Ubiquitous] 구간할인형 가격 계산

시스템은 항상 구간할인형 상품(문구, 아크릴, 굿즈)에 대해 수량 구간별 단가 체감 테이블을 적용하고, 슬라이더 UI로 구간별 단가를 시각적으로 표시해야 한다.

#### REQ-PRD-059 [Ubiquitous] 면적형 가격 계산

시스템은 항상 면적형 상품(실사/사인, 아크릴 커스텀)에 대해 출력 면적(m2 또는 mm2) 기반으로 가격을 계산해야 한다.

#### REQ-PRD-060 [Ubiquitous] 고정가형 가격 계산

시스템은 항상 고정가형 상품(포토북, 디자인캘린더, 액세서리)에 대해 사이즈+타입별 고정 단가를 적용해야 한다.

#### REQ-PRD-061 [Event-Driven] 가격 실시간 갱신

WHEN 사용자가 주문페이지에서 가격에 영향을 미치는 옵션(사이즈, 종이/소재, 인쇄방식, 별색인쇄, 코팅, 커팅, 수량 등)을 변경하면 THEN 시스템은 300ms debounce 후 BFF API를 호출하여 가격을 재계산하고 가격 Summary를 실시간 갱신해야 한다.

#### REQ-PRD-062 [Ubiquitous] 부가세 별도 표시

시스템은 항상 모든 상품의 가격 Summary에 상품가와 부가세(10%)를 별도 표시하고, 합계금액 = 상품가 + 부가세 형식으로 표시해야 한다.

#### REQ-PRD-063 [Event-Driven] 구간할인 슬라이더 UI

WHEN 구간할인형 상품(문구, 아크릴, 굿즈)의 주문페이지에서 사용자가 수량을 변경하면 THEN 시스템은 슬라이더와 단가 테이블을 통해 현재 수량의 단가와 대량 주문 시 단가 혜택을 시각적으로 표시해야 한다.

#### REQ-PRD-064 [Event-Driven] 할인금액 마이너스 표시 (굿즈)

WHEN 굿즈/파우치 상품의 구간할인이 적용되면 THEN 시스템은 가격 Summary에 "할인금액"을 마이너스(-) 표시로 보여주어야 한다.

#### 3.6.4 별색인쇄 및 특수 가공 UI

#### REQ-PRD-065 [Event-Driven] 별색인쇄 5종 선택 (디지털인쇄)

WHEN 디지털인쇄 주문페이지에서 사용자가 인쇄방식을 선택하면 THEN 시스템은 5종 별색인쇄(화이트, 클리어, 핑크, 금색, 은색) 옵션을 각각 없음/단면/양면 Button으로 표시해야 한다.

#### REQ-PRD-066 [Event-Driven] 박/형압 Collapsible UI

WHEN 디지털인쇄 또는 제본 상품에서 사용자가 "박/형압 가공" 섹션을 펼치면 THEN 시스템은 다음 복합 UI를 표시해야 한다:
- 박(앞면): 있음/없음 Button -> 칼라 8종 ColorChip + 크기 Input(mm)
- 박(뒷면): 있음/없음 Button -> 칼라 8종 ColorChip + 크기 Input(mm)
- 형압: 없음/양각/음각 Button -> 크기 Input(mm)

#### REQ-PRD-067 [Event-Driven] 후가공 Collapsible UI (디지털인쇄/스티커)

WHEN 사용자가 "후가공" 섹션을 펼치면 THEN 시스템은 해당 상품에 적용 가능한 후가공 옵션(귀돌이, 오시, 미싱, 가변인쇄 등)을 개별 선택 가능한 체크박스/버튼으로 표시해야 한다.

#### 3.6.5 크기 직접입력

#### REQ-PRD-068 [Event-Driven] 아크릴 크기 직접입력

WHEN 아크릴 주문페이지에서 사용자가 크기 직접입력 필드에 가로/세로 치수를 입력하면 THEN 시스템은 입력 범위를 검증하고 면적 기반 가격을 재계산해야 한다.

#### REQ-PRD-069 [Event-Driven] 실사/사인 크기 직접입력

WHEN 실사/사인 주문페이지에서 사용자가 "직접입력" 사이즈를 선택하면 THEN 시스템은 가로(200~1300mm) x 세로(200~8000mm) 입력 필드를 표시하고 면적 기반 가격(m2당 단가)을 계산해야 한다.

#### REQ-PRD-070 [State-Driven] 크기 직접입력 범위 제한

IF 사용자가 허용 범위 밖의 치수를 입력한 상태 THEN 시스템은 인라인 경고("가로는 200~1300mm, 세로는 200~8000mm 범위 내에서 입력해주세요")를 표시하고 가격 계산을 차단해야 한다.

#### 3.6.6 파일 업로드 패턴

#### REQ-PRD-071 [Ubiquitous] 파일 업로드 5패턴 분류

시스템은 항상 상품 카테고리에 따라 5가지 파일 업로드 패턴을 적용해야 한다:
- 패턴 A (PDF+에디터+장바구니): 인쇄, 제본, 스티커, 문구
- 패턴 B (에디터만): 포토북, 디자인캘린더, 아크릴
- 패턴 C (PDF만+장바구니): 캘린더, 실사/사인
- 패턴 D (장바구니만, 파일 불필요): 액세서리
- 패턴 E (PDF+에디터+구간할인 슬라이더): 굿즈

#### REQ-PRD-072 [State-Driven] 파일 업로드 패턴별 UI 분기

IF 패턴 D(액세서리) 상품인 상태 THEN 시스템은 파일 업로드 영역을 표시하지 않고 "장바구니 담기" 버튼만 표시해야 한다.

#### 3.6.7 상품별 특화 옵션

#### REQ-PRD-073 [Event-Driven] 제본 상품 내지/표지 분리 옵션

WHEN 책자/제본 주문페이지에서 사용자가 옵션을 선택하면 THEN 시스템은 내지(종이, 인쇄방식, 페이지수)와 표지(종이, 인쇄방식, 코팅, 투명커버)를 명확히 분리하여 2트랙으로 표시해야 한다.

#### REQ-PRD-074 [Event-Driven] 제본 부속품 선택 (띠걸이/띠사별)

WHEN 책자/제본에서 무선제본이 선택되면 THEN 시스템은 띠걸이(6종 Image Button)와 띠사별(5종 Image Button) 옵션을 표시해야 한다.

#### REQ-PRD-075 [Event-Driven] 캘린더 부속품 선택 (집게/고리)

WHEN 캘린더 주문페이지에서 캘린더가공 방식을 선택하면 THEN 시스템은 가공방식에 따라 집게 색상(ColorChip) 또는 고리 색상(ColorChip 3종)을 표시해야 한다.

#### REQ-PRD-076 [Event-Driven] 굿즈 소재 컬러 선택

WHEN 굿즈/파우치 주문페이지에서 사용자가 컬러를 선택하면 THEN 시스템은 대형 ColorChip UI(10종+)를 통해 소재 색상을 선택할 수 있도록 하고, 선택된 색상에 따라 가격을 재계산해야 한다.

#### REQ-PRD-077 [Event-Driven] 디자인캘린더 레디 디자인 선택

WHEN 디자인캘린더 주문페이지에서 사용자가 "레디자" 옵션을 선택하면 THEN 시스템은 기성 디자인 템플릿 목록을 Select 드롭다운으로 표시해야 한다.

#### 3.6.8 가격 Summary UI

#### REQ-PRD-078 [Ubiquitous] 가격 구성 내역 항목별 표시

시스템은 항상 매트릭스형 상품의 가격 Summary에 구성 내역(기본 출력비, 용지비, 코팅비, 후가공비, 제본비, 할증 등)을 항목별로 분해하여 표시해야 한다.

#### REQ-PRD-079 [State-Driven] 간소화 가격 Summary

IF 고정가형 상품(포토북, 디자인캘린더, 액세서리)인 상태 THEN 시스템은 가격 Summary를 합계금액만 표시하는 간소화 형태로 렌더링해야 한다.

#### REQ-PRD-080 [Event-Driven] 단가(매당) 표시

WHEN 가격 계산이 완료되면 THEN 시스템은 최종 가격과 함께 단가(매당 가격 = 총액 / 수량)를 계산하여 표시해야 한다.

## 4. 비기능 요구사항

### 4.1 성능

#### REQ-PRD-044 [Ubiquitous] 가격 계산 응답 시간

시스템은 항상 가격 계산 API 응답을 500ms 이내에 반환해야 한다.

#### REQ-PRD-045 [Ubiquitous] 옵션 팝업 로딩 시간

시스템은 항상 옵션 선택 팝업을 300ms 이내에 렌더링해야 한다.

#### REQ-PRD-046 [Ubiquitous] 마스터 데이터 캐싱

시스템은 항상 사이즈/소재/용지 마스터 데이터를 캐싱하여 반복 조회 시 DB 부하를 최소화해야 한다.

#### REQ-PRD-081 [Ubiquitous] 쇼핑몰 주문페이지 초기 로딩

시스템은 항상 쇼핑몰 주문페이지의 옵션 설정 JSON을 1초 이내에 로드하여 옵션 UI를 렌더링해야 한다.

#### REQ-PRD-082 [Ubiquitous] 종속옵션 응답 시간

시스템은 항상 종속옵션 변경 시 하위 옵션 목록을 200ms 이내에 반환해야 한다 (Redis 캐시 활용).

### 4.2 보안

#### REQ-PRD-047 [Ubiquitous] 관리자 권한 검증

시스템은 항상 상품관리 API 호출 시 관리자 권한(ADMIN role)을 검증해야 한다.

#### REQ-PRD-048 [Unwanted] 가격 데이터 클라이언트 노출 금지

시스템은 가격 매트릭스 원본 데이터를 클라이언트에 직접 노출하지 않아야 한다.

### 4.3 데이터 무결성

#### REQ-PRD-049 [Ubiquitous] 가격 변경 이력 관리

시스템은 항상 가격 매트릭스 변경 시 변경 전/후 값, 변경일시, 변경자를 이력으로 보관해야 한다.

#### REQ-PRD-050 [Ubiquitous] 옵션-상품 참조 무결성

시스템은 항상 옵션 마스터와 상품 간의 참조 무결성을 외래키 또는 소프트 참조로 보장해야 한다.
