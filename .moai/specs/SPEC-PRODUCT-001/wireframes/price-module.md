## SCR-PRD-014: 가격관리 팝업 (매트릭스형)

> 대상: 디지털출력비, 후가공(미싱/오시/접지/가변), 제본비, 스티커 가공비
> 수량 x 옵션 그리드, 최대 23단계 수량구간, 기준판형 토글(국4절/3절)

### 1. 화면 개요

14종 가격 유형 중 "수량 x 옵션" 형태의 매트릭스를 편집하는 범용 팝업이다.
디지털출력비의 경우 행이 23단계 수량구간(1~1,000,000), 열이 11종 인쇄방식(없음~양면핑크)이며, 기준판형(국4절/3절) 토글로 완전히 다른 가격 세트를 전환한다.
후가공(미싱/오시)은 수량 x 줄수(1~5줄), 접지는 수량 x 접지방식(7종), 가변은 수량 x 개수(1~3개), 제본비는 수량 x 제본방식, 스티커 가공비는 수량 x 모양코드 매트릭스이다.

## SCR-PRD-015: 가격관리 팝업 (고정단가형)

> 대상: 명함 단가, 굿즈 단가, 악세사리 단가
> 상품 x 옵션 x 단가 테이블, 행 단위 CRUD

### 1. 화면 개요

고정 단가 방식의 가격을 관리하는 팝업이다.
명함 단가는 상품명(스탠다드명함, 코팅명함, 펄명함 등 11종) x 용지 x 인쇄면(단면/양면) -> 100장 기준 가격으로, 가격 범위는 3,500~19,000원이다.
굿즈 단가는 상품명 x 옵션(컬러/사이즈) -> 기본가(100~60,000원)이며, 악세사리 단가도 유사하게 상품 x 사이즈 -> 단가 구조이다.

## SCR-PRD-016: 가격관리 팝업 (면적형)

> 대상: 아크릴 단가, 실사/포스터 단가, 박-동판비
> 가로 x 세로 2D 매트릭스 그리드 + 면적 자동 계산

### 1. 화면 개요

면적 기반 가격을 2D 매트릭스(가로 x 세로)로 관리하는 팝업이다.
아크릴 단가는 가로(20~100mm) x 세로(20~100mm) -> 개당 단가(2,500~12,600원, 3T 기준)이며, 두께(1.5mm/3mm/8T/10T) 선택에 따라 다른 가격 세트를 표시한다.

박-동판비는 가로(30~170mm) x 세로(30~170mm) -> 동판 1회 비용(11,000~64,000원)이다.

## SCR-PRD-019: 가격 시뮬레이터

> 상품등록 폼 내 삽입형 Section
> 상품유형 -> 옵션 조합 -> 실시간 가격 계산
> 견적방식 3종 시뮬레이션 (자동견적/고정단가/수량할인)

### 1. 화면 개요

관리자가 옵션 조합을 선택하면 가격을 실시간으로 계산하고, 가격 구성 내역을 분해하여 표시하는 시뮬레이터이다.

가격 구성 내역은 출력비 + 지대(용지비) + 코팅비 + 후가공비 + 제본비 + 기타로 분해되며, 쇼핑몰 API 가격과의 일치 여부를 검증한다.

### 4. API 바인딩

| UI 요소 | API Endpoint | Method | 필드 매핑 |
|---------|-------------|--------|----------|
| 상품유형 셀렉트 | `/api/admin/price/types` | GET | `types[].code`, `.name`, `.pricingMethod` |
| 사이즈/용지 셀렉트 | `/api/admin/product/options/{productType}` | GET | `sizes[]`, `papers[]` (캐스케이딩) |
| 후가공 목록 | `/api/admin/price/finishings` | GET | `finishings[].id`, `.name`, `.unitPrice` |
| 시뮬레이션 실행 | `/api/admin/price/simulate` | POST | Request: `{ productType, pricingMethod, sizeId, paperId, printSide, coatingType, quantity, finishingIds[], foilType, foilWidth, foilHeight, isExpress }` |
| 가격 분해 결과 | (simulate Response) | - | `result.breakdown[].item`, `.unitPrice`, `.subtotal` |
| 견적방식별 비교 | (simulate Response) | - | `result.comparison[].method`, `.unitPrice`, `.total`, `.diff` |
| 쇼핑몰 가격 검증 | (simulate Response) | - | `result.shopbyPrice`, `result.isMatched` |
| 수량별 단가 차트 | (simulate Response) | - | `result.quantityChart[].quantity`, `.priceByMethod{}` |
| 시뮬레이션 이력 | `/api/admin/price/simulate/history` | GET | `history[].id`, `.summary`, `.totalPrice`, `.createdAt` |
| 이력 복원 | `/api/admin/price/simulate/history/{id}` | GET | `options{}` (옵션 자동 채움 데이터) |

### 5. 인터랙션

| 사용자 동작 | 시스템 반응 |
|------------|-----------|
| 견적방식 변경 | 옵션 선택 영역의 필드 구성은 유지, 결과 영역에 해당 방식 기반 계산 결과 표시 |

| 사이즈 변경 | 용지/소재 셀렉트 필터링 (캐스케이딩) |
| 후가공 체크 | 해당 후가공 단가를 분해 테이블에 실시간 반영 |

| 가격 불일치 | 검증 카드 보더 #E53E3E, "불일치" 뱃지 빨간색, 차이 금액 + "가격 수정" 링크 |

| 이력 항목 클릭 | 해당 옵션 자동 채움 + 자동 시뮬레이션 실행 |
| 쇼핑몰 미리보기 | 새 탭에서 해당 상품 주문페이지 열기 |

## 부록: 공통 패턴 정리

### A. 가격 셀 인라인 편집 패턴

| 단계 | 동작 | 시각적 피드백 |
|------|------|-------------|
| 1. 읽기 모드 | 천단위 콤마 포맷 숫자 표시 | Font: JetBrains Mono 14px, 우측 정렬 |
| 2. 클릭 | 셀 편집 모드 진입 | 보더 #5538B6 2px, 배경 흰색, 텍스트 전체 선택 |
| 3. 입력 | 숫자만 허용, 콤마 자동 제거 | 커서 활성 |
| 4. Tab/Enter | 다음 셀로 이동 + 현재 셀 blur | 다음 셀 편집 모드 |
| 5. Blur | 값 저장 (로컬), 포맷 적용 | 변경 셀: 배경 #FFF9E6, 좌측 바 #DD6B20 |
| 6. ESC | 편집 취소, 원래 값 복원 | 읽기 모드 복귀 |

### B. 벌크 편집 워크플로우

| 단계 | 동작 | 설명 |
|------|------|------|
| 1. 셀 선택 | 클릭/Shift+클릭/Ctrl+클릭/드래그 | 복수 셀 선택 |
| 2. 비율 입력 | 일괄 조정 필드에 % 입력 | +10 = 10% 인상, -5 = 5% 인하 |
| 3. 적용 | "적용" 버튼 클릭 | 선택 셀에 비율 적용, 소수점 반올림 |
| 4. 확인 | 변경 요약 패널에서 확인 | 원래값 -> 새값 (증감률) 표시 |
| 5. 되돌리기 | "되돌리기" 클릭 | 마지막 벌크 편집 전 상태 복원 |

### C. 버전 충돌 처리 플로우

| 단계 | 동작 | UI |
|------|------|-----|
| 1. 저장 시도 | PUT 요청에 현재 version 포함 | 저장 버튼 스피너 |
| 2. 충돌 감지 | API 409 Conflict 응답 | 경고 토스트 (#DD6B20) |
| 3. 머지 UI | 충돌 셀 비교 다이얼로그 표시 | 서버값 vs 내값 라디오 선택 |
| 4. 해결 | "전체 서버값" / "전체 내값" / "선택 병합" | 버튼 3개 |
| 5. 재저장 | 병합된 데이터 + 최신 version으로 PUT | 저장 버튼 재활성화 |
