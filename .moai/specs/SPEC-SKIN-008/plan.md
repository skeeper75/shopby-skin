# SPEC-SKIN-008: 관리자 거래처/원장/통계 - 구현 계획

> **SPEC ID**: SPEC-SKIN-008

## 1. 마일스톤

### Primary Goal: 월별 매출통계 (핵심 경영지표)
- 월별 매출통계 대시보드 (요약 카드 + 차트 + 상세 테이블)
- 기간별 필터링 (년/월 단위)
- Excel 다운로드
- 카테고리별 매출 비교 (인쇄/굿즈/패키지)

### Secondary Goal: 원장관리 + 거래처
- 계좌관리 (원장용 계좌 등록/수정)
- 원장관리 (오프라인 거래 등록, Excel/청구서 업로드, 매칭 처리)
- 업체별 미수금 관리 (Excel 다운로드)
- 거래처관리 (등록/수정/삭제, 업체목록 .txt 생성)

### Final Goal: 상품통계 + 팀별통계
- 인쇄/제본 상품통계 (상세보기)
- 굿즈 상품통계 (상세보기)
- 패키지 상품통계 (상세보기)
- 수작 상품통계 (상세보기)
- 굿즈 발주/정산 (Excel 다운, 외주 정산)
- 제작상품 팀별통계 (Excel 다운, 공정관리)

### Optional Goal: 매장게시판
- 매장게시판 CRUD

## 2. 기술적 접근
- **전량 커스텀 개발**: 모든 API가 [C] 분류 — Shopby API 없음
- **차트 라이브러리**: Recharts (React 친화적, 가벼움) 또는 Chart.js
- **Excel 처리**: SheetJS (xlsx) — 업로드/다운로드 모두 지원
- **데이터 집계**: 서버 사이드 집계 쿼리 → API 응답으로 전달
- **테이블 컴포넌트**: TanStack Table + shadcn/ui DataTable 패턴 재활용
- **원장 매칭**: Excel 업로드 → 파싱 → 거래 매칭 UI (드래그&드롭 또는 자동매칭)

## 3. 리스크

| 리스크 | 영향도 | 대응 |
|--------|--------|------|
| 전량 커스텀 API (백엔드 의존) | High | API 스펙 우선 확정, Mock API로 프론트 병렬 개발 |
| 통계 데이터 대용량 (수년치) | Medium | 서버 사이드 페이지네이션 + 기간 필터 필수 |
| Excel 파싱 오류 | Medium | 업로드 전 유효성 검사 + 에러 행 표시 |
| 원장 매칭 복잡도 | High | 자동매칭 알고리즘 + 수동매칭 UI 병행 |
| 차트 렌더링 성능 | Low | 데이터 포인트 제한 (월별 → 최대 36개월) |
