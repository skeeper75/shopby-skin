## 3. 분석 갭 — shopby Native vs 미분석 영역

### shopby에서 제공하지만 아직 분석/구현하지 않은 영역

| 카테고리 | 기능 | shopby Native | 커스텀 필요 여부 |
|----------|------|--------------|-----------------|
| 클레임관리 | 취소/교환/반품/환불 | ✅ Native API | ❓ 미확인 |
| 프로모션 | 추가할인/사은품 | ✅ Native API | ❓ 미확인 |
| 운영관리 | SMS/메일/알림톡 | ✅ Native API | ❓ 미확인 |
| 전시관리 | 배너/팝업/카테고리 | ✅ Native API | ❓ Skin 레벨 |
| 파트너관리 | 파트너사/정산 | ✅ Native API | ❓ 미확인 |
| 서비스관리 | 약관/도메인/권한 | ✅ Native API | ❓ 미확인 |

### Playwright 분석으로 확인해야 할 핵심 질문

1. shopby 관리자 페이지 중 **인쇄업에 불필요한 기능**은 무엇인가?
2. shopby Native UI를 **그대로 사용할 수 있는 영역**은 어디인가?
3. **Skin 커스터마이징만으로 충분한 영역** vs **별도 Custom 개발이 필요한 영역**은?
4. 현재 SPEC-SKIN-005~008에서 **중복 구현된 기능**이 있는가?
5. shopby API만 활용하고 **관리자 UI는 shopby Native를 사용할 수 있는 영역**은?

## 4. Playwright 분석 전략

### 크롤링 대상 페이지 (예상 80~120 페이지)

```
https://service.shopby.co.kr/
├── /admin/product/* (상품관리 ~30 페이지)
├── /admin/order/* (주문관리 ~20 페이지)
├── /admin/claim/* (클레임관리 ~15 페이지)
├── /admin/member/* (회원관리 ~10 페이지)
├── /admin/promotion/* (프로모션관리 ~10 페이지)
├── /admin/management/* (운영관리 ~25 페이지)
├── /admin/statistic/* (통계 ~10 페이지)
├── /admin/service/* (서비스관리 ~15 페이지)
├── /admin/appearance/* (전시관리 ~20 페이지)
└── /admin/partner/* (파트너관리 ~10 페이지)
```

### 분석 방법론

1. **자동 로그인**: 환경 변수 기반 인증
2. **메뉴 크롤링**: 사이드바 메뉴 자동 탐색으로 모든 페이지 URL 수집
3. **스크린캡처**: 각 페이지 풀 스크린샷 저장
4. **DOM 분석**: 각 페이지의 폼/테이블/버튼 구조 추출
5. **기능 매핑**: 문서 기반 기능 설명과 실제 UI 매핑
