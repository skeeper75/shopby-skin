# SPEC-ORDER-001: 수용 기준

> A6B8-ORDER 주문관리 도메인 인수 테스트 시나리오

---

## 1. 모듈별 수용 기준

### 1.1 모듈 1: 파일 업로드/정보입력

#### AC-ORDER-001: 드래그앤드롭 파일 업로드

```gherkin
GIVEN 인쇄 상품 주문 페이지에 있고
WHEN PDF 파일(50MB)을 업로드 영역에 드래그앤드롭하면
THEN 업로드 진행률이 실시간으로 표시되고
AND S3에 파일이 저장되고
AND 클라이언트 기본 검증(포맷, 크기)이 즉시 실행된다
```

- 관련 요구사항: REQ-ORDER-004, REQ-ORDER-005
- 검증 방법: E2E 테스트 (Playwright)

#### AC-ORDER-002: PitStop 자동 검증 성공

```gherkin
GIVEN 파일이 S3에 업로드 완료되었고
WHEN PitStop 프리플라이트 검증이 실행되면
THEN 해상도(300DPI), 컬러모드(CMYK), 도련(3mm), 폰트 임베딩 항목이 표시되고
AND 모든 항목이 통과 시 녹색 "검증 완료" 배지가 표시된다
```

- 관련 요구사항: REQ-ORDER-006, REQ-ORDER-007
- 검증 방법: 통합 테스트 (PitStop Mock)

#### AC-ORDER-003: PitStop 경고 결과 처리

```gherkin
GIVEN 파일 검증 결과가 '경고'(해상도 200DPI)이고
WHEN 경고 내용이 표시된 상태에서 "인쇄 품질 저하 가능성 동의" 체크박스를 선택하면
THEN 주문 진행이 허용된다
AND 주문 데이터에 경고 동의 기록이 저장된다
```

- 관련 요구사항: REQ-ORDER-008
- 검증 방법: 통합 테스트

#### AC-ORDER-004: PitStop 오류 결과 차단

```gherkin
GIVEN 파일 검증 결과가 '오류'(RGB 컬러모드)이고
WHEN 사용자가 주문 진행을 시도하면
THEN "인쇄 파일을 수정 후 재업로드해주세요" 메시지가 표시되고
AND 주문 진행이 차단되고
AND "재업로드" 버튼이 제공된다
```

- 관련 요구사항: REQ-ORDER-009, REQ-ORDER-012
- 검증 방법: E2E 테스트

#### AC-ORDER-005: 업로드 중 이탈 방지

```gherkin
GIVEN 파일 업로드가 진행 중(70%)이고
WHEN 사용자가 브라우저 뒤로가기 또는 탭 닫기를 시도하면
THEN "파일 업로드가 진행 중입니다. 페이지를 떠나시겠습니까?" 확인 다이얼로그가 표시된다
```

- 관련 요구사항: REQ-ORDER-013
- 검증 방법: E2E 테스트

### 1.2 모듈 2: 보관함/장바구니

#### AC-ORDER-006: 옵션보관함 저장 및 재주문

```gherkin
GIVEN 인쇄 옵션(A4, 아트지 200g, 양면컬러, 100부)과 파일이 설정되어 있고
WHEN "보관함에 저장" 버튼을 클릭하면
THEN 옵션보관함에 해당 조합이 저장되고
AND 보관함 페이지에서 저장된 항목이 표시되고
AND "재주문" 클릭 시 동일 옵션과 파일이 주문서에 자동 구성된다
```

- 관련 요구사항: REQ-ORDER-014, REQ-ORDER-016
- 검증 방법: E2E 테스트

#### AC-ORDER-007: 보관함 30일 만료

```gherkin
GIVEN 30일 전에 저장된 보관함 항목이 있고
WHEN 시스템 배치 작업이 실행되면
THEN 해당 항목이 자동 삭제되고
AND 연결된 S3 파일도 함께 삭제된다
```

- 관련 요구사항: REQ-ORDER-015, REQ-ORDER-017
- 검증 방법: 배치 테스트

#### AC-ORDER-008: 인쇄 상품 파일 없이 장바구니 추가 차단

```gherkin
GIVEN 인쇄 상품(전단지)을 선택했고
WHEN 파일 업로드 없이 "장바구니 담기"를 시도하면
THEN "인쇄 파일을 먼저 업로드해주세요" 메시지가 표시되고
AND 장바구니 추가가 차단된다
```

- 관련 요구사항: REQ-ORDER-019
- 검증 방법: E2E 테스트

### 1.3 모듈 3: 배송/결제

#### AC-ORDER-009: 기본 배송비 적용

```gherkin
GIVEN 주문 금액이 30,000원이고
WHEN 주문서 작성 화면에서 배송비를 확인하면
THEN 기본 배송비 3,000원이 표시되고
AND 총 결제금액은 33,000원이다
```

- 관련 요구사항: REQ-ORDER-021
- 검증 방법: 단위 테스트

#### AC-ORDER-010: 무료배송 적용

```gherkin
GIVEN 주문 금액이 55,000원이고
WHEN 주문서 작성 화면에서 배송비를 확인하면
THEN 배송비가 0원(무료배송)으로 표시되고
AND 총 결제금액은 55,000원이다
```

- 관련 요구사항: REQ-ORDER-020
- 검증 방법: 단위 테스트

#### AC-ORDER-011: 제주 추가 배송비

```gherkin
GIVEN 주문 금액이 30,000원이고 배송지가 제주도이고
WHEN 주문서에서 배송비를 확인하면
THEN 기본 배송비 3,000원 + 제주 추가 3,000원 = 총 6,000원이 표시된다
```

- 관련 요구사항: REQ-ORDER-022
- 검증 방법: 단위 테스트

#### AC-ORDER-012: KG이니시스 결제 성공

```gherkin
GIVEN 주문서 작성이 완료되고 신용카드를 선택했고
WHEN "결제하기" 버튼을 클릭하면
THEN KG이니시스 결제 팝업이 표시되고
AND 결제 완료 후 주문 완료 페이지로 이동하고
AND 주문번호와 결제 금액이 표시된다
```

- 관련 요구사항: REQ-ORDER-028, REQ-ORDER-032
- 검증 방법: E2E 테스트 (PG 테스트 모드)

#### AC-ORDER-013: 결제 실패 팝업 + 알림톡

```gherkin
GIVEN 결제를 시도했고
WHEN 결제가 실패(카드 한도 초과)하면
THEN 팝업으로 "결제에 실패했습니다: 카드 한도 초과" 메시지가 표시되고
AND "다른 결제수단으로 재시도" 버튼이 제공되고
AND 카카오 알림톡으로 결제 실패 안내와 재결제 링크가 발송된다
```

- 관련 요구사항: REQ-ORDER-030, REQ-ORDER-031
- 검증 방법: E2E 테스트

### 1.4 모듈 4: 주문관리 (관리자)

#### AC-ORDER-014: 관리자 주문 목록 필터링

```gherkin
GIVEN 관리자 주문관리 페이지에 접근하고
WHEN 상태 필터를 "파일확인중"으로 설정하면
THEN FILE_CHECKING 상태의 주문만 표시되고
AND 각 주문에 주문번호, 고객명, 상품명, 접수일, 현재 상태가 표시된다
```

- 관련 요구사항: REQ-ORDER-037
- 검증 방법: E2E 테스트

#### AC-ORDER-015: 주문상태 순방향 변경

```gherkin
GIVEN FILE_CHECKING 상태의 주문이 있고
WHEN 관리자가 상태를 "PRINT_READY"로 변경하면
THEN shopby 주문 상태가 업데이트되고
AND 커스텀 인쇄 상태가 동기화되고
AND 고객에게 "제작 시작" 알림톡이 발송된다
```

- 관련 요구사항: REQ-ORDER-039, REQ-ORDER-040
- 검증 방법: 통합 테스트

#### AC-ORDER-016: 역방향 상태 변경 차단

```gherkin
GIVEN PRINTING 상태의 주문이 있고
WHEN 관리자가 상태를 "FILE_CHECKING"으로 변경을 시도하면
THEN "역방향 상태 변경은 불가합니다" 오류가 표시되고
AND 상태 변경이 차단된다
```

- 관련 요구사항: REQ-ORDER-043
- 검증 방법: 통합 테스트

#### AC-ORDER-017: 주문 일괄 상태변경

```gherkin
GIVEN PACKAGING 상태의 주문 5건을 선택하고
WHEN "일괄 상태변경 > SHIPPED" 버튼을 클릭하면
THEN 5건 모두 SHIPPED 상태로 변경되고
AND 5명의 고객에게 출고 알림톡이 각각 발송된다
```

- 관련 요구사항: REQ-ORDER-041
- 검증 방법: E2E 테스트

### 1.5 모듈 5: 파일확인/재업로드

#### AC-ORDER-018: 파일 검수 승인

```gherkin
GIVEN FILE_CHECKING 상태의 주문 파일을 확인하고
WHEN 관리자가 "승인" 버튼을 클릭하면
THEN 주문 상태가 PRINT_READY로 변경되고

```

- 관련 요구사항: REQ-ORDER-046
- 검증 방법: 통합 테스트

#### AC-ORDER-019: 재업로드 요청 발송

```gherkin
GIVEN 파일 검수 결과 문제가 발견되어
WHEN 관리자가 사유("해상도 부족")를 선택하고 "재업로드 요청" 버튼을 클릭하면
THEN 고객에게 알림톡+SMS로 문제 내용과 재업로드 링크가 발송되고
AND 주문 상태에 "재업로드 대기" 태그가 추가된다
```

#### AC-ORDER-020: 재업로드 48시간 타임아웃

```gherkin

WHEN 고객이 아직 재업로드하지 않았으면
THEN 관리자에게 "미응답 재업로드" 알림이 표시되고
AND 고객에게 리마인드 SMS가 자동 재발송된다
```

- 관련 요구사항: REQ-ORDER-049
- 검증 방법: 배치 테스트

### 1.6 모듈 6: 후불결제/증빙/SMS

#### AC-ORDER-021: B2B 후불결제 주문

```gherkin
GIVEN B2B 거래처(후불 승인 등급) 계정으로 로그인하고

THEN 결제 없이 주문이 생성되고
AND 주문 상태가 "미결제"로 기록되고
AND 관리자 후불결제 목록에 해당 주문이 표시된다
```

- 관련 요구사항: REQ-ORDER-051
- 검증 방법: E2E 테스트

#### AC-ORDER-022: 후불결제 비승인 차단

```gherkin
GIVEN 일반 회원(후불 비승인) 계정으로 로그인하고
WHEN 결제 수단에서 "후불결제" 옵션을 확인하면
THEN 후불결제 옵션이 표시되지 않거나 비활성화 상태이다
```

- 관련 요구사항: REQ-ORDER-054
- 검증 방법: E2E 테스트

#### AC-ORDER-023: 세금계산서 발급

```gherkin
GIVEN 결제 완료된 주문이 있고
WHEN 관리자가 "세금계산서 발급" 버튼을 클릭하고 사업자 정보를 입력하면
THEN 팝빌 API를 통해 전자세금계산서가 발급되고
AND 발급된 세금계산서가 이메일로 전송된다
```

- 관련 요구사항: REQ-ORDER-055
- 검증 방법: 통합 테스트 (팝빌 테스트 모드)

#### AC-ORDER-024: 고객 SMS 일괄 발송

```gherkin
GIVEN 주문 3건을 선택하고
```

- 검증 방법: 통합 테스트
