## 2. 모듈 1: 파일 업로드/정보입력

### 기능 개요

인쇄 전용 파일 업로드 시스템. shopby에 완전 미지원되는 XL 규모 CUSTOM 개발.

### 핵심 요구사항

- 드래그앤드롭 업로드 (react-dropzone + S3 multipart)
- 클라이언트 기본 검증: 포맷(PDF/AI/PSD/JPG/PNG), 크기(300MB/파일, 1GB/주문)
- 서버 상세 검증: PitStop Server 연동 (해상도, CMYK, 도련, 폰트)
- 검증 결과 3단계: 통과(녹색) / 경고(노란색) / 오류(빨간색)
- 상품별 PDF 가이드라인 템플릿 다운로드

### 의존성

- AWS S3: 파일 스토리지
- PitStop Server: PDF 프리플라이트 검증
- 상품 옵션 시스템: 옵션 조합과 파일 연결

## 3. 모듈 2: 보관함/장바구니

### 기능 개요

shopby 장바구니에 인쇄 옵션 저장이 불가하여 커스텀 옵션보관함 구현.

### 핵심 요구사항

- shopby 장바구니 API + 커스텀 옵션 스토리지 이중 저장
- 30일 보관 후 자동 삭제 (파일 포함)
- 보관함에서 1클릭 재주문

### 설계 고려사항

- shopby 장바구니에는 기본 상품 정보만 저장
- 인쇄 옵션(용지, 후가공, 수량 등) + 파일 참조는 커스텀 DB에 저장
- 주문 시 양쪽 데이터를 조합하여 주문서 구성

## 4. 모듈 3: 배송/결제

### 기능 개요

shopby NATIVE 배송/결제 Provider 활용. PG(KG이니시스), 간편결제(네이버페이) 연동.

### 핵심 요구사항

- 배송비 정책: 50,000원 이상 무료, 기본 3,000원, 제주 +3,000원, 도서산간 +5,000원
- 배너/현수막 별도 배송비 템플릿 (무료배송 제외)
- 혼합주문 최대배송비 1건 부과
- KG이니시스 PG + 네이버페이 간편결제
- 결제 실패 시 팝업 + 알림톡

## 8. 엣지 케이스 및 리스크

| 시나리오 | 영향 | 대응 |
|---------|------|------|
| 대용량 파일(300MB) 업로드 중 네트워크 끊김 | 업로드 실패, 고객 이탈 | S3 multipart + 이어받기 구현 |
| PitStop Server 장애 | 파일 검증 불가, 주문 진행 차단 | 수동 검수 fallback + 관리자 수동 승인 경로 |
| shopby 상태와 인쇄 상태 불일치 | 고객 혼란, 알림 오류 | 이벤트 기반 동기화 + 불일치 감지 배치 |
| 동시 결제 중복 요청 | 이중 결제 | 멱등성 키 + 결제 상태 잠금 |
| 보관함 파일 30일 만료 직전 재주문 시도 | 파일 참조 오류 | 만료 3일 전 알림 + 유예기간 제공 |
| 알림톡 대량 발송 실패 | 고객 미도달 | 발송 큐 + 실패 시 SMS 자동 폴백 + 재시도 |
