# SPEC-SKIN-007 v2: 관리자 게시판/회원/쿠폰 - 인수 기준

## 1. 공지사항/FAQ 관리 (REQ-007-001)

### AC-007-001: 공지사항 등록
```gherkin
Given 관리자가 공지사항 등록 페이지에서
And 제목 "3월 휴무 안내"를 입력하고
And 내용을 HTML 에디터로 작성하면
When "등록" 버튼을 클릭하면
Then 공지사항이 저장되고
And Snackbar로 "공지사항이 등록되었습니다" 알림이 표시되고
And 사용자 사이트 공지사항에 즉시 노출된다
```

### AC-007-002: 공지사항 공개/비공개 토글
```gherkin
Given 공지사항 목록에서 게시글이 "공개" 상태이고
When 관리자가 Switch를 토글하면
Then 상태가 "비공개"로 변경되고
And 사용자 사이트에서 해당 공지가 숨겨진다
```

## 2. 문의/상담 답변 (REQ-007-002, REQ-007-003)

### AC-007-003: 견적문의 답변 등록
```gherkin
Given 관리자가 견적문의 목록에서 "대기" 상태의 문의를 클릭하고
And 우측 빠른 답변 패널이 열리면
When 답변 에디터에 내용을 작성하고 "답변 등록" 버튼을 클릭하면
Then 답변이 저장되고
And 상태가 "대기"에서 "완료"로 변경되고
And Chip 색상이 변경되고
And 고객에게 답변 알림이 발송된다
```

### AC-007-004: Q&A 답변
```gherkin
Given 관리자가 상품Q&A 목록에서 미답변 문의를 선택하면
When 답변을 작성하고 등록하면
Then shopby API를 통해 답변이 저장되고
And 고객에게 알림이 발송된다
```

### AC-007-005: 상담 상태 필터
```gherkin
Given 관리자가 견적문의 목록에서
When 상태 필터를 "대기"로 선택하면
Then 대기 상태의 문의만 필터링되어 표시된다
```

## 3. 체험단/후기 (REQ-007-004, REQ-007-005)

### AC-007-006: 체험단 모집글 등록
```gherkin
Given 관리자가 체험단관리 페이지에서 "모집글 등록" 버튼을 클릭하고
And HTML 에디터에 모집 내용을 입력하면
When "등록" 버튼을 클릭하면
Then 모집글이 DB에 저장되고 사용자 사이트에 표시된다
```

### AC-007-007: 체험단 당첨 처리
```gherkin
Given 체험단 신청자 목록에서
And 관리자가 Checkbox로 당첨 대상 3명을 선택하면
When "당첨 처리" 버튼을 클릭하면
Then 선택된 3명의 상태가 "당첨"으로 변경되고
And 당첨자에게 알림이 발송되고
And 당첨자만 후기 작성 권한이 활성화된다
```

### AC-007-008: 관리자 후기 임의 등록
```gherkin
Given 관리자가 이용후기관리에서 "후기 등록" 버튼을 클릭하고
And 상품을 선택하고 별점 5점, 내용 "품질이 우수합니다"를 입력하면
When "등록" 버튼을 클릭하면
Then 해당 상품에 관리자 작성 후기가 등록된다
```

## 4. 회원관리 (REQ-007-006, REQ-007-007)

### AC-007-009: 회원 검색
```gherkin
Given 관리자가 회원관리 페이지에서
When 검색 TextField에 "홍길동"을 입력하면 (debounce 300ms)
Then 이름에 "홍길동"이 포함된 회원 목록이 필터링되어 표시된다
```

### AC-007-010: 회원 상세 Drawer
```gherkin
Given 관리자가 회원 목록에서 "홍길동" 행을 클릭하면
When 우측 Drawer가 열리면
Then 기본정보(이름, 이메일, 전화번호, 가입일)가 표시되고
And 프린팅머니 잔액이 표시되고
And 최근 주문내역이 표시되고
And 보유 쿠폰 수가 표시된다
```

### AC-007-011: 탈퇴회원 개인정보 마스킹
```gherkin
Given 관리자가 탈퇴회원 목록에 접속하면
Then 이름은 "홍**"으로 마스킹되고
And 전화번호는 "010-****-5678"으로 마스킹되어 표시된다
```

## 5. 프린팅머니 (REQ-007-008, REQ-007-011)

### AC-007-012: 프린팅머니 지급
```gherkin
Given 관리자가 회원 상세에서 "프린팅머니 지급" 버튼을 클릭하고
And Dialog에서 금액 "5000", 사유 "이벤트 당첨"을 입력하면
When "지급" 버튼을 클릭하면
Then 회원의 프린팅머니 잔액이 5000원 증가하고
And 변경 이력에 기록되고
And Snackbar로 "5,000원이 지급되었습니다" 알림이 표시된다
```

### AC-007-013: 프린팅머니 음수 잔액 방지
```gherkin
Given 회원의 프린팅머니 잔액이 3000원이고
When 관리자가 5000원 차감을 시도하면
Then "잔액이 부족합니다. 현재 잔액: 3,000원" 에러가 표시되고
And 차감이 실행되지 않는다
```

### AC-007-014: 사유 미입력 시 처리 불가
```gherkin
Given 관리자가 프린팅머니 조정 Dialog에서
And 금액은 입력했지만 사유를 비워두면
When "지급" 또는 "차감" 버튼을 클릭하면
Then Field에 "사유를 입력해주세요" 에러가 표시되고
And 처리가 실행되지 않는다
```

## 6. 쿠폰관리 (REQ-007-009, REQ-007-010, REQ-007-013)

### AC-007-015: 쿠폰 발행
```gherkin
Given 관리자가 쿠폰 발행 폼에서
And 쿠폰명 "봄맞이 할인", 할인유형 "정율(10%)", 매칭 "출력상품 전체"를 설정하고
And 유효기간 "2026-04-01 ~ 2026-04-30"을 설정하면
When "발행하기" 버튼을 클릭하면
Then 쿠폰이 생성되고
And Snackbar로 "쿠폰이 발행되었습니다" 알림이 표시되고
And 사용자 측에서 즉시 적용 가능하다
```

### AC-007-016: 쿠폰 활성/비활성 토글
```gherkin
Given 쿠폰 목록에서 "봄맞이 할인" 쿠폰이 활성 상태이고
When 관리자가 Switch를 토글하면
Then 쿠폰 상태가 비활성으로 변경되고
And 사용자 측에서 해당 쿠폰 사용이 불가능해진다
```

### AC-007-017: 쿠폰 내역 탭 전환
```gherkin
Given 관리자가 쿠폰관리 페이지에서
When Tabs에서 "사용내역" 탭을 선택하면
Then 쿠폰 사용내역 목록(쿠폰명, 회원, 사용일, 할인금액, 주문번호)이 표시된다
```

## 7. 컴포넌트 렌더링 검증

### AC-007-018: Huni DS 컴포넌트 렌더링
```gherkin
Given 관리자 게시판/회원/쿠폰 페이지에 접속하면
Then Tabs는 Huni Tabs로 렌더링되고 (line variant, indicator animation)
And Dialog는 Huni Dialog로 렌더링되고 (lazyMount 적용)
And Chip은 Huni Chip으로 상태 표시하고 (data-selected)
And Switch는 Huni Switch로 토글 동작하고 (data-checked)
And 모든 스타일은 --huni-* 토큰을 사용한다
```
