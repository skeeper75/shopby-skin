## 1. 핵심 의사결정 사항

### KD-CS-01: 공지사항 분류 체계 (A-4-1)

**결정 항목**: 공지사항 카테고리 분류 방식

**선택 가능 옵션**:
| 옵션 | 내용 | 장점 | 단점 |
|------|------|------|------|
| A | 3개 카테고리 (공지/이벤트/상품) | 직관적, 관리 간편 | 세분화 부족 |
| B | 5개 카테고리 (공지/이벤트/상품/인쇄가이드/업데이트) | 상세 분류 | 관리 부담 |
| C | 태그 기반 자유 분류 | 유연함 | UX 복잡, shopby 기본 미지원 |

**권장 결정**: **3개 카테고리 (옵션 A)**

**근거**:
- shopby 게시판 카테고리 기능으로 즉시 구현 가능
- 소규모 운영 초기에는 3개 카테고리로 충분
- 인쇄가이드는 FAQ에서 충분히 커버 가능
- 필요 시 카테고리 추가 용이 (shopby 관리자 설정)

**shopby 구현 가능 여부**: 가능 (게시판 카테고리 설정)

### KD-CS-02: FAQ 분류 체계 (A-4-2)

**결정 항목**: FAQ 카테고리 분류 기준

**선택 가능 옵션**:
| 옵션 | 내용 | 장점 | 단점 |
|------|------|------|------|
| A | 6개 카테고리 (주문방법/배송/결제/인쇄품질/파일업로드/기타) | 인쇄 업종 특화 | 카테고리 수 많음 |
| B | 4개 카테고리 (주문/배송/결제/기타) | 일반적, 간결 | 인쇄 특화 부족 |
| C | 검색 중심 (카테고리 최소화) | 유연 | 브라우징 불편 |

**권장 결정**: **6개 카테고리 (옵션 A)**

**근거**:
- 인쇄 업종은 파일 업로드, 인쇄 품질 관련 문의가 빈번
- 카테고리 세분화로 고객 자가 해결률 향상
- shopby 게시판 카테고리 기능으로 구현 가능
- 벤치마크: 레드프린팅 6개, 와우프레스 5개 카테고리 운영

**shopby 구현 가능 여부**: 가능 (게시판 카테고리 설정)

### KD-CS-03: 비회원 주문조회 인증 방식 (A-4-7)

**결정 항목**: 비회원 주문조회 시 필요한 인증 정보 조합

**선택 가능 옵션**:
| 옵션 | 내용 | 보안성 | 편의성 |
|------|------|--------|--------|
| A | 주문번호만 | 낮음 | 높음 |
| B | 주문번호+이메일 | 중간 | 중간 |
| C | 주문번호+이메일+휴대폰 | 높음 | 낮음 |

**권장 결정**: **주문번호+이메일+휴대폰 3중 조합 (옵션 C)**

**근거**:
- 인쇄물 주문은 개인/기업 정보 포함으로 보안 중요
- 비회원 주문 시 이메일+휴대폰 필수 수집 (SPEC-MEMBER-001 KD-07)
- Rate Limiting과 결합하여 무차별 대입 방지
- shopby GuestOrderProvider 기본 지원 범위 내

**shopby 구현 가능 여부**: 기본 주문번호+비밀번호 지원, 이메일+휴대폰 조합은 프론트 추가 검증 필요

### KD-CS-04: 1:1문의 답변 알림 방식 (B-5-7)

**결정 항목**: 1:1문의 답변 완료 시 고객 알림 채널

**선택 가능 옵션**:
| 옵션 | 내용 | 인지율 | 비용 |
|------|------|--------|------|
| A | 이메일만 | 중간 | 낮음 |
| B | 알림톡만 | 높음 | 중간 |
| C | 이메일+알림톡 동시 | 최고 | 높음 |

**권장 결정**: **이메일+알림톡 동시 발송 (옵션 C)**

**근거**:
- 결제 실패 안내도 팝업+알림톡으로 확정됨 (정책 #10)
- 인쇄 업종은 납기 관련 문의 응답 인지가 중요
- 초기에는 이메일만 구현하고, 알림톡은 카카오 비즈메시지 연동 후 추가

**shopby 구현 가능 여부**: 이메일은 shopby 기본, 알림톡은 외부 연동 필요

### KD-CS-05: 체험단 운영 방식 (B-5-8)

**결정 항목**: 체험단 모집/관리 시스템 구현 수준

**선택 가능 옵션**:
| 옵션 | 내용 | 개발 규모 | 운영 편의성 |
|------|------|----------|-----------|
| A | 최소 기능 (HTML 공고 + 수동 당첨) | M | 낮음 |
| B | 중간 기능 (신청 폼 + 자동 추첨) | L | 중간 |
| C | 풀 기능 (신청/심사/당첨/후기/포인트 연동) | XL | 높음 |

**권장 결정**: **최소 기능 (옵션 A) 로 시작, 단계적 확장**

**근거**:
- P3 우선순위로 런칭 후 개발
- 초기에는 관리자가 HTML 공고를 직접 입력하고 수동으로 당첨 처리
- 사용 빈도와 피드백에 따라 옵션 B/C로 확장
- custom-dev-data.json에서 "html DB직접입력, 당첨처리"로 정의됨

**shopby 구현 가능 여부**: 완전 미지원 (CUSTOM 필수)

### KD-CS-07: 관리자 역할 단계 (B-1-1)

**결정 항목**: 관리자 역할 체계의 단계 수

**선택 가능 옵션**:
| 옵션 | 내용 | 관리 복잡성 | 보안성 |
|------|------|-----------|--------|
| A | 3단계 (대표/운영자/부운영자) | 낮음 | 중간 |
| B | 4단계 (대표/총괄운영자/운영자/부운영자) | 중간 | 높음 |

**권장 결정**: **3단계 (옵션 A)**

**근거**:
- 후니프린팅은 소규모 운영 (직원 수 제한적)
- 3단계로 충분한 권한 분리 가능
- shopby 운영자관리에서 역할 그룹 커스터마이징 지원
- 필요 시 4단계로 확장 용이

**shopby 구현 가능 여부**: 가능 (운영자관리 역할 설정)

### KD-CS-08: 관리자 2FA 도입 여부 (B-1-1)

**결정 항목**: 관리자 로그인 시 2FA(2단계 인증) 도입 시점

**선택 가능 옵션**:
| 옵션 | 내용 | 보안성 | 구현 난이도 |
|------|------|--------|-----------|
| A | 즉시 도입 (이메일 OTP) | 높음 | M |
| B | P2 이후 검토 | 중간 | - |
| C | 미도입 | 낮음 | - |

**권장 결정**: **P2 이후 검토 (옵션 B)**

**근거**:
- 초기 런칭은 비밀번호 인증 + IP 제한으로 충분
- 관리자 수가 소규모이므로 2FA 긴급성 낮음
- P2에서 이메일 OTP 또는 Google Authenticator 검토

**shopby 구현 가능 여부**: shopby 자체 미지원, 커스텀 개발 필요

## 2. 모듈 1: 쇼핑몰 고객센터

### 2.1 공지사항 (A-4-1)

**기능 요약**: shopby 게시판 기능을 활용한 공지사항 노출

**구현 방식**: NATIVE (shopby BoardProvider)

**주요 기능**:
- 카테고리별 공지사항 목록 (공지/이벤트/상품)
- 상단 고정 게시글
- 페이지네이션 (10건/페이지)
- 게시글 상세 조회 (제목, 내용, 작성일, 조회수)
- 이전글/다음글 네비게이션

**인쇄 업종 특화**:
- 상품 카테고리에서 신규 인쇄물/특수 재질 안내 가능
- 이벤트 카테고리에서 할인/체험단 안내 연동

### 2.2 FAQ (A-4-2)

**기능 요약**: 카테고리별 FAQ 아코디언 표시

**구현 방식**: NATIVE (shopby BoardProvider)

**주요 기능**:
- 6개 카테고리: 주문방법, 배송, 결제, 인쇄품질, 파일업로드, 기타
- 아코디언 UI (질문 클릭 시 답변 펼침)
- 카테고리 필터링
- 키보드 접근성 (Enter/Space 토글)

**인쇄 업종 특화**:
- 인쇄품질 카테고리: DPI, 색상모드(CMYK), 재단선 관련 FAQ
- 파일업로드 카테고리: 지원 파일 형식, 용량 제한, 템플릿 안내

### 2.3 비회원 주문조회 (A-4-7)

**기능 요약**: 비회원이 주문 상태를 확인할 수 있는 조회 기능

**구현 방식**: NATIVE (shopby GuestOrderProvider)

**주요 기능**:
- 3중 인증 (주문번호+이메일+휴대폰)
- 주문 상태 표시 (결제완료/제작중/배송중/배송완료)
- 배송 추적 링크
- 조회 실패 시 회원가입 유도 CTA
- Rate Limiting (IP당 10회/시간)

**SPEC-MEMBER-001 연동**:
- KD-07에서 비회원 주문은 굿즈만 허용 (인쇄물은 회원 필수)
- 비회원 주문 시 이메일+휴대폰 필수 수집 → 조회 시 3중 인증에 활용

## 3. 모듈 2: 관리자 게시판 관리

### 3.1 공지사항 관리 (B-5-1)

**구현 방식**: NATIVE (shopby Admin Board API)

**관리자 기능**:
- 공지사항 CRUD (등록/수정/삭제)
- 상단 고정 설정/해제
- 카테고리 지정
- 에디터를 통한 HTML 내용 입력
- 예약 게시 (가능 시)

### 3.2 FAQ 관리 (B-5-2)

**구현 방식**: NATIVE (shopby Admin Board API)

**관리자 기능**:
- FAQ CRUD (등록/수정/삭제)
- 카테고리 관리 (추가/수정/삭제)
- 노출 순서 변경 (드래그앤드롭)
- 비활성화 (노출/비노출 토글)

### 3.3 1:1문의 관리 (B-5-7)

**구현 방식**: NATIVE (shopby Admin Inquiry API)

**관리자 기능**:
- 문의 목록 조회 (상태별 필터: 대기/처리중/완료)
- 답변 작성 + 등록
- 문의 상태 변경
- 답변 완료 시 고객 알림 발송 (이메일, 추후 알림톡)
- 문의 검색 (제목/작성자/날짜)

### 3.4 이용후기 관리 (B-5-9)

**구현 방식**: NATIVE (shopby Admin Review API)

**관리자 기능**:
- 이용후기 목록 조회
- 이용후기 삭제 + 적립금 자동 회수
- 관리자 답변 작성
- 관리자 임의 후기 등록 (초기 리뷰 시딩)
- 포토 리뷰 관리

**적립금 연동**:
- 일반 리뷰 적립금: 1,000원 (정책 #14)
- 포토 리뷰 적립금: 2,000원 (정책 #25)
- 삭제 시 자동 회수 (정책 #24)

### 3.5 체험단관리 (B-5-8)

**구현 방식**: CUSTOM (자체 DB + 관리 UI)

**관리자 기능**:
- 체험단 모집 공고 등록 (HTML 에디터)
- 모집기간/모집인원/체험상품 설정
- 신청자 목록 관리
- 당첨자 수동 선정 + 알림 발송
- 체험 후기 관리 (연동)
- 모집 기간 자동 마감

**DB 설계 방향**:
- `experience_campaigns`: 체험단 공고 테이블
- `experience_applications`: 신청자 테이블
- `experience_results`: 당첨 결과 테이블

## 5. 엣지 케이스 및 리스크

### 5.1 비회원 주문조회 엣지 케이스

| 케이스 | 대응 |
|--------|------|
| 동일 이메일로 여러 비회원 주문 존재 | 주문번호로 특정 → 해당 주문만 조회 |
| 비회원 주문 후 회원가입한 경우 | 비회원 조회와 회원 주문내역 모두 접근 가능 |
| Rate Limiting 초과 후 정상 사용자 차단 | IP 차단 해제 시간 안내 (1시간 후) |

### 5.2 체험단관리 엣지 케이스

| 케이스 | 대응 |
|--------|------|
