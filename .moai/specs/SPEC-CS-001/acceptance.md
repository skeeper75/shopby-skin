# SPEC-CS-001: 수용 기준

> A4B5-CS + B1-ADMIN 고객센터/관리자 도메인 인수 테스트 시나리오

---

## 1. 모듈별 수용 기준

### 1.1 모듈 1: 쇼핑몰 고객센터

#### AC-CS-001: 공지사항 목록 조회

```gherkin
GIVEN 공지사항이 3개 카테고리(공지/이벤트/상품)에 등록되어 있고
WHEN 사용자가 고객센터 > 공지사항 페이지에 접근하면
THEN 전체 공지사항이 최신순으로 표시되고
AND 상단 고정 게시글이 항상 최상단에 노출되며
AND 카테고리 필터 탭이 제공된다
```

- 관련 요구사항: REQ-CS-001, REQ-CS-003
- 검증 방법: E2E 테스트 (Playwright)

#### AC-CS-002: 공지사항 카테고리 필터링

```gherkin
GIVEN 공지사항에 공지 5건, 이벤트 3건, 상품 2건이 있고
WHEN 사용자가 '이벤트' 카테고리 탭을 클릭하면
THEN 이벤트 카테고리의 3건만 표시되고
AND 다른 카테고리의 게시글은 숨겨진다
```

- 관련 요구사항: REQ-CS-001
- 검증 방법: 단위 테스트 + E2E 테스트

#### AC-CS-003: 공지사항 상세 조회

```gherkin
GIVEN 공지사항 목록이 표시된 상태에서
WHEN 사용자가 특정 게시글 제목을 클릭하면
THEN 해당 게시글의 제목, 내용, 작성일, 조회수가 표시되고
AND 이전글/다음글 네비게이션이 제공된다
```

- 관련 요구사항: REQ-CS-002
- 검증 방법: E2E 테스트

#### AC-CS-004: FAQ 카테고리별 아코디언 표시

```gherkin
GIVEN FAQ가 6개 카테고리(주문방법/배송/결제/인쇄품질/파일업로드/기타)에 등록되어 있고
WHEN 사용자가 FAQ 페이지에 접근하면
THEN 전체 FAQ가 카테고리별로 그룹핑되어 표시되고
AND 각 FAQ 항목은 질문만 표시되고 답변은 접힌 상태이다
```

- 관련 요구사항: REQ-CS-005, REQ-CS-006
- 검증 방법: E2E 테스트

#### AC-CS-005: FAQ 아코디언 토글

```gherkin
GIVEN FAQ 목록이 표시된 상태에서
WHEN 사용자가 특정 질문을 클릭하면
THEN 해당 답변이 아코디언 방식으로 펼쳐지고
AND 이미 펼쳐진 다른 답변은 접힌다 (단일 열림 모드)
AND 키보드(Enter/Space)로도 동일하게 동작한다
```

- 관련 요구사항: REQ-CS-006
- 검증 방법: 단위 테스트 + 접근성 테스트

#### AC-CS-006: 비회원 주문조회 성공

```gherkin
GIVEN 비회원으로 주문한 이력이 있고
WHEN 주문번호, 이메일, 휴대폰번호를 정확히 입력하고 조회 버튼을 클릭하면
THEN 일치하는 주문 정보(주문 상태, 상품 목록, 배송 추적)가 표시된다
```

- 관련 요구사항: REQ-CS-008, REQ-CS-011
- 검증 방법: E2E 테스트

#### AC-CS-007: 비회원 주문조회 실패 및 Rate Limiting

```gherkin
GIVEN 비회원 주문조회 화면에서
WHEN 일치하지 않는 정보로 조회를 시도하면
THEN "일치하는 주문을 찾을 수 없습니다" 오류 메시지가 표시되고
AND 회원가입 유도 CTA가 제공되며
AND 동일 IP에서 10회 초과 시 "잠시 후 다시 시도해주세요" 메시지가 표시된다
```

- 관련 요구사항: REQ-CS-009, REQ-CS-010
- 검증 방법: 통합 테스트 + E2E 테스트

### 1.2 모듈 2: 관리자 게시판 관리

#### AC-CS-008: 공지사항 등록 (관리자)

```gherkin
GIVEN 관리자가 공지사항 관리 화면에 접근하고
WHEN 제목, 내용, 카테고리(공지)를 입력하고 등록 버튼을 클릭하면
THEN 공지사항이 저장되고 목록에 즉시 반영되며
AND 쇼핑몰 공지사항 페이지에서도 확인 가능하다
```

- 관련 요구사항: REQ-CS-020, REQ-CS-021
- 검증 방법: 통합 테스트

#### AC-CS-009: 공지사항 상단 고정 (관리자)

```gherkin
GIVEN 관리자가 공지사항 관리 화면에서
WHEN 특정 게시글의 '상단 고정' 토글을 활성화하면
THEN 해당 게시글이 쇼핑몰 공지사항 목록 최상단에 고정 표시된다
```

- 관련 요구사항: REQ-CS-003, REQ-CS-020
- 검증 방법: 통합 테스트

#### AC-CS-010: FAQ 순서 변경 (관리자)

```gherkin
GIVEN 관리자가 FAQ 관리 화면에서 FAQ 목록이 표시된 상태에서
WHEN 특정 FAQ 항목을 드래그앤드롭으로 순서를 변경하면
THEN 변경된 순서가 저장되고 쇼핑몰 FAQ 페이지에 즉시 반영된다
```

- 관련 요구사항: REQ-CS-023, REQ-CS-024
- 검증 방법: E2E 테스트

#### AC-CS-011: 1:1문의 답변 및 알림 발송

```gherkin
GIVEN 고객이 1:1문의를 작성한 상태에서
WHEN 관리자가 답변을 작성하고 등록하면
THEN 문의 상태가 '완료'로 변경되고
AND 고객에게 답변 알림(이메일/알림톡)이 발송된다
```

- 관련 요구사항: REQ-CS-025, REQ-CS-026
- 검증 방법: 통합 테스트 + 알림 발송 모의 테스트

#### AC-CS-012: 이용후기 삭제 시 적립금 자동 회수

```gherkin
GIVEN 고객이 일반 이용후기를 작성하고 적립금 1,000원이 지급된 상태에서
WHEN 관리자가 해당 이용후기를 삭제하면
THEN 이용후기가 삭제되고
AND 해당 고객의 적립금(프린팅머니)에서 1,000원이 자동 차감된다
```

- 관련 요구사항: REQ-CS-028, REQ-CS-030
- 검증 방법: 통합 테스트 (적립금 API 연동 확인)

#### AC-CS-013: 이용후기 관리자 임의등록

```gherkin
GIVEN 관리자가 이용후기 관리 화면에서 '후기 등록' 버튼을 클릭하고
WHEN 상품, 작성자명, 평점, 내용을 입력하고 등록하면
THEN 관리자 작성 후기가 등록되고 쇼핑몰에 노출된다
```

- 관련 요구사항: REQ-CS-029
- 검증 방법: 통합 테스트

### 1.3 모듈 3: 관리자 등록/관리

#### AC-CS-014: 관리자 계정 등록

```gherkin
GIVEN 대표관리자가 관리자 등록 화면에 접근하고
WHEN 이름, 이메일, 역할(운영자), 권한 그룹을 입력하고 등록하면
THEN 운영자 계정이 생성되고
AND 초기 비밀번호가 해당 이메일로 발송되며
AND 관리자 목록에 추가된다
```

- 관련 요구사항: REQ-CS-040, REQ-CS-041
- 검증 방법: 통합 테스트

#### AC-CS-015: 관리자 권한별 메뉴 접근 제어

```gherkin
GIVEN 부운영자 역할로 로그인한 상태에서
WHEN 사이드바 메뉴에 접근하면
THEN 부운영자 권한 그룹에 허용된 메뉴만 표시되고
AND 허용되지 않은 메뉴는 표시되지 않는다
```

- 관련 요구사항: REQ-CS-045
- 검증 방법: E2E 테스트

#### AC-CS-016: 대표관리자 보호

```gherkin
GIVEN 유일한 대표관리자 계정이 있을 때
WHEN 해당 계정을 삭제 또는 비활성화하려고 시도하면
THEN 시스템이 "유일한 대표관리자는 삭제/비활성화할 수 없습니다" 오류를 표시하고
AND 작업이 차단된다
```

- 관련 요구사항: REQ-CS-044
- 검증 방법: 단위 테스트 + 통합 테스트

### 1.4 체험단관리 (CUSTOM)

#### AC-CS-017: 체험단 모집 공고 등록

```gherkin
GIVEN 관리자가 체험단관리 화면에서 '공고 등록' 버튼을 클릭하고
WHEN 제목, 내용(HTML), 모집기간, 모집인원, 체험상품을 입력하고 등록하면
THEN 체험단 공고가 저장되고 상태가 '모집중'으로 설정되며
AND 쇼핑몰 체험단 페이지에 노출된다
```

- 관련 요구사항: REQ-CS-031, REQ-CS-032
- 검증 방법: E2E 테스트

#### AC-CS-018: 체험단 당첨 처리

```gherkin
GIVEN 모집 기간이 종료된 체험단에 신청자 20명이 있고
WHEN 관리자가 10명을 선택하고 '당첨 처리' 버튼을 클릭하면
THEN 선택된 10명의 상태가 '당첨'으로 변경되고
AND 당첨 알림이 발송되며
AND 나머지 10명은 '미당첨' 상태로 변경된다
```

- 관련 요구사항: REQ-CS-033
- 검증 방법: 통합 테스트

#### AC-CS-019: 체험단 중복 신청 차단

```gherkin
GIVEN 특정 체험단에 이미 신청한 회원이
WHEN 동일 체험단에 다시 신청하려고 시도하면
THEN "이미 신청한 체험단입니다" 오류 메시지가 표시되고
AND 중복 신청이 차단된다
```

- 관련 요구사항: REQ-CS-035
- 검증 방법: 단위 테스트 + 통합 테스트

## 2. 품질 게이트 기준

### 2.1 Definition of Done

| 기준 | 목표 |
|------|------|
| EARS 요구사항 커버리지 | 100% (모든 REQ에 대응하는 AC 존재) |
| E2E 테스트 통과 | 100% |
| 접근성 | FAQ 아코디언 키보드 네비게이션 통과 |
| 성능 | 게시판 목록 1초 이내, 비회원 조회 2초 이내 |
| 보안 | Rate Limiting 동작, XSS 방지 (체험단 HTML) |

### 2.2 검증 도구

| 도구 | 용도 |
|------|------|
| Playwright | E2E 테스트 |
| Vitest | 단위/통합 테스트 |
| axe-core | 접근성 검증 |
| Lighthouse | 성능 측정 |
