## 1. 로그인/회원가입

### 1.1 화면 구성

**로그인 화면 핵심 요소:**
- 상단: 로고/브랜드 아이덴티티 (중앙 정렬)
- 중앙: 이메일/아이디 입력 필드 + 비밀번호 필드 (세로 스택)
- 로그인 상태 유지 체크박스 (좌측 정렬)
- 아이디/비밀번호 찾기 링크 (우측 정렬, 작은 텍스트)
- 메인 CTA: "로그인" 버튼 (전체 너비, 브랜드 컬러)
- 구분선: "또는 SNS로 로그인" 텍스트 양옆에 구분선
- SNS 로그인 버튼 영역: 카카오(노란색) > 네이버(초록색) > 구글(흰색) 순서
- 하단: "아직 회원이 아니신가요?" + 회원가입 링크
- 비회원 주문조회 링크 (텍스트 링크, 덜 강조)

**SNS 로그인 버튼 배치 패턴 (한국 표준):**
- 카카오 로그인: 노란색(#FEE500), 검정 로고 아이콘, 가장 큰 사용률로 최상단 배치
- 네이버 로그인: 초록색(#03C75A), 흰색 텍스트, 두 번째 배치
- 구글 로그인: 흰색 배경, 회색 테두리, 멀티컬러 G 아이콘, 세 번째 배치
- 버튼 형태: 원형 아이콘 3개 나란히 배치 또는 전체 너비 버튼 3개 세로 스택
- 모바일에서는 아이콘+텍스트 조합, 데스크톱에서는 텍스트 포함 버튼

**회원가입 화면 핵심 요소:**
- 단계 표시: Step 1(기본정보) → Step 2(상세정보) → Step 3(완료) 인디케이터
- 필수 필드: 이메일, 비밀번호, 비밀번호 확인, 이름, 휴대폰 번호
- 본인인증: 휴대폰 SMS 인증 (인증번호 발송 버튼 + 타이머 카운트다운 표시)
- 약관 동의: 전체 동의 체크박스 + 개별 약관 항목 (필수/선택 뱃지 표시)
- 약관 내용 보기: 오른쪽 화살표 아이콘 → 모달 또는 아코디언으로 펼치기
- 회원등급 안내: 가입 직후 화면 또는 완료 페이지에 등급 혜택 간략 표시

### 1.2 인터랙션 패턴

- 폼 유효성: 실시간 인라인 검증 (필드 blur 시 에러 메시지 즉시 표시)
- 비밀번호 강도 표시: 입력 중 진행률 바 또는 아이콘으로 강도 피드백
- SNS 로그인 클릭 시: 팝업 창 또는 리다이렉트로 OAuth 처리
- 소셜 로그인 후 기존 계정 연동 필요 시: "이미 가입된 이메일입니다. 연동하시겠습니까?" 확인 다이얼로그

### 1.3 컴포넌트 매핑

- 레이아웃: `PageShell` (maxWidth: sm, 중앙 정렬 카드)
- 모바일: 전체 화면 폼
- 데스크톱: 중앙 550px 카드 (양옆 배경 이미지 또는 브랜드 영역 가능)
- 폼 요소: `FormLayout` (스택 방향, 레이블-인풋 세트)

### 1.4 벤치마크 참조

- 카카오커머스: 카카오 로그인 최상단, 간결한 최소형 UI
- 네이버 스마트스토어: 네이버 로그인 강조, 약관 아코디언 패턴
- 무신사: SNS 로그인 3개 아이콘 수평 나열, 이메일 로그인 아래 배치
- 29CM: 세련된 흰 배경, 미니멀 폼, 소셜 로그인 아이콘형

### 1.5 와이어프레임 텍스트 설명

## 2. 마이페이지

### 2.1 화면 구성

**마이페이지 메인 (대시보드형):**
- 상단 프로필 영역: 회원 이름, 등급 뱃지 (일반/실버/골드/VIP), 포인트 잔액, 쿠폰 수
- 등급 진행 표시: 다음 등급까지 필요 금액 프로그레스 바
- 빠른 메뉴 그리드: 주문내역, 관심상품, 쿠폰, 포인트, 배송지관리, 회원정보수정 (2x3 또는 3x2 아이콘 그리드)
- 최근 주문 섹션: 최근 3-5건 주문 카드형 표시 (상품 썸네일 + 주문상태 + 날짜)

**주문조회 목록:**
- 상단 탭: 전체 | 입금대기 | 결제완료 | 배송준비 | 배송중 | 배송완료 | 취소/반품/교환
- 기간 필터: 1개월 / 3개월 / 6개월 / 직접입력 (라디오 + 날짜 범위 피커)
- 주문 카드 (각 주문 단위):
  - 주문번호 + 주문일자 (한 줄)
  - 배송 상태 뱃지 (색상 코딩: 결제완료=파랑, 배송중=주황, 완료=초록)
  - 상품 대표 이미지 60x60 + 상품명 + 옵션 + 수량 + 금액
  - 같은 주문 내 상품이 2개 이상이면 "+N개 더보기" 접기
  - 하단 액션 버튼: "주문상세", "배송조회", "구매확정", "리뷰작성", "교환/반품"

**주문상세 화면:**
- 주문 상태 타임라인: 결제완료 → 상품준비중 → 배송중 → 배송완료 (스텝 아이콘)
- 배송 추적 링크 (택배사 + 운송장 번호)
- 주문 상품 목록 (상세 정보)
- 결제 정보 요약 (상품금액 + 배송비 + 할인금액 = 최종결제금액)
- 배송지 정보
- 증빙서류 발급 버튼 (현금영수증/세금계산서)

**쿠폰 관리:**
- 탭: 사용 가능 쿠폰 (N) | 사용 완료 | 기간 만료
- 쿠폰 카드: 할인액/할인율 크게 + 쿠폰명 + 사용 조건 + 유효기간
- 쿠폰 등록: 쿠폰 코드 입력 필드 + 등록 버튼 (상단 고정)
- 만료 임박 쿠폰 상단 강조 표시

**포인트(적립금) 내역:**
- 현재 보유 포인트 크게 표시
- 소멸 예정 포인트 경고 표시
- 입/출금 내역 리스트 (날짜 | 내용 | 적립/차감 | 잔액)
- 포인트 사용처 안내

### 2.2 인터랙션 패턴

- 주문 카드 탭: 확장/축소 아코디언 (여러 상품 있을 때)
- 배송 상태 뱃지 색상: 상태별 즉각적인 시각 피드백
- 쿠폰 카드 hover: 사용 가능 쿠폰 클릭 시 해당 상품 목록으로 이동 또는 쿠폰 복사
- 무한 스크롤 또는 페이지네이션 (주문 목록)
- 쿠팡 스타일: 원터치 재구매 버튼 (이전 주문 동일 상품 장바구니 담기)

### 2.3 컴포넌트 매핑

- 마이페이지 메인: `PageShell` + `ResponsiveGrid` (모바일 1컬럼, 데스크톱 2컬럼)
- 주문목록: `PageShell` + `SplitLayout` (사이드 필터 데스크톱에서 활성화)
- 쿠폰/포인트: `PageShell` + `FormLayout` (탭 포함)
- 회원정보수정: `PageShell` + `FormLayout` (수평 배치)

### 2.4 벤치마크 참조

- 쿠팡: 주문상태 타임라인, 재구매 버튼, 배송 추적 인라인 표시
- 무신사: 브랜드별 주문 그룹핑, 패션 특화 리뷰 사진 첨부
- 29CM: 감성적 카드 UI, 주문내역 타이포그래피 강조
- 카카오커머스: 카카오 계정 연동 프로필, 포인트 시각화

## 3. 주문/결제

### 3.1 화면 구성

**장바구니:**
- 선택 체크박스 (전체 선택 / 개별 선택)
- 상품 카드: 이미지 80x80 + 브랜드명 + 상품명 + 옵션명 + 수량 조절(+/-) + 금액
- 삭제 버튼 (X 아이콘, 우상단)
- 품절/옵션 변경 필요 상품 경고 표시
- 배송비 안내 (무료배송 기준 표시, 얼마 더 담으면 무료 안내)
- 쿠폰 즉시 적용 가능 개수 표시
- 최종 합계 영역 (상품금액 + 배송비 + 쿠폰할인 = 주문금액) - 하단 고정 또는 우측 sticky
- CTA: "주문하기" 버튼 (전체 너비, 브랜드 컬러)

**주문서 (체크아웃):**
- 섹션 구성:
  1. 배송지 정보 (기본 배송지 선택 + 새 배송지 추가 + 배송지 목록 드로어)
  2. 배송 옵션 (일반배송 / 새벽배송 / 방문수령 등 해당되는 경우)
  3. 주문 상품 목록 (축소형, 펼치기 가능)
  4. 쿠폰/포인트 적용 (드롭다운 쿠폰 선택 + 포인트 입력 + 최대 사용 버튼)
  5. 결제 수단 선택
  6. 최종 결제 금액 요약
  7. 약관 동의 (필수) + 주문 확인 버튼

- 배송지 목록 드로어 (우측에서 슬라이드):
  - 저장된 배송지 카드 리스트
  - 기본 배송지 라디오 선택
  - 새 배송지 추가 폼 (이름, 수령인, 주소 검색, 상세주소, 연락처, 배송메모)

**결제 수단 선택 UI:**
- 카드결제: 카드 선택 그리드 (BC, 신한, KB, 삼성, 현대 등 카드사 로고 버튼)
- 간편결제 (최상단 강조):
  - 카카오페이 (노란 배경)
  - 네이버페이 (초록 배경)
  - 토스페이 (파란 배경)
  - 삼성페이 (흰 배경)
- 실시간 계좌이체 (은행 선택 드롭다운)
- 무통장입금/가상계좌
- B2B 후불결제 (기업회원 전용, 별도 섹션)
- 선택된 결제수단 강조 하이라이트

### 3.2 인터랙션 패턴

- 배송지 변경 시 즉시 배송비 재계산 표시
- 쿠폰 선택 시 할인 금액 즉시 반영 (애니메이션 효과)
- 포인트 최대 사용 클릭 시 최대 사용 가능 금액 자동 입력
- 결제 버튼 클릭 시 로딩 스피너 → 결제창 (PG 팝업 또는 인앱 처리)
- 주문 완료 페이지: 주문번호 + "주문내역 확인" + "쇼핑 계속하기" CTA

### 3.3 핵심 UX 원칙

- 한국 이커머스 특성: 간편결제(카카오페이, 네이버페이) 최상단 배치
- 장바구니 포기율 70% 대응: 진행 단계 최소화 (최대 3단계)
- 배송비 조기 표시: 장바구니 단계부터 정확한 배송비 표시

## 7. 어드민 상품관리

### 7.1 화면 구성

**상품 목록:**
- 검색 필터: 카테고리 드롭다운 + 판매상태(판매중/중지) + 상품명/코드 검색
- 상품 테이블: 체크박스 | 이미지 | 상품명 | 카테고리 | 판매가 | 재고 | 판매상태 | 등록일 | 관리
- 일괄 처리: 판매상태 변경, 진열 여부 변경, 카테고리 이동, 삭제

**상품 등록 폼 (탭 또는 단계별 섹션):**

섹션 1. 기본정보:
- 상품명 (필수)
- 카테고리 선택 (계층형 드롭다운 또는 트리 팝업)
- 상품 요약 설명
- 상품 상세 설명 (WYSIWYG 에디터)
- 태그 입력 (쉼표 구분)

섹션 2. 판매정보:
- 정상가 / 판매가 (% 할인율 자동 계산)
- 과세 여부 (과세/비과세 라디오)
- 재고 관리 여부 (재고 직접 관리 / API 연동)
- 최소/최대 구매 수량

섹션 3. 옵션 설정:
- 옵션 없음 / 직접 입력 / 일괄 등록 탭
- 옵션 속성 추가: "색상", "사이즈" 등 속성명 + 값 (콤마 구분)
- 옵션 조합 테이블 자동 생성 (속성 조합 × 가격 + 재고)
- 인쇄물 특화: 수량 옵션 (10매/20매/50매/100매/직접입력)

섹션 4. 이미지:
- 대표 이미지 (필수): 드래그&드롭 영역 + 파일 선택 버튼
  - 업로드 후 크롭/회전 편집 가능
- 추가 이미지 (최대 10장): 섬네일 그리드, 드래그로 순서 변경
- 이미지 용량/크기 제한 안내

섹션 5. 배송정보:
- 배송방법 선택 (일반택배/퀵배송/직접배송)
- 배송비 설정 (무료/유료/조건부무료)
- 별도 배송비 상품 여부
- 배송 예정일 안내 텍스트 (선택)

섹션 6. 추가 설정:
- 구매 후기 표시 여부
- 상품 문의 허용 여부
- 판매 기간 설정 (선택)
- 성인 인증 상품 여부

하단 저장 버튼: "임시저장" + "저장 후 계속" + "저장 완료"

**카테고리 관리:**
- 트리 UI: 부모 카테고리 > 자식 카테고리 드래그&드롭 순서 변경
- 카테고리 추가/수정/삭제 인라인 에디팅
- 카테고리별 진열 설정

### 7.2 인터랙션 패턴

- 상품 등록 폼: 좌측 목차 (섹션 스크롤 네비게이션) + 우측 폼 콘텐츠
- 이미지 업로드: 진행 상태 바 표시
- 옵션 조합 테이블: 체크박스로 일부 옵션 비활성화 가능

  - 승인: "게시중"으로 상태 변경

## 12. 증빙서류 발급 관리

### 12.1 화면 구성

**증빙서류 발급 현황 테이블:**
- 컬럼: 주문번호 | 회원명 | 서류 유형 | 신청일 | 발급일 | 상태 | 관리
- 서류 유형: 현금영수증 | 세금계산서 | 거래명세서
- 상태 4단계 뱃지:
  - 신청: 회색 (접수됨)
  - 발급중: 파란 로딩 아이콘 (처리 중)
  - 발급완료: 초록 체크 (완료)
  - 오류: 빨간 X (재발급 필요)

**현금영수증 발급 처리:**
- 발급 타입: 소득공제용(개인) / 지출증빙용(사업자)
- 팝빌 API 연동 상태 표시
- 발급 처리 버튼 (신청 건 자동/수동 처리)
- 국세청 신고 결과 반영 (WebHook 수신 후 상태 업데이트)

**세금계산서 발급 처리:**
- 사업자 정보 확인 (사업자번호, 상호, 이메일)
- 공급가액 / 세액 / 합계 표시
- 팝빌 API로 전자 세금계산서 발행
- 발행 후 이메일 자동 전송 여부 체크박스

**고객 발급 신청 UI (마이페이지):**
- 주문 상세에서 "증빙서류 발급 신청" 버튼
- 서류 유형 선택 (라디오): 현금영수증 / 세금계산서
- 현금영수증: 휴대폰 번호 또는 사업자번호 입력
- 세금계산서: 사업자 정보 폼 (사업자번호, 상호, 대표자명, 업태, 종목, 이메일)
- 신청 후 처리 상태 조회 가능

### 12.2 상태 전이 다이어그램

```
신청 → 발급중 → 발급완료
              └→ 오류 → (재신청)
```

- 신청: 고객이 신청 또는 주문 완료 후 자동 신청
- 발급중: 팝빌 API 호출 완료, 국세청 처리 대기
- 발급완료: 국세청 처리 완료 + 고객에게 이메일 발송
- 오류: 잘못된 사업자번호 등으로 발급 실패 → 고객에게 오류 사유 안내

### 12.3 컴포넌트 매핑

- 관리 화면: `PageShell` + DataTable (상태 필터 탭 포함)
- 고객 신청 UI: `PageShell` + `FormLayout` (주문 상세 내 인라인)
- 상태 배지: 색상 코딩 Badge 컴포넌트

### 12.4 벤치마크 참조

- 팝빌(POPBiLL): 전자세금계산서 발급 상태 관리 UI 표준
- 국세청 홈택스: 현금영수증 조회 UI
- 쿠팡: 세금계산서 신청 및 상태 확인 패턴

## 부록: 공통 UI 패턴 원칙

### 한국 쇼핑몰 특화 원칙

1. **간편결제 우선**: 카카오페이, 네이버페이를 결제 UI 최상단에 배치
2. **SNS 로그인 순서**: 카카오 > 네이버 > 구글 (사용율 기반 순서)
3. **포인트/적립금**: "프린팅머니"와 같은 브랜드명 포인트 체계를 명확히 UI에 표시
4. **회원등급 가시성**: 마이페이지 상단에 등급과 다음 등급까지의 진행도 표시
5. **배송비 조기 노출**: 장바구니 단계부터 정확한 배송비 표시 (숨김 없음)
6. **주문 상태 색상**: 한국 표준 색상 코딩 준수 (빨강=문제/취소, 초록=완료, 파랑=처리중, 주황=배송중)

### 어드민 DataTable 설계 원칙

1. **고정 헤더**: 스크롤 시 컬럼 헤더 항상 표시
2. **체크박스 일괄 처리**: 여러 항목 선택 후 액션 버튼 활성화
3. **상태 탭 + 필터**: 상태별 탭 (빠른 필터) + 상세 검색 폼 조합
4. **행 hover 액션**: 마우스 오버 시 수정/삭제 버튼 표시 (공간 절약)
5. **페이지당 행 수**: 20/50/100 선택 가능
6. **우측 슬라이드 패널**: 상세 보기 시 페이지 이동 없이 패널 오픈
7. **제브라 스트라이프**: 대량 데이터 행 구분을 위한 홀짝 행 배경색 교차

### 색상 코딩 표준 (SPEC-LAYOUT-001 디자인 토큰 기반)

- 브랜드 컬러: #5538B6 (후니프린팅 보라)
- 주문/결제 상태:
  - 입금대기: `warning` (노랑)
  - 결제완료: `primary` (파랑)
  - 배송준비: `secondary` (회색)
  - 배송중: `warning-dark` (주황)
  - 배송완료: `success` (초록)
  - 취소/반품: `error` (빨강)
- 증빙서류 상태:
  - 신청: `gray`
  - 발급중: `blue`
  - 발급완료: `green`
  - 오류: `red`
