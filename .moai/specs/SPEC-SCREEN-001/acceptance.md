# SPEC-SCREEN-001: 수락 기준

## 1. 공통 품질 기준

### 1.1 반응형 검증 (전체 화면 공통)

**Given** 모든 쇼핑몰 화면이 구현되었을 때
**When** 다음 4개 뷰포트에서 렌더링하면
**Then** 레이아웃이 정상적으로 표시되어야 한다

| 뷰포트 | 해상도 | 검증 포인트 |
|--------|--------|-----------|
| Mobile | 375px | 단일 컬럼, 터치 타겟 44px+, 패딩 16px |
| Tablet | 768px | 2컬럼 그리드, 패딩 24px |
| Desktop | 1024px | SplitLayout 활성화, 패딩 32px |
| Wide Desktop | 1280px | PageShell maxWidth 적용, 중앙 정렬 |

### 1.2 성능 기준

**Given** 사용자가 페이지에 접근할 때
**When** 네트워크 3G Fast 조건에서 측정하면
**Then** 다음 성능 기준을 만족해야 한다

| 메트릭 | 기준 | 대상 |
|--------|------|------|
| 초기 로드 (LCP) | < 3초 | 전체 페이지 |
| 인터랙션 (FID) | < 100ms | 전체 페이지 |
| 레이아웃 이동 (CLS) | < 0.1 | 전체 페이지 |
| Configurator 가격 응답 | < 500ms | SCR-A10-PRINT-PRODUCT |
| 파일 업로드 검증 | < 2초 | SCR-A6-FILE-UPLOAD |
| DataTable 렌더링 | < 1초 (1000행) | 관리자 DataTable |

### 1.3 접근성 기준 (WCAG 2.1 AA)

**Given** 모든 화면이 구현되었을 때
**When** 접근성 검사를 수행하면
**Then** 다음 기준을 만족해야 한다

- 색상 대비: 최소 4.5:1 (일반 텍스트), 3:1 (큰 텍스트)
- 키보드 네비게이션: 모든 인터랙티브 요소 탭 접근 가능
- 스크린 리더: 주요 콘텐츠 영역 ARIA 라벨 제공
- 포커스 표시: 가시적 포커스 인디케이터 제공
- 폼 레이블: 모든 입력 필드에 라벨 연결

### 1.4 API 통합 기준

**Given** shopby API와 연동된 화면일 때
**When** API 호출이 실패하면
**Then** 다음 처리를 수행해야 한다

- HTTP 401: 토큰 자동 갱신 후 재요청 (`fetchHttpRequest()`)
- HTTP 4xx: 사용자 친화적 에러 메시지 표시
- HTTP 5xx: 재시도 옵션 제공 + 에러 리포트
- 네트워크 오류: 오프라인 상태 안내 + 재시도 버튼

## 2. Phase 1: 핵심 고객 여정

### 2.1 로그인/회원 (SCR-A1-*)

**Scenario: 일반 로그인 성공**
**Given** 가입된 회원이 로그인 페이지에 접근했을 때
**When** 유효한 이메일과 비밀번호를 입력하고 로그인 버튼을 클릭하면
**Then** JWT 토큰이 발급되고 메인 페이지로 리다이렉트된다
**And** 데스크톱에서 카드 스타일 레이아웃이 적용된다 (SPEC-LAYOUT-002)

**Scenario: 회원가입 완료**
**Given** 비회원이 회원가입 페이지에 접근했을 때
**When** 약관 동의, 필수 정보 입력, 휴대전화 인증을 완료하면
**Then** 회원이 생성되고 가입완료 안내가 표시된다
**And** FormLayout으로 모바일/데스크톱 폼 배치가 전환된다

### 2.2 메인/상품 (SCR-A10-*)

**Scenario: 메인 페이지 렌더링**
**Given** 사용자가 메인 페이지에 접근했을 때
**When** 페이지가 로드되면
**Then** HeroBanner, CategoryNav, ProductRecommendGrid가 표시된다
**And** 데스크톱에서 PageShell maxWidth="7xl" 풀와이드가 적용된다 (SPEC-LAYOUT-002)

**Scenario: 상품 목록 필터링**
**Given** 사용자가 상품 목록 페이지에 접근했을 때
**When** 카테고리 필터와 정렬 옵션을 변경하면
**Then** 상품 목록이 실시간으로 업데이트된다
**And** ResponsiveGrid가 모바일 2컬럼, 데스크톱 4컬럼으로 표시된다

**Scenario: 상품 상세 조회**
**Given** 사용자가 상품 상세 페이지에 접근했을 때
**When** 페이지가 로드되면
**Then** 상품 이미지, 옵션, 가격, 탭(상세/리뷰/Q&A)이 표시된다
**And** 데스크톱에서 SplitLayout (이미지 + 주문 정보)이 적용된다

### 2.3 주문 플로우 (SCR-A6-*)

**Scenario: 장바구니 추가 및 확인**
**Given** 사용자가 상품 옵션을 선택했을 때
**When** 장바구니에 추가하고 장바구니 페이지로 이동하면
**Then** 상품명, 옵션, 수량, 가격이 표시된다
**And** 데스크톱에서 SplitLayout (상품 목록 + 주문 요약)이 적용된다

**Scenario: 결제 완료**
**Given** 사용자가 배송 정보를 입력하고 결제 수단을 선택했을 때
**When** 결제를 완료하면
**Then** 주문 완료 페이지에서 주문 번호와 예상 배송일이 표시된다

## 3. Phase 2: 인쇄 특화 기능

### 3.1 출력상품 주문 (SCR-A10-PRINT-PRODUCT)

**Scenario: 단계형 옵션 선택 (Wizard)**
**Given** 사용자가 출력상품 페이지에 접근했을 때
**When** 규격 --> 도수 --> 용지 --> 코팅 --> 수량 순서로 옵션을 선택하면
**Then** StepIndicator가 현재 단계를 표시하고, 선택마다 가격이 실시간 업데이트된다
**And** 상위 옵션 변경 시 하위 옵션이 초기화되고 안내 문구가 표시된다

**Scenario: 실시간 가격 표시**
**Given** 사용자가 옵션을 변경할 때
**When** 수량 또는 옵션이 변경되면
**Then** RealTimePriceWidget이 500ms 이내에 가격을 업데이트한다
**And** QuantityPricingTable이 수량별 단가를 포함하여 표시한다

**Scenario: 용지 시각적 미리보기**
**Given** 사용자가 용지 선택 단계에 있을 때
**When** 용지 옵션을 탐색하면
**Then** PaperSampleCard가 용지/코팅의 시각적 미리보기를 제공한다

### 3.2 파일 업로드 (SCR-A6-FILE-UPLOAD)

**Scenario: 드래그앤드롭 파일 업로드**
**Given** 사용자가 파일 업로드 페이지에 접근했을 때
**When** PDF 파일을 드래그앤드롭 영역에 놓으면
**Then** 파일이 업로드되고 실시간 검증이 시작된다
**And** FileValidationAlert가 해상도/CMYK/블리드/서체 검증 결과를 표시한다

**Scenario: 파일 검증 실패**
**Given** 업로드된 파일이 인쇄 기준에 미달할 때
**When** PitStop 검증이 완료되면
**Then** 구체적인 오류 항목과 수정 가이드가 표시된다
**And** "재업로드" 버튼이 활성화된다

**Scenario: 하이브리드 진입점**
**Given** 사용자가 파일 업로드 단계에 있을 때
**When** "디자인 없음" 옵션을 선택하면
**Then** 템플릿 갤러리 또는 디자인 의뢰 페이지로 안내된다

### 3.3 옵션 보관함 (SCR-A3-OPTION-STORAGE)

**Scenario: 옵션+파일 세트 저장**
**Given** 사용자가 주문을 완료했을 때
**When** "보관함에 저장" 옵션을 선택하면
**Then** 상품 옵션 + 업로드 파일 세트가 보관함에 저장된다

**Scenario: 재주문**
**Given** 사용자가 옵션 보관함에 접근했을 때
**When** 저장된 항목의 "재주문" 버튼을 클릭하면
**Then** 동일 옵션+파일로 장바구니에 추가된다

### 3.4 프린팅머니 (SCR-A3-PRINTING-MONEY, SCR-A3-MONEY-CHARGE)

**Scenario: 잔액 확인**
**Given** 로그인한 사용자가 프린팅머니 페이지에 접근했을 때
**When** 페이지가 로드되면
**Then** AccountBalanceWidget에 현재 잔액이 표시되고 거래 내역이 조회된다

**Scenario: PG 충전**
**Given** 사용자가 충전 페이지에 접근했을 때
**When** 충전 금액을 선택하고 결제를 완료하면
**Then** 프린팅머니 잔액이 즉시 업데이트된다

### 3.5 주문 추적 (SCR-A3-ORDER-DETAIL)

**Scenario: 8단계 생산 타임라인**
**Given** 인쇄 주문의 상세 페이지에 접근했을 때
**When** 주문 상태가 "인쇄중"일 때
**Then** OrderTrackerTimeline이 8단계 중 현재 단계를 강조 표시한다
**And** 모바일에서 수직, 데스크톱에서 수평 타임라인으로 전환된다

**Scenario: 주문 보류 상태**
**Given** 파일 검수에서 오류가 발견되었을 때
**When** Job 상태가 STAT_900(오류)으로 변경되면
**Then** "주문 보류" 상태가 표시되고 재업로드 안내가 제공된다
**And** 알림톡이 고객에게 발송된다

## 4. Phase 3: 관리자 핵심

### 4.1 상품 등록 (SCR-B4-PRINT-REG)

**Scenario: 인쇄 상품 등록**
**Given** 관리자가 인쇄 상품 등록 페이지에 접근했을 때
**When** 기본 정보, 옵션 의존성, 가격 매트릭스를 설정하면
**Then** 상품이 등록되고 쇼핑몰에 노출된다

**Scenario: 마스터 데이터 관리**
**Given** 관리자가 사이즈/소재/용지 마스터에 접근했을 때
**When** CRUD 작업을 수행하면
**Then** DataTable에 실시간 반영되고 관련 상품에 적용된다

### 4.2 주문 관리 (SCR-B8-*)

**Scenario: 파일 확인 처리**
**Given** 관리자가 파일 확인 페이지에 접근했을 때
**When** PDF 미리보기를 확인하고 "확인완료"를 클릭하면
**Then** 주문 상태가 "제작접수"로 변경된다

**Scenario: 일괄 상태 변경**
**Given** 관리자가 주문 목록에서 다건을 선택했을 때
**When** BulkActionBar에서 상태 변경을 실행하면
**Then** 선택된 모든 주문의 상태가 일괄 변경된다

**Scenario: DataTable 검색/필터/페이징**
**Given** 관리자가 주문관리 DataTable에 접근했을 때
**When** 검색어 입력, 날짜 필터, 상태 필터를 적용하면
**Then** 결과가 1초 이내에 업데이트되고 페이징이 정상 작동한다

**When** 카테고리 탭을 선택하고 검색어를 입력하면

**When** 회사명/담당자/연락처/인쇄물종류/수량/파일을 입력하고 제출하면
**Then** 견적 요청이 등록되고 확인 메시지가 표시된다

**Given** 관리자 영역 화면 전체에 대해
**When** 렌더링할 때
**Then** 다음 기준을 만족해야 한다
