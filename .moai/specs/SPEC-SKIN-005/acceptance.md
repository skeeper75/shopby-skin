## 1. 테스트 시나리오

### 1.1 UI 마이그레이션 시나리오

#### Scenario 1: CSS 토큰 전환 검증

- **Given** 모든 관리자 UI 파일이 마이그레이션된 상태에서
- **When** `--po-*` CSS 변수로 검색하면
- **Then** 관리자 관련 파일에서 검색 결과가 0건이어야 한다

#### Scenario 2: Chip 주문 상태 뱃지

- **Given** 주문 목록 페이지에서
- **When** 다양한 상태의 주문이 표시되면
- **Then** Huni `Chip` 컴포넌트로 상태별 색상이 구분된 뱃지가 렌더링된다
- **And** 접수중은 brand, 제작중은 warning, 배송중은 success, 완료는 neutral 톤이다

#### Scenario 3: Dialog 주문 상세

- **Given** 주문 목록에서 주문을 클릭하면
- **When** 주문 상세 Dialog가 열리면
- **Then** Huni `Dialog` 컴포넌트가 lazyMount로 렌더링되고, 내부에 `Tabs`로 정보/파일/이력 탭이 표시된다

#### Scenario 4: TextField 검색

- **Given** 주문 관리 페이지에서
- **When** 검색 바에 주문번호를 입력하면
- **Then** Huni `TextField`(clearable)로 렌더링되고, 기존과 동일하게 검색 결과가 필터링된다

#### Scenario 5: Switch 설정 토글

- **Given** 관리자 설정 페이지에서
- **When** 알림 자동발송 토글을 클릭하면
- **Then** Huni `Switch` 컴포넌트가 토글되고, 설정이 변경된다

#### Scenario 6: Snackbar 액션 피드백

- **Given** 주문 상태를 "제작진행중"으로 변경하면
- **When** 변경이 완료되면
- **Then** Huni `Snackbar`로 "상태가 변경되었습니다" 메시지가 표시된다

#### Scenario 7: Pagination 주문 목록

- **Given** 주문 목록이 페이지 크기를 초과할 때
- **When** 다음 페이지 버튼을 클릭하면
- **Then** Huni `Pagination` 컴포넌트가 페이지를 전환하고, API에서 해당 페이지 데이터를 로드한다

#### Scenario 8: Skeleton 데이터 로딩

- **Given** 대시보드 페이지에 접속하면
- **When** API 응답 대기 중일 때
- **Then** StatCard 영역에 Huni `Skeleton` 컴포넌트가 표시된다

### 1.2 백엔드 연동 시나리오

#### Scenario 9: 관리자 로그인 (RBAC)

- **Given** 등록된 관리자 계정으로
- **When** 올바른 ID/비밀번호로 로그인하면
- **Then** JWT 토큰이 발급되고, 권한에 따른 메뉴가 표시된다
- **And** 슈퍼관리자는 모든 메뉴, 뷰어는 조회 전용 메뉴만 표시된다

#### Scenario 10: 관리자 로그인 실패 잠금

- **Given** 관리자가 로그인을 시도할 때
- **When** 5회 연속 실패하면
- **Then** 30분간 로그인이 잠기고, 잠금 안내 메시지가 표시된다

#### Scenario 11: 주문 검색 (API 연동)

- **Given** 주문관리 목록에서
- **When** 주문번호 "HN-001"로 검색하면
- **Then** 실제 API에서 해당 주문만 필터링되어 표시된다

#### Scenario 12: 파일검수 자동+수동

- **Given** 파일확인 페이지에서 주문 파일을 열면
- **When** 자동검수(해상도/색상모드)를 통과한 파일에 대해 수동검수를 진행하면
- **Then** "확인완료" 시 주문 상태가 "파일승인"으로 자동 변경되고, 작업지시서가 생성된다

#### Scenario 13: 재업로드 요청 (3회 제한)

- **Given** 파일검수에서 문제를 발견하고
- **When** "재업로드 요청"을 클릭하면
- **Then** 고객에게 알림톡+이메일이 발송되고, 재업로드 횟수가 카운트된다
- **And** 3회 초과 시 "1:1 문의로 안내" 옵션이 표시된다

#### Scenario 14: 일괄 상태변경

- **Given** 주문 목록에서 5건을 선택하고
- **When** "제작진행중"으로 일괄 변경하면
- **Then** 5건 모두 상태가 변경되고, 각 고객에게 제작중 알림톡(005)이 발송된다
- **And** 실패 건이 있으면 실패 목록이 Snackbar로 표시된다

#### Scenario 15: SMS/알림톡 발송

- **Given** SMS 발송 Dialog에서
- **When** 고객을 선택하고 템플릿 메시지를 작성하여 발송하면
- **Then** NHN 알림톡 API로 발송되고, 성공/실패 결과가 표시된다

#### Scenario 16: 후불결제 관리

- **Given** 후불결제 관리 페이지에서
- **When** B2B 고객의 미결제 주문에 대해 "결제완료" 처리하면
- **Then** 결제 상태가 업데이트되고, Snackbar로 완료 메시지가 표시된다

#### Scenario 17: 주문서 출력

- **Given** 주문 목록에서 주문을 선택하고
- **When** "주문서 출력" 버튼을 클릭하면
- **Then** A4 인쇄 최적화된 작업지시서가 새 창에 표시되고 인쇄 기능이 제공된다
