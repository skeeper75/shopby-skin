# SPEC-MYPAGE-001: 수용 기준

> A3-MYPAGE 마이페이지 도메인 인수 테스트 시나리오

## 1. 모듈별 수용 기준

### 1.1 모듈 1: 주문조회 (Order History)

#### AC-MYP-001: 주문 목록 조회 및 기간 필터링

```gherkin
GIVEN 로그인한 회원이 마이페이지에 접근했고
AND 최근 3개월 내 주문이 5건 존재할 때
WHEN "주문조회" 메뉴를 클릭하면
THEN 주문 목록이 최신순으로 표시되고
AND 각 주문에 주문번호, 상품명, 주문일, 금액, 상태가 표시되며
AND 기간 필터에서 "1개월"을 선택하면 해당 기간 내 주문만 표시된다
```

- 관련 요구사항: REQ-MYP-001, REQ-MYP-002, REQ-MYP-003
- 검증 방법: E2E 테스트 (Playwright)

#### AC-MYP-002: 주문 상세 조회

```gherkin
GIVEN 주문 목록에 주문이 있을 때
WHEN 특정 주문을 클릭하면
THEN 주문 상세 페이지에 상품명, 옵션, 수량, 결제금액, 배송정보가 표시되고
AND 주문 상태에 따른 액션 버튼(결제/취소/리뷰쓰기/재주문)이 표시된다
```

- 관련 요구사항: REQ-MYP-004, REQ-MYP-008
- 검증 방법: E2E 테스트

#### AC-MYP-003: 편집 미리보기 (CUSTOM)

```gherkin
GIVEN 인쇄 상품 주문이 있고 인쇄 파일이 업로드된 상태에서
WHEN "미리보기" 버튼을 클릭하면
THEN 업로드된 인쇄 파일의 썸네일 이미지가 모달로 표시되고
AND 이미지 확대/축소가 가능하다
```

- 관련 요구사항: REQ-MYP-005
- 검증 방법: 통합 테스트 + E2E 테스트

#### AC-MYP-004: 주문 목록 빈 상태

```gherkin
GIVEN 로그인한 회원이 마이페이지에 접근했고
AND 조회 기간 내 주문이 없을 때
WHEN "주문조회" 메뉴를 클릭하면
THEN "주문 내역이 없습니다" 안내 메시지가 표시되고
AND 상품 페이지로 이동하는 CTA 버튼이 제공된다
```

- 관련 요구사항: REQ-MYP-006
- 검증 방법: 단위 테스트 + E2E 테스트

#### AC-MYP-005: 재주문

```gherkin
GIVEN 배송완료된 주문이 있을 때
WHEN "재주문" 버튼을 클릭하면
THEN 동일한 상품과 옵션이 장바구니에 추가되고
AND 장바구니 이동 안내가 표시된다
```

- 관련 요구사항: REQ-MYP-007
- 검증 방법: E2E 테스트

### 1.2 모듈 2: 쿠폰관리 (Coupon)

#### AC-MYP-006: 쿠폰 목록 및 탭 구분

```gherkin
GIVEN 로그인한 회원이 사용 가능 쿠폰 2개, 사용완료 1개, 만료 1개를 보유할 때
WHEN "쿠폰관리" 메뉴를 클릭하면
THEN "사용가능(2)" 탭에 2개 쿠폰이 표시되고
AND 각 쿠폰에 이름, 할인금액, 최소주문금액, 유효기간이 표시되며
AND "사용완료" 탭과 "만료" 탭에 각각 해당 쿠폰이 표시된다
```

- 관련 요구사항: REQ-MYP-009, REQ-MYP-010
- 검증 방법: E2E 테스트

#### AC-MYP-007: 쿠폰 코드 등록

```gherkin
GIVEN 로그인한 회원이 쿠폰 관리 페이지에 있을 때
WHEN 유효한 쿠폰 코드를 입력하고 등록 버튼을 클릭하면
THEN "쿠폰이 등록되었습니다" 성공 메시지가 표시되고
AND 사용 가능 쿠폰 목록에 새 쿠폰이 추가된다

WHEN 유효하지 않은 쿠폰 코드를 입력하면
THEN "유효하지 않은 쿠폰 코드입니다" 오류 메시지가 표시된다
```

- 관련 요구사항: REQ-MYP-011, REQ-MYP-012
- 검증 방법: 단위 테스트 + E2E 테스트

### 1.3 모듈 3: 프린팅머니 (Printing Money)

#### AC-MYP-008: 프린팅머니 잔액 및 내역 조회

```gherkin
GIVEN 로그인한 회원의 프린팅머니 잔액이 15,000원이고
AND 적립 2건, 사용 1건, 충전 1건의 내역이 있을 때
WHEN "프린팅머니" 메뉴를 클릭하면
THEN 상단에 잔액 "15,000원"이 표시되고
AND 내역 목록에 유형(적립/사용/충전), 금액, 일시, 주문번호가 날짜순으로 표시된다
```

- 관련 요구사항: REQ-MYP-014, REQ-MYP-015, REQ-MYP-016
- 검증 방법: E2E 테스트

#### AC-MYP-009: 프린팅머니 충전 (CUSTOM)

```gherkin
GIVEN 로그인한 회원이 프린팅머니 충전 페이지에 있을 때
WHEN 30,000원 충전을 선택하고 신용카드로 결제를 완료하면
THEN PG 결제가 정상 처리되고
AND shopby 적립금이 30,000원 증가하고
AND 프린팅머니 잔액이 갱신되며
AND 충전 내역에 기록이 추가된다
```

- 관련 요구사항: REQ-MYP-017
- 검증 방법: 통합 테스트 (PG Mock) + E2E 테스트

#### AC-MYP-010: 최소 충전 금액 제한

```gherkin
GIVEN 프린팅머니 충전 화면에서
WHEN 5,000원을 충전 금액으로 입력하면
THEN "최소 충전 금액은 10,000원입니다" 안내가 표시되고
AND 결제 버튼이 비활성화된다
```

- 관련 요구사항: REQ-MYP-018
- 검증 방법: 단위 테스트

### 1.4 모듈 4: 리뷰 (Review)

#### AC-MYP-011: 텍스트 리뷰 작성 + 적립금 자동지급

```gherkin
GIVEN 배송완료된 주문 상품이 있고 리뷰를 작성하지 않은 상태에서
WHEN 텍스트 리뷰(내용 + 평점)를 작성하고 등록하면
THEN 리뷰가 즉시 등록되어 상품 페이지에 노출되고
AND 프린팅머니 1,000원이 자동 적립되며
AND "리뷰 작성 완료! 1,000원이 적립되었습니다" 메시지가 표시된다
```

- 관련 요구사항: REQ-MYP-023, REQ-MYP-022
- 검증 방법: 통합 테스트 + E2E 테스트

#### AC-MYP-012: 포토 리뷰 작성 + 적립금 자동지급

```gherkin
GIVEN 배송완료된 주문 상품이 있을 때
WHEN 사진 3장을 첨부하여 리뷰를 작성하고 등록하면
THEN 리뷰가 사진과 함께 즉시 등록되고
AND 프린팅머니 2,000원이 자동 적립되며
AND "포토리뷰 작성 완료! 2,000원이 적립되었습니다" 메시지가 표시된다
```

- 관련 요구사항: REQ-MYP-024, REQ-MYP-028
- 검증 방법: 통합 테스트 + E2E 테스트

#### AC-MYP-013: 리뷰 삭제 + 적립금 자동 회수

```gherkin
GIVEN 텍스트 리뷰를 작성하여 1,000원이 적립된 상태에서
WHEN 리뷰 삭제 버튼을 클릭하면
THEN "리뷰 삭제 시 적립금 1,000원이 회수됩니다. 삭제하시겠습니까?" 확인 다이얼로그가 표시되고
AND 확인을 클릭하면 리뷰가 삭제되고 프린팅머니에서 1,000원이 차감된다
```

- 관련 요구사항: REQ-MYP-026, REQ-MYP-027
- 검증 방법: 통합 테스트 + E2E 테스트

### 1.5 모듈 5: 증빙서류 (Document Issuance)

#### AC-MYP-014: 세금계산서 발급 요청

```gherkin
GIVEN 결제 완료된 주문이 있고 사업자정보가 등록된 상태에서
WHEN 세금계산서 발급을 요청하면
THEN 시스템이 팝빌 API를 통해 세금계산서를 발행하고
AND 발급 상태가 "발급요청"에서 "발급완료"로 변경되며
```

- 관련 요구사항: REQ-MYP-030, REQ-MYP-032
- 검증 방법: 통합 테스트 (팝빌 Sandbox) + E2E 테스트

#### AC-MYP-015: 증빙서류 PDF 다운로드

```gherkin
GIVEN 발급완료 상태의 세금계산서가 있을 때
WHEN "다운로드" 버튼을 클릭하면
THEN 해당 세금계산서의 PDF 파일이 다운로드된다
```

- 관련 요구사항: REQ-MYP-033
- 검증 방법: E2E 테스트

```gherkin
THEN 정보가 저장되고 이후 현금/계좌이체 결제 시 자동 발급에 활용된다
```

### 1.7 모듈 7: 1:1문의 (Inquiry)

#### AC-MYP-018: 1:1문의 작성 및 조회


### 1.8 모듈 8: 회원계정 (Account)
