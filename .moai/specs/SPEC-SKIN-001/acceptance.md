# SPEC-SKIN-001: 인증/회원가입 - 인수 조건

> **SPEC ID**: SPEC-SKIN-001
> **문서 유형**: 인수 조건 (acceptance.md)

## 1. 테스트 시나리오 (Given-When-Then)

### Scenario 1: 정상 로그인
- **Given** 가입된 회원이 로그인 페이지에 접속했을 때
- **When** 올바른 이메일과 비밀번호를 입력하고 "로그인하기"를 클릭하면
- **Then** JWT 토큰이 저장되고 메인 페이지로 이동한다

### Scenario 2: 로그인 실패
- **Given** 로그인 페이지에서
- **When** 잘못된 비밀번호를 입력하면
- **Then** "이메일 또는 비밀번호를 확인해주세요" 에러가 표시된다

### Scenario 3: 회원가입 정상 플로우
- **Given** 비회원이 회원가입 페이지에 접속했을 때
- **When** 약관동의 -> 정보입력(이메일 중복확인, 휴대전화 인증 완료) -> 가입하기를 수행하면
- **Then** 가입이 완료되고 완료 페이지가 표시된다

### Scenario 4: 이메일 중복
- **Given** 회원가입 정보입력 단계에서
- **When** 이미 가입된 이메일로 중복확인을 하면
- **Then** "이미 가입된 이메일입니다" 메시지가 빨간색으로 표시된다

### Scenario 5: 휴대전화 인증 타임아웃
- **Given** 인증번호가 발송된 상태에서
- **When** 3분이 경과하면
- **Then** 타이머가 0:00이 되고 재발송 안내가 표시된다

### Scenario 6: 아이디 찾기
- **Given** 아이디 찾기 페이지에서
- **When** 가입된 휴대전화 번호를 입력하면
- **Then** 마스킹된 이메일과 가입일이 표시된다

### Scenario 7: 비밀번호 찾기
- **Given** 비밀번호 찾기 페이지에서
- **When** 이메일과 휴대전화를 입력하고 발송을 클릭하면
- **Then** 임시 비밀번호가 알림톡으로 발송된다
