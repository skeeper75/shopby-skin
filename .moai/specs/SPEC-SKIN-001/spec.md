# SPEC-SKIN-001: 인증/회원가입

> **SPEC ID**: SPEC-SKIN-001
> **상태**: Draft
> **작성일**: 2026-03-14
> **우선순위**: High
> **Phase**: 1
> **IA 항목**: No. 1~7
> **기술 스택**: React 18 + @shopby/react-components + shadcn/ui + Tailwind CSS

## 1. 개요

사용자 인증 및 회원가입 기능 일체를 구현한다. 일반 로그인, 아이디 찾기(이메일), 비밀번호 찾기(알림톡), 회원가입(약관동의 + 정보입력 + 본인인증), 가입완료 메일 발송까지 포함한다. Aurora Skin의 SignIn, SignUp, FindId, FindPassword 페이지를 기반으로 확장하되, Huni 디자인 시스템을 적용한다.

## 6. 인수 조건 (EARS Format)

### REQ-SKIN-001-001: 일반 로그인

**WHEN** 사용자가 이메일과 비밀번호를 입력하고 "로그인하기" 버튼을 클릭하면
**THE SYSTEM SHALL** Shopby 인증 API로 로그인을 수행하고, JWT 토큰을 저장하며, 이전 페이지 또는 메인으로 리다이렉트한다

**수용 기준**:
- [ ] 이메일/비밀번호 빈값 시 "입력해주세요" 안내 표시
- [ ] 로그인 실패 시 "이메일 또는 비밀번호를 확인해주세요" 메시지 표시
- [ ] 로그인 성공 시 JWT 토큰 localStorage 저장
- [ ] 로그인 후 이전 페이지로 복귀 (없으면 메인)
- [ ] 입력 필드 포커스 시 보라색(#5538B6) 테두리 표시

### REQ-SKIN-001-002: 이메일 유효성 검사

시스템은 **항상** 이메일 입력 필드에 대해 실시간 포맷 유효성 검사를 수행해야 한다

**수용 기준**:
- [ ] 이메일 형식이 아닌 경우 "올바른 이메일 형식을 입력해주세요" 표시
- [ ] debounce 300ms 적용
- [ ] 유효한 이메일: 녹색 체크 아이콘 표시

### REQ-SKIN-001-003: 아이디 찾기

**WHEN** 사용자가 휴대전화 번호를 입력하고 "아이디 찾기" 버튼을 클릭하면
**THE SYSTEM SHALL** 가입된 이메일을 마스킹하여 표시한다

**수용 기준**:
- [ ] 미가입 번호: "가입 정보가 없습니다" 메시지 표시
- [ ] 가입된 경우: 마스킹된 이메일 + 가입일 표시
- [ ] "로그인하기" 버튼으로 로그인 페이지 이동

### REQ-SKIN-001-004: 비밀번호 찾기

**WHEN** 사용자가 이메일과 휴대전화를 입력하고 "임시 비밀번호 발송" 버튼을 클릭하면
**THE SYSTEM SHALL** 임시 비밀번호를 생성하여 알림톡으로 발송한다

**수용 기준**:
- [ ] 정보 불일치 시 "정보가 일치하지 않습니다" 표시
- [ ] 발송 성공 시 안내 메시지 표시
- [ ] 임시 비밀번호 로그인 후 비밀번호 변경 유도

### REQ-SKIN-001-005: 회원가입 - 약관동의

**WHEN** 사용자가 필수 약관에 모두 동의하면
**THE SYSTEM SHALL** "다음" 버튼을 활성화하고, 정보입력 스텝으로 전환한다

**수용 기준**:
- [ ] "전체 동의" 체크 시 모든 약관 자동 체크
- [ ] 개별 해제 시 "전체 동의" 자동 해제
- [ ] 필수 미동의 시 "다음" 비활성화
- [ ] 각 약관 "보기" 클릭 시 전문 표시 (바텀시트)

### REQ-SKIN-001-006: 이메일 중복확인

**WHEN** 사용자가 이메일을 입력하고 "중복확인" 버튼을 클릭하면
**THE SYSTEM SHALL** Shopby API로 중복 여부를 확인하여 결과를 표시한다

**수용 기준**:
- [ ] 사용 가능: 녹색 메시지
- [ ] 이미 가입: 빨간색 메시지
- [ ] 중복확인 미완료 시 가입 버튼 비활성화

### REQ-SKIN-001-007: 휴대전화 인증

**WHEN** 사용자가 휴대전화 번호를 입력하고 "인증요청" 버튼을 클릭하면
**THE SYSTEM SHALL** SMS 인증번호를 발송하고, 3분 타이머를 시작한다

**수용 기준**:
- [ ] 인증번호 6자리 발송
- [ ] 3분(180초) 카운트다운 타이머 표시
- [ ] 타임아웃 시 재요청 안내
- [ ] 인증 성공 시 녹색 완료 메시지

### REQ-SKIN-001-008: 회원가입 완료

**WHEN** 모든 필수 정보를 입력하고 "가입하기" 버튼을 클릭하면
**THE SYSTEM SHALL** 회원가입 API를 호출하고, 가입완료 페이지로 이동한다

**수용 기준**:
- [ ] 성공 시 환영 메시지 표시
- [ ] 가입완료 이메일 자동 발송
- [ ] 비밀번호: 영문+숫자+특수문자 8자 이상

### REQ-SKIN-001-009: 비밀번호 보기/숨기기

시스템은 **항상** 비밀번호 입력 필드에 보기/숨기기 토글을 제공해야 한다

**수용 기준**:
- [ ] 기본: 마스킹, 눈 아이콘 클릭 시 평문
- [ ] 재클릭 시 다시 마스킹

### 8.1 소셜 로그인 우선 배치

Kakao/Naver/Apple 소셜 로그인 버튼을 이메일 로그인 위에 우선 배치한다 (Toss 접근 방식). 경쟁사 모두 소셜 로그인을 메인으로 제공하며, 이메일 로그인은 하위 옵션으로 배치한다.

**변경 레이아웃**:

│                              │
│  ┌──────────────────────┐   │

### 8.2 휴대폰 번호 로그인 옵션 추가

이메일 로그인 외에 휴대폰 번호 + OTP 로그인 옵션을 추가한다. redprinting이 이 방식을 우선시하며, 인쇄업 고객 특성상 이메일보다 전화번호 기반 인증이 편리한 경우가 많다.

- 로그인 폼 상단에 [이메일 | 휴대폰] 탭 전환 제공
- 휴대폰 탭 선택 시: 전화번호 입력 -> OTP 발송 -> 인증 후 로그인

### 8.3 자동 로그인 체크박스

로그인 폼 하단에 "자동 로그인" 체크박스를 추가한다 (wowpress 패턴).
- 모바일: 기본 체크됨
- 데스크톱: 기본 해제

### 8.4 회원가입 단계 최소화

회원가입 플로우를 최대 3단계로 축소한다:
1. **본인인증** (약관동의 + 휴대폰 인증을 통합)
2. **정보입력** (이메일, 비밀번호, 이름)
3. **완료**

기존 약관동의와 정보입력을 분리했던 구조에서, 약관동의를 본인인증 단계 내 인라인으로 통합하여 사용자 이탈을 줄인다.

### 8.5 비밀번호 strength 인디케이터

비밀번호 입력 필드 하단에 시각적 강도 표시 프로그레스 바를 추가한다:
- **빨강** (약함): 8자 미만 또는 단일 종류 문자
- **주황** (보통): 영문+숫자 조합 8자 이상
- **초록** (강함): 영문+숫자+특수문자 조합 10자 이상

### 8.6 신규 컴포넌트: HuniSocialLogin

소셜 로그인 전용 컴포넌트:
- 3개 버튼: Kakao | Naver | Apple (각 100% 너비, 50px 높이)
- 각 버튼 아이콘 + 텍스트 조합 (예: "카카오로 계속하기")
- 구분자: "또는 이메일로 계속하기" + 좌우 `#CACACA` 수평선
- 위치: 로그인 폼 상단, 이메일/비밀번호 필드 이전
- border-radius: 5px (Huni OrderCTA 사양 동일)
