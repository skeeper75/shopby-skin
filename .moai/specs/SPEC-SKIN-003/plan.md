# SPEC-SKIN-003: 상품/주문/결제 - 구현 계획

> **SPEC ID**: SPEC-SKIN-003
> **문서 유형**: 구현 계획 (plan.md)

## 1. 마일스톤

### Primary Goal: 출력상품 주문 플로우
- 인쇄 옵션 컨피규레이터 (사이즈/용지/코팅/수량/후가공)
- 가격 시뮬레이터 (실시간 가격 계산)
- 장바구니 + 주문서 작성
- 이니시스 PG 결제 연동
- 주문 완료

### Secondary Goal: 확장 상품 + 간편결제
- 포장재/굿즈 주문 UI
- 간편결제 3종 (네이버/카카오/토스)
- 견적서 인쇄 팝업
- 파일 업로드 (인쇄 파일)

### Optional Goal: 부가 기능
- 수작 브랜드 페이지
- 수동카드결제 (오프라인)
- 디자인 의뢰하기

## 2. 기술적 접근
- 인쇄 옵션 컨피규레이터: React 상태 관리 + 옵션 조합 유효성 검사
- 가격 계산: 서버사이드 API 호출 (debounce 300ms)
- PG 연동: 이니시스 JavaScript SDK 브라우저 연동
- 파일 업로드: Presigned URL 방식 대용량 처리

## 3. 리스크

| 리스크 | 영향도 | 대응 |
|--------|--------|------|
| 인쇄 옵션 조합 복잡도 | High | 옵션 의존성 매트릭스 사전 설계 |
| PG SDK 연동 | High | 이니시스 테스트 환경 사전 구축 |
| 간편결제 계약 | Medium | 기본 PG 우선, 간편결제 후순위 |
| 도서산간 DB | Low | 우체국 도서산간 목록 활용 |
