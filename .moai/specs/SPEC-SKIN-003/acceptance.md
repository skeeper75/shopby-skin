# SPEC-SKIN-003 v2 인수 조건

> **SPEC**: SPEC-SKIN-003
> **버전**: 2.0.0
> **작성일**: 2026-03-17

## 1. 종속옵션 엔진 (TAG-001)

### AC-001: 옵션 트리 렌더링

**Given** 사용자가 출력상품(명함) 상세 페이지에 진입했을 때
**When** 페이지가 로딩되면
**Then** 8단계 종속옵션(용지종류, 용지두께, 사이즈, 수량, 인쇄면수, 후가공, 코팅, 납기일)이 순서대로 표시된다
**And** 각 옵션은 RadioGroup 또는 Checkbox Huni DS 컴포넌트로 렌더링된다

### AC-002: 종속 필터링

**Given** 사용자가 용지종류로 "아트지"를 선택했을 때
**When** 용지두께 옵션이 갱신되면
**Then** 아트지에 가능한 두께(150g, 200g, 250g, 300g)만 표시된다
**And** 불가능한 옵션은 `data-disabled` 상태로 표시된다

### AC-003: 상품별 옵션 범위

**Given** 사용자가 포스터 상품 상세에 진입했을 때
**When** 옵션이 렌더링되면
**Then** 후가공 옵션은 표시되지 않는다 (포스터는 후가공 미지원)

### AC-004: 후가공 다중선택

**Given** 사용자가 전단지 상품의 후가공 단계에 있을 때
**When** "박" 과 "형압"을 동시에 선택하면
**Then** 두 옵션 모두 `data-checked` 상태가 되고 Checkbox 컴포넌트로 표시된다
**And** 가격 계산에 두 후가공 추가금이 모두 반영된다

## 2. 가격 매트릭스 계산 (TAG-002)

### AC-005: 실시간 가격 갱신

**Given** 사용자가 종속옵션을 모두 선택한 상태일 때
**When** 수량을 100에서 500으로 변경하면
**Then** HuniPriceCalculator에 새로운 가격이 0.3초 이내에 표시된다
**And** Skeleton이 로딩 중에 표시된다

### AC-006: 가격 상세 내역

**Given** 가격이 계산된 상태일 때
**When** 가격 상세를 확인하면
**Then** 기본단가, 후가공 추가금, 코팅 추가금, 급행할증(해당 시)이 각각 표시된다
**And** 최종 가격 = 기본단가 + 후가공 + 코팅 + 급행(30%) - 할인 이 정확하다

### AC-007: 급행 할증

**Given** 사용자가 납기일을 "급행(1~2영업일)"으로 선택했을 때
**When** 가격이 재계산되면
**Then** 기본단가 대비 30% 급행 할증이 적용된다

### AC-008: 견적 문의 전환

**Given** 사용자가 수량을 2,000매 초과로 입력했을 때
**When** 가격 조회가 실행되면
**Then** "대량 주문 견적 문의" 안내가 표시되고 가격은 표시되지 않는다

### AC-009: 가격 보안

**Given** 가격 계산 요청이 발생했을 때
**When** 네트워크 요청을 검사하면
**Then** 가격 매트릭스 원본 데이터는 응답에 포함되지 않고, 계산된 결과만 반환된다

## 3. 옵션 저장/견적서 (TAG-003)

### AC-010: 옵션 저장

**Given** 로그인한 사용자가 옵션을 선택한 상태일 때
**When** "옵션저장" 버튼을 클릭하면
**Then** 옵션 조합이 보관함에 저장되고 Snackbar "옵션이 저장되었습니다"가 표시된다

### AC-011: 비로그인 옵션 저장

**Given** 비로그인 사용자가 "옵션저장" 버튼을 클릭했을 때
**When** 저장을 시도하면
**Then** 로그인 유도 Dialog가 표시된다

### AC-012: 견적서 팝업

**Given** 사용자가 옵션을 선택한 상태일 때
**When** "견적서" 버튼을 클릭하면
**Then** Dialog에 상품명, 옵션, 수량, 단가, 합계가 포함된 견적서가 표시된다
**And** 인쇄(window.print) 및 PDF 다운로드 버튼이 제공된다

## 4. 상품 목록/필터 (TAG-004)

### AC-013: 카테고리 탭

**Given** 사용자가 상품 목록 페이지에 진입했을 때
**When** 카테고리 탭이 렌더링되면
**Then** Tabs 컴포넌트로 수평 스크롤 탭이 표시된다
**And** 활성 탭은 `--huni-fg-brand` 색상 + 2px 하단 보더를 갖는다

### AC-014: 상품 그리드 반응형

**Given** 상품 목록이 표시된 상태일 때
**When** 뷰포트 너비가 768px 미만이면
**Then** 2컬럼 그리드로 표시된다
**When** 뷰포트 너비가 1024px 이상이면
**Then** 4컬럼 그리드 + 필터 사이드바로 표시된다

### AC-015: 스켈레톤 로딩

**Given** 상품 목록이 로딩 중일 때
**When** 데이터가 아직 도착하지 않으면
**Then** HuniSkeletonCard가 그리드 위치에 표시된다

## 5. 장바구니 (TAG-005)

### AC-016: 장바구니 추가 확인

**Given** 사용자가 옵션을 선택한 상태에서 "장바구니" 버튼을 클릭했을 때
**When** 상품이 장바구니에 추가되면
**Then** Dialog "장바구니로 이동하시겠습니까?"가 표시된다
**And** Snackbar "장바구니에 추가되었습니다"가 표시된다

### AC-017: 인쇄옵션 표시

**Given** 장바구니에 인쇄 상품이 있을 때
**When** 장바구니 페이지를 확인하면
**Then** 각 상품의 인쇄 옵션(용지, 사이즈, 수량, 후가공, 코팅)이 Chip 형태로 표시된다

### AC-018: GNB 뱃지 업데이트

**Given** 사용자가 상품을 장바구니에 추가했을 때
**When** 추가가 성공하면
**Then** GNB 장바구니 아이콘의 뱃지 수량이 즉시 업데이트된다

## 6. 파일 업로드 (TAG-006)

### AC-019: 드래그앤드롭 업로드

**Given** 사용자가 주문서 파일 업로드 영역에 있을 때
**When** PDF 파일을 드래그하여 영역에 드롭하면
**Then** S3 Presigned URL이 발급되고 브라우저에서 직접 업로드가 시작된다
**And** 업로드 진행률 바가 표시된다

### AC-020: 포맷 검증

**Given** 사용자가 파일을 선택했을 때
**When** 파일 확장자가 PDF/AI/PSD가 아니면
**Then** "허용되지 않는 파일 형식입니다" 오류 메시지가 표시된다
**And** 업로드가 진행되지 않는다

### AC-021: 크기 제한

**Given** 사용자가 파일을 선택했을 때
**When** 파일 크기가 100MB를 초과하면
**Then** "파일 크기가 100MB를 초과합니다" 오류 메시지가 표시된다

### AC-022: 나중에 보내기

**Given** 사용자가 "나중에 보내기"를 체크했을 때
**When** 주문을 진행하면
**Then** 파일 없이 주문이 완료된다
**And** 주문 상태는 "파일미접수"로 시작된다

### AC-023: 파일 미리보기

**Given** 사용자가 파일 업로드를 완료했을 때
**When** 미리보기 버튼을 클릭하면
**Then** Dialog에서 파일 미리보기가 표시된다

## 7. 파일 검수/재업로드 (TAG-007)

### AC-024: 자동 검증 분류

**Given** PDF 파일이 업로드 완료되었을 때
**When** 자동 검증이 수행되면
**Then** 결과가 PASS(즉시 제작대기), WARNING(CS 확인), FAIL(재업로드 요청) 중 하나로 분류된다

### AC-025: 재업로드 요청 발송

**Given** 자동 검증 결과가 FAIL일 때
**When** 결과가 확정되면
**Then** 알림톡(003번)이 반려 사유 + 재업로드 링크와 함께 자동 발송된다

### AC-026: 재업로드 횟수 제한

**Given** 고객이 재업로드를 3회 수행했을 때
**When** 4번째 재업로드를 시도하면
**Then** "재업로드 횟수가 초과되었습니다. 고객센터에 문의해주세요" 안내가 표시된다

### AC-027: 재업로드 기한

**Given** 재업로드 요청이 발송된 후 5일이 경과했을 때
**When** 고객이 파일을 재업로드하지 않으면
**Then** 주문이 자동 보류 처리된다
**And** 기한 1일 전 리마인더 알림톡이 발송된다

## 8. 배송정보 입력 (TAG-008)

### AC-028: 도서산간 배송비

**Given** 사용자가 배송지 주소를 입력했을 때
**When** 주소가 도서산간 지역이면
**Then** 추가 배송비(3,000~5,000원)가 자동으로 적용되고 결제 금액에 반영된다

### AC-029: 배송지 저장

**Given** 사용자가 배송정보를 입력한 후 저장했을 때
**When** 저장이 완료되면
**Then** 최근 10개까지 배송지가 저장되고 다음 주문에서 선택 가능하다

## 9. 결제 (TAG-009)

### AC-030: 이니시스 카드 결제

**Given** 사용자가 결제수단으로 "신용카드"를 선택했을 때

### AC-031: 간편결제 (네이버페이)

**Given** 사용자가 결제수단으로 "네이버페이"를 선택했을 때
**When** "결제하기" 버튼을 클릭하면
**Then** 네이버페이 SDK 결제 화면이 열린다
**And** 결제 성공 시 주문완료 페이지로 이동한다

### AC-032: 간편결제 (카카오페이)

**Given** 사용자가 결제수단으로 "카카오페이"를 선택했을 때
**When** "결제하기" 버튼을 클릭하면
**Then** 카카오페이 SDK 결제 화면이 열린다

### AC-033: 간편결제 (토스페이)

**Given** 사용자가 결제수단으로 "토스페이"를 선택했을 때
**When** "결제하기" 버튼을 클릭하면
**Then** 토스페이 SDK 결제 화면이 열린다

### AC-034: 프린팅머니 전액결제

**Given** 사용자가 프린팅머니 잔액이 결제금액 이상일 때
**When** 프린팅머니 전액 결제를 선택하면
**Then** PG를 경유하지 않고 프린팅머니에서 차감되어 결제가 완료된다

### AC-035: 복합결제

**Given** 사용자가 프린팅머니 5,000원을 사용하고 잔여금액을 카드 결제로 선택했을 때
**When** "결제하기"를 클릭하면
**Then** 프린팅머니 5,000원이 차감되고 나머지 금액으로 이니시스 결제가 진행된다

### AC-036: 쿠폰 할인

**Given** 사용자가 쿠폰을 적용했을 때
**When** 쿠폰 할인이 적용되면
**Then** 총 결제금액에서 쿠폰 할인액이 차감되어 표시된다

### AC-037: 결제 실패 처리

**Given** 결제 진행 중 오류가 발생했을 때
**When** PG에서 실패 응답이 반환되면
**Then** Snackbar로 에러 메시지가 표시되고 재시도 가능하다

### AC-038: 가상계좌 입금기한

**Given** 사용자가 무통장입금(가상계좌)으로 결제했을 때
**When** 48시간 이내에 입금이 되지 않으면
**Then** 주문이 자동 취소된다
**And** 알림톡(011번)이 발송된다

## 10. 주문완료 (TAG-010)

### AC-039: 주문완료 페이지

### AC-040: 주문 알림톡

**Given** 주문이 완료되었을 때
**When** 주문 처리가 완료되면
**Then** 알림톡(002번 결제완료)이 고객에게 발송된다

**Given** 비회원 사용자가 주문을 진행할 때

## 12. 주문 상태 관리 (TAG-012)

### AC-043: 고객 상태 표시

**Given** 주문이 내부적으로 "인쇄중" 상태일 때
**When** 고객이 주문 상세를 확인하면
**Then** "제작중" Chip이 표시된다 (11단계 -> 9단계 매핑)

### AC-044: 취소 가능 범위

**Given** 주문이 "파일확인중" 상태일 때

**Then** 취소가 가능하다

**Given** 주문이 "파일승인" 이후 상태일 때
**When** 고객이 주문 취소를 요청하면
**Then** "제작이 착수되어 취소가 불가합니다" 안내가 표시된다

### AC-045: 상태 변경 알림

**Given** 주문 상태가 변경될 때
**When** 주요 상태(결제완료, 파일승인, 출고완료 등)로 전환되면
**Then** 해당 알림톡이 자동 발송된다

## 13. Huni DS 컴포넌트 검증

### AC-046: `--po-*` 변수 부재

**Given** SPEC-SKIN-003 범위의 모든 파일을 검사할 때
**When** CSS 변수를 검색하면
**Then** `--po-*` 패턴이 0건이다
**And** 모든 디자인 토큰이 `--huni-*` 패턴을 사용한다

### AC-047: 직접 HTML 요소 부재

**Given** SPEC-SKIN-003 범위의 모든 파일을 검사할 때
**When** 폼 요소를 검색하면

**And** 커스텀 모달이 0건이다 (Dialog 사용)
**And** alert/window.confirm 사용이 0건이다

### AC-048: data-* 상태 패턴

**When** 상태가 변경되면
**Then** `data-checked`, `data-disabled`, `data-focus-visible` 등 data-* 속성이 올바르게 적용된다
