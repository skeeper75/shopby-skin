## 1. 개요

인쇄 전문 쇼핑몰의 핵심인 상품 탐색, 인쇄 옵션 설정, 장바구니, 주문서 작성, 결제, 주문완료까지의 전체 구매 플로우를 구현한다. 출력상품 4종(디지털/책자/실사/패키지), 포장재, 굿즈, 수작 상품의 주문 프로세스를 포함하며, 인쇄 옵션 컨피규레이터가 핵심 차별화 요소이다.

## 6. 인수 조건 (EARS Format)

### REQ-SKIN-003-001: 인쇄 옵션 컨피규레이터

**WHEN** 사용자가 출력상품 상세 페이지에 접속하면
**THE SYSTEM SHALL** 사이즈, 용지, 인쇄면, 코팅, 수량, 후가공 옵션을 표시하고, 옵션 변경 시 실시간으로 예상 가격을 계산하여 표시한다

**수용 기준**:
- [ ] 옵션 칩(Chip) 형태로 선택 가능
- [ ] 옵션 변경 시 0.3초 내 가격 갱신
- [ ] 선택 불가 조합: 해당 칩 비활성화 + 툴팁
- [ ] "커스텀" 사이즈 선택 시 가로x세로 직접 입력

### REQ-SKIN-003-002: 옵션 저장

**WHEN** 사용자가 옵션을 선택한 상태에서 "옵션저장" 버튼을 클릭하면
**THE SYSTEM SHALL** 현재 옵션 조합을 옵션보관함에 저장한다

**수용 기준**:
- [ ] 로그인 사용자만 저장 가능
- [ ] 비로그인 시 로그인 유도
- [ ] 저장 성공 시 토스트 알림 "옵션이 저장되었습니다"

### REQ-SKIN-003-003: 견적서 인쇄

**WHEN** 사용자가 "견적서" 버튼을 클릭하면
**THE SYSTEM SHALL** 현재 옵션 조합의 견적서를 팝업으로 표시하고 인쇄 기능을 제공한다

**수용 기준**:
- [ ] 견적서에 상품명, 옵션, 수량, 단가, 합계 포함
- [ ] 인쇄 버튼 (window.print)
- [ ] PDF 다운로드 버튼

### REQ-SKIN-003-004: 장바구니

**WHEN** 사용자가 "장바구니" 버튼을 클릭하면
**THE SYSTEM SHALL** 선택한 옵션과 함께 상품을 장바구니에 추가하고, 이동 여부를 확인한다

**수용 기준**:
- [ ] "장바구니로 이동하시겠습니까?" 확인 다이얼로그
- [ ] 장바구니에서 인쇄 옵션 확인 가능
- [ ] GNB 장바구니 뱃지 수량 업데이트

### REQ-SKIN-003-005: 파일 업로드

**WHEN** 주문서 작성 시 파일 업로드 영역에 파일을 첨부하면
**THE SYSTEM SHALL** 파일 포맷(PDF/AI/PSD)과 크기(100MB)를 검증하고 업로드한다

**수용 기준**:
- [ ] 드래그 앤 드롭 지원
- [ ] 허용 포맷: PDF, AI, PSD, JPG, PNG
- [ ] 최대 100MB
- [ ] 업로드 진행률 표시
- [ ] "나중에 보내기" 체크 시 파일 없이 주문 진행

### REQ-SKIN-003-006: 배송비 계산

시스템은 **항상** 배송지 입력 시 도서산간 지역 여부를 판별하고, 추가 배송비를 자동으로 적용해야 한다

**수용 기준**:
- [ ] 기본 배송비: 3,000원 (조건부 무료)
- [ ] 도서산간 추가: 3,000~5,000원
- [ ] 주소 입력 즉시 배송비 갱신

### REQ-SKIN-003-007: 결제

**WHEN** 사용자가 결제 수단을 선택하고 "결제하기" 버튼을 클릭하면
**THE SYSTEM SHALL** 선택한 PG(이니시스/간편결제)를 통해 결제를 진행하고, 완료 시 주문완료 페이지로 이동한다

**수용 기준**:
- [ ] 이니시스 SDK 연동 (신용카드, 계좌이체, 가상계좌)
- [ ] 간편결제: 네이버페이/카카오페이/토스페이
- [ ] 프린팅머니 전액 결제 지원
- [ ] 쿠폰 + 프린팅머니 복합 할인 적용
- [ ] 결제 실패 시 에러 메시지 + 재시도

### REQ-SKIN-003-008: 주문 완료

**WHEN** 결제가 성공하면
**THE SYSTEM SHALL** 주문완료 페이지에 주문번호, 결제금액, 예상 제작일을 표시하고 주문확인 메일을 발송한다

**수용 기준**:
- [ ] 주문번호 표시
- [ ] "주문 상세보기" / "쇼핑 계속하기" 버튼
- [ ] 주문확인 이메일 자동 발송
