# SPEC-MEMBER-001: 수용 기준

> A1A2-MEMBER 로그인/회원 도메인 인수 테스트 시나리오

---

## 1. 모듈별 수용 기준

### 1.1 모듈 1: 인증 (Authentication)

#### AC-MEMBER-001: 정상 이메일 로그인

```gherkin
GIVEN 가입된 이메일 계정이 있고
WHEN 올바른 이메일과 비밀번호로 로그인 시도하면
THEN accessToken과 refreshToken이 발급되고
AND 메인 페이지로 이동하며
AND 헤더에 로그인 상태(회원 이름)가 표시된다
```

- 관련 요구사항: REQ-MEMBER-001, REQ-MEMBER-002, REQ-MEMBER-005
- 검증 방법: E2E 테스트 (Playwright)

#### AC-MEMBER-002: 로그인 5회 실패 계정 잠금

```gherkin
GIVEN 가입된 계정이 있고
WHEN 잘못된 비밀번호로 5회 연속 로그인 시도하면
THEN 계정이 일시적으로 잠금 처리되고
AND "로그인 시도 횟수를 초과했습니다. 30분 후 다시 시도해주세요" 메시지가 표시되고
AND 잠금 해제 후 올바른 비밀번호로 정상 로그인이 가능하다
```

- 관련 요구사항: REQ-MEMBER-007, REQ-MEMBER-011
- 검증 방법: 통합 테스트 + E2E 테스트

#### AC-MEMBER-003: accessToken 자동 갱신

```gherkin
GIVEN 로그인 상태에서 accessToken이 만료되었고
AND 유효한 refreshToken이 존재할 때
WHEN API 요청이 401 응답을 받으면
THEN 시스템은 refreshToken으로 새 accessToken을 발급받고
AND 원래 API 요청을 자동으로 재시도하여 성공한다
```

- 관련 요구사항: REQ-MEMBER-009
- 검증 방법: 단위 테스트 (fetchHttpRequest 401 핸들링)

#### AC-MEMBER-004: 카카오 최초 SNS 로그인

```gherkin
GIVEN 카카오 계정이 있고 후니프린팅 계정이 없을 때
WHEN 카카오 로그인 버튼을 클릭하고 카카오 인증을 완료하면
THEN 추가 정보 입력 화면이 표시되고 (이름, 휴대전화)
AND 정보 입력 완료 후 자동으로 계정이 생성되고
AND 메인 페이지로 이동한다
```

- 관련 요구사항: REQ-MEMBER-025, REQ-MEMBER-026, REQ-MEMBER-027
- 검증 방법: E2E 테스트 (OAuth Mock)

#### AC-MEMBER-005: 비밀번호 재설정 링크 만료

```gherkin
GIVEN 비밀번호 찾기 요청으로 재설정 링크가 발송된 상태에서
WHEN 24시간 경과 후 재설정 링크를 클릭하면
THEN "링크가 만료되었습니다" 오류가 표시되고
AND 재설정 이메일 재발송 버튼이 제공된다
```

- 관련 요구사항: REQ-MEMBER-023
- 검증 방법: 통합 테스트

#### AC-MEMBER-006: 로그인 실패 시 계정 열거 방지

```gherkin
GIVEN 존재하지 않는 이메일로 로그인 시도할 때
WHEN 잘못된 이메일과 비밀번호를 입력하면
THEN "이메일 또는 비밀번호가 일치하지 않습니다" 메시지가 표시되고
AND 이메일 존재 여부를 구별할 수 있는 정보가 노출되지 않는다
```

- 관련 요구사항: REQ-MEMBER-006, REQ-MEMBER-013
- 검증 방법: 단위 테스트 + 보안 검증

### 1.2 모듈 2: 회원가입 (Registration)

#### AC-MEMBER-007: 정상 회원가입 전체 플로우

```gherkin
GIVEN 비회원 상태에서
WHEN 이용약관 및 개인정보처리방침에 동의하고
AND 이메일, 비밀번호, 이름, 휴대전화를 입력하고
AND 이메일 중복 확인이 "사용 가능"이고
AND 휴대전화 SMS 인증을 완료하고
AND "가입하기" 버튼을 클릭하면
THEN 회원 계정이 생성되고
AND 가입 완료 이메일이 발송되고
AND 신규 가입 쿠폰(10,000원)이 발급되고
AND 가입 완료 페이지가 표시된다
```

- 관련 요구사항: REQ-MEMBER-032~061
- 검증 방법: E2E 테스트 (전체 플로우)

#### AC-MEMBER-008: 이메일 중복 가입 차단

```gherkin
GIVEN 이미 이메일로 가입된 계정이 있을 때
WHEN 동일 이메일을 입력하고 300ms가 경과하면
THEN "이미 사용 중인 이메일입니다" 오류가 빨간색 X 아이콘과 함께 표시되고
AND "가입하기" 버튼이 비활성화 상태를 유지한다
```

- 관련 요구사항: REQ-MEMBER-043, REQ-MEMBER-046, REQ-MEMBER-050
- 검증 방법: 단위 테스트 (debounce + API mock)

#### AC-MEMBER-009: 필수 약관 미동의 시 다음 단계 차단

```gherkin
GIVEN 약관동의 화면에서
WHEN 필수 약관(이용약관, 개인정보처리방침) 중 하나라도 미동의한 상태에서
THEN "다음" 버튼이 비활성화 상태이고
AND 필수 약관 전체 동의 후에만 "다음" 버튼이 활성화된다
```

- 관련 요구사항: REQ-MEMBER-037, REQ-MEMBER-038
- 검증 방법: 단위 테스트 (상태 관리)

#### AC-MEMBER-010: SMS 인증 타이머 만료

```gherkin
GIVEN 인증번호 발송 후 3분이 경과한 상태에서
WHEN 만료된 인증번호를 입력하면
THEN "인증번호가 만료되었습니다" 오류가 표시되고
AND "인증번호 재발송" 버튼이 활성화된다
AND 만료된 인증번호로 인증이 허용되지 않는다
```

- 관련 요구사항: REQ-MEMBER-056, REQ-MEMBER-058
- 검증 방법: 단위 테스트 (타이머 mock)

#### AC-MEMBER-011: 전체 동의 체크박스 동작

```gherkin
GIVEN 약관동의 화면에서 모든 체크박스가 해제된 상태일 때
WHEN "전체 동의" 체크박스를 클릭하면
THEN 이용약관, 개인정보처리방침, 마케팅 수신동의, 14세 이상 확인이 모두 선택되고
AND "전체 동의"를 다시 클릭하면 모든 항목이 해제된다
```

- 관련 요구사항: REQ-MEMBER-034, REQ-MEMBER-035
- 검증 방법: 단위 테스트 (체크박스 상태)

### 1.3 모듈 3: 회원관리 (Member Management)

#### AC-MEMBER-012: 비회원 인쇄물 주문 차단

```gherkin
GIVEN 비회원이 인쇄물 상품을 장바구니에 담고 주문을 시도할 때
WHEN 주문 버튼을 클릭하면
THEN "인쇄물 주문은 회원만 가능합니다" 안내가 표시되고
AND 로그인/가입 유도 CTA 버튼이 표시되고
AND "가입 시 10,000원 쿠폰 제공" 혜택 안내가 함께 표시된다
```

- 관련 요구사항: REQ-MEMBER-065
- 검증 방법: E2E 테스트

#### AC-MEMBER-013: 비회원 굿즈 주문 허용

```gherkin
GIVEN 비회원이 단순 굿즈 상품을 주문하려 할 때
WHEN 주문 절차를 진행하면
THEN 이름, 휴대전화, 이메일 입력 화면이 제공되고
AND 주문 완료 후 주문번호가 이메일 및 SMS로 발송된다
```

- 관련 요구사항: REQ-MEMBER-064, REQ-MEMBER-066
- 검증 방법: E2E 테스트

#### AC-MEMBER-014: 회원 등급 상승 알림

```gherkin
GIVEN 실버 등급 회원이 3개월 내 구매액이 30만원을 초과했을 때
WHEN shopby 등급 산정 배치가 실행되면
THEN 해당 회원의 등급이 골드로 변경되고
AND 등급 상승 알림 이메일이 발송되고
AND 마이페이지에서 변경된 등급 배지와 골드 혜택이 표시된다
```

- 관련 요구사항: REQ-MEMBER-070~074
- 검증 방법: 통합 테스트 (shopby 배치 연동 확인)

#### AC-MEMBER-015: 비회원 주문 조회

```gherkin
GIVEN 비회원이 주문을 완료한 상태에서
WHEN 주문번호와 휴대전화번호를 입력하고 주문 조회를 하면
THEN 해당 주문의 상태, 배송 정보, 상품 내역이 표시된다
```

- 관련 요구사항: REQ-MEMBER-063
- 검증 방법: E2E 테스트

### 1.4 모듈 4: 관리자 회원운영 (Admin Operations)

#### AC-MEMBER-016: 신규 가입 쿠폰 자동 발급

```gherkin
GIVEN 새로운 회원이 가입을 완료했을 때
WHEN 가입 완료 이벤트가 발생하면
THEN 10,000원 쿠폰이 자동으로 발급되고 (최소 주문금액 50,000원)
AND 쿠폰 유효기간이 30일로 설정되고
AND 가입 완료 이메일에 쿠폰 정보가 포함된다
```

- 관련 요구사항: REQ-MEMBER-098
- 검증 방법: 통합 테스트 (shopby 쿠폰 API 연동)

#### AC-MEMBER-017: 미완료 주문 회원 탈퇴 거부

```gherkin
GIVEN 배송 중인 주문이 있는 회원이 탈퇴를 요청했을 때
WHEN 탈퇴 요청을 제출하면
THEN "진행 중인 주문 N건이 있어 탈퇴가 불가합니다" 오류가 표시되고
AND 주문 조회 링크가 제공되고
AND 탈퇴가 실행되지 않는다
```

- 관련 요구사항: REQ-MEMBER-083
- 검증 방법: 단위 테스트 + E2E 테스트

#### AC-MEMBER-018: 관리자 프린팅머니 일괄 지급

```gherkin
GIVEN 관리자가 VIP 등급 회원 전체에게 프린팅머니 지급을 실행했을 때
WHEN 일괄 지급을 확정하면
THEN VIP 등급 회원 전체에게 지정된 금액이 지급되고
AND 각 회원에게 지급 알림이 발송되고
AND 관리자 화면에 지급 완료 내역(대상 수, 총 금액)이 기록된다
```

- 관련 요구사항: REQ-MEMBER-090
- 검증 방법: 통합 테스트 (관리자 API)

#### AC-MEMBER-019: 탈퇴 시 잔여 혜택 소멸

```gherkin
GIVEN 잔여 쿠폰 3건과 프린팅머니 5,000원이 있는 회원이
WHEN 탈퇴를 완료하면
THEN 잔여 쿠폰 3건이 즉시 소멸되고
AND 프린팅머니 5,000원이 즉시 소멸되고
AND 탈퇴 완료 화면에서 소멸된 혜택 내역이 안내된다
```

- 관련 요구사항: REQ-MEMBER-084, REQ-MEMBER-091
- 검증 방법: 통합 테스트

## 2. 통합 수용 기준

### AC-MEMBER-020: 로그인 -> 마이페이지 -> 등급 확인 통합 플로우

```gherkin
GIVEN 골드 등급 회원이 있고
WHEN 이메일과 비밀번호로 로그인하고
AND 마이페이지를 방문하면
THEN 헤더에 회원 이름이 표시되고
AND 마이페이지 상단에 골드 등급 배지가 표시되고
AND "VIP까지 N원 남았습니다" 프로그레스 바가 표시된다
```

- 관련 요구사항: REQ-MEMBER-005, REQ-MEMBER-070, REQ-MEMBER-073
- 검증 방법: E2E 테스트 (통합 시나리오)

### AC-MEMBER-021: 회원가입 -> 로그인 -> 쿠폰 확인 통합 플로우

```gherkin
GIVEN 비회원 상태에서 회원가입을 완료했을 때
WHEN 가입 완료 후 로그인하고
AND 마이페이지의 쿠폰 목록을 확인하면
THEN 신규 가입 쿠폰(10,000원, 유효기간 30일)이 발급되어 있다
```

- 관련 요구사항: REQ-MEMBER-045, REQ-MEMBER-098
- 검증 방법: E2E 테스트 (통합 시나리오)

### AC-MEMBER-022: SNS 기존 이메일 연동 플로우

```gherkin
GIVEN 일반 이메일 계정으로 이미 가입된 상태에서
WHEN 동일 이메일의 카카오 계정으로 로그인 시도하면
THEN "이미 가입된 이메일입니다. 기존 계정과 연동할까요?" 안내가 표시되고
AND 연동 승인 시 기존 계정에 카카오 로그인이 연동된다
```

- 관련 요구사항: REQ-MEMBER-028
- 검증 방법: E2E 테스트 (OAuth Mock)

## 3. 비기능 수용 기준

### 3.1 성능

| 항목 | 기준 | 검증 방법 |
|------|------|----------|
| 로그인 응답시간 | P95 기준 2초 이내 | Lighthouse / API 응답시간 측정 |
| 이메일 중복확인 | P95 기준 500ms 이내 (debounce 300ms 포함) | 네트워크 탭 모니터링 |
| SMS 발송 | 발송 요청 후 3초 이내 수신 | 실제 기기 테스트 |
| 회원가입 완료 | P95 기준 3초 이내 | API 응답시간 측정 |
| 동시 접속 | 최소 500 동시 사용자 지원 | shopby Enterprise 기본 지원 확인 |

### 3.2 보안

| 항목 | 기준 | 검증 방법 |
|------|------|----------|
| 비밀번호 평문 전달 | URL, 로그, 스토리지에 평문 노출 없음 | 코드 리뷰 + 보안 스캔 |
| XSS 방어 | 사용자 입력값 이스케이핑 적용 | OWASP ZAP 스캔 |
| CSRF 방어 | JWT Bearer Token 방식으로 CSRF 표면 최소화 | 보안 검증 |
| Rate Limiting | 로그인 5회/5분, SMS 1분 간격 | 자동화 테스트 |
| 개인정보 마스킹 | 이메일, 휴대전화 UI 표시 시 마스킹 | UI 검증 |
| 계정 열거 방지 | 로그인 실패 메시지에서 이메일 존재 여부 구분 불가 | 보안 검증 |

### 3.3 접근성

| 항목 | 기준 | 검증 방법 |
|------|------|----------|
| WCAG 2.1 AA | 모든 인증/가입 페이지 AA 준수 | axe-core 자동 검사 |
| 키보드 탐색 | Tab 키로 모든 입력 필드 순차 이동 가능 | 수동 테스트 |
