### 3. 상태 변이

| 상태 | 조건 | UI 변경 |
|------|------|---------|
| idle | 초기 진입 | 폼 입력 대기, 조회 버튼 활성 |
| loading | 조회 버튼 클릭 후 | 버튼 내 스피너, 폼 비활성화 |
| success | 주문 조회 성공 | 주문 상세 페이지로 이동 (또는 인라인 결과 표시) |
| empty | 일치 주문 없음 | 폼 하단 "일치하는 주문을 찾을 수 없습니다" 에러 메시지 (빨간색 텍스트) |
| error | API 오류 | 토스트: "주문 조회에 실패했습니다. 다시 시도해주세요" |
| validation_error | 필드 미입력 | 해당 필드 하단 인라인 에러 "필수 입력 항목입니다" |

### 4. 반응형 노트 + 접근성

**반응형**: 모바일(< 768px)에서 카드 패딩 24px로 축소, 풀 너비 적용
**접근성**:
- 각 입력 필드에 `aria-label` 및 `aria-required="true"` 적용
- 에러 메시지에 `aria-live="polite"` 적용
- 조회 버튼 로딩 시 `aria-busy="true"`, `aria-label="주문 조회 중"`
- Tab 순서: 주문번호 -> 휴대전화 -> 조회 버튼 -> 회원가입 링크
- 포커스 링: 2px solid #5538B6, offset: 2px
