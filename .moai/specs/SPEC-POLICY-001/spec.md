# SPEC-POLICY-001: 후니프린팅 사이트 리뉴얼 정책수립 도구

## 개요

후니프린팅 사이트 리뉴얼을 위한 정책수립 지원 도구(스킬 + 에이전트) 제작.
shopby Enterprise 플랫폼 기반으로 94개 IA 기능의 정책을 도메인별로 수립하고,
각 기능이 shopby 네이티브/스킨커스텀/커스텀개발 중 어디에 해당하는지 즉시 판별 가능하도록 지원.

## 배경

- 후니프린팅 사이트: 인쇄 전문 쇼핑몰 리뉴얼 프로젝트
- 플랫폼: shopby Enterprise + Aurora React Skin
- IA 규모: 쇼핑몰 43개 + 관리자 51개 = 총 94개 기능
- 핵심 과제: 어떤 기능이 shopby에서 되는지/안되는지 빠르게 판별하여 정책 결정

## 요구사항 (EARS 형식)

### REQ-1: 정책 자문 스킬 (innojini-shopby-policy-advisor)
WHEN 사용자가 특정 IA 기능의 정책에 대해 질의하면,
THE SYSTEM SHALL shopby 네이티브/스킨커스텀/커스텀개발/외부연동 4단계로 분류하여
해당 기능의 구현 가능 여부, 업계 벤치마크, 정책 결정 체크리스트를 제공한다.

### REQ-2: 정책수립 에이전트 (expert-printing-policy)
WHEN 사용자가 도메인 단위의 종합 정책 수립을 요청하면,
THE SYSTEM SHALL 13개 정책 도메인별로 구조화된 정책안을 생성하며,
shopby 참조문서와 업계 벤치마크를 근거로 한 구체적 추천안을 포함한다.

### REQ-3: IA 기능 매핑
THE SYSTEM SHALL 94개 IA 기능 각각에 대해 다음을 포함하는 매핑 데이터를 유지한다:
- 기능번호, 시스템(쇼핑몰/관리자), 영역, 기능명
- 분류 (NATIVE/SKIN/CUSTOM/EXTERNAL)
- shopby 관련 기능 (관리자 메뉴 경로 또는 API 엔드포인트)
- 정책 결정 필요사항

### REQ-4: 업계 벤치마크 데이터
THE SYSTEM SHALL 국내 주요 인쇄사이트 8곳의 정책 벤치마크를 포함하며,
회원등급, 배송, 취소/반품, B2B, 리뷰/체험단 등 핵심 정책 영역을 커버한다.

## 사용 방법

### 개별 기능 정책 질의
```
"프린팅머니 정책을 어떻게 수립하면 좋을까요?"
→ innojini-shopby-policy-advisor 스킬이 활성화되어
  shopby 적립금 시스템 활용 방안 + 충전 기능 커스텀 개발 안내 + 업계 벤치마크 제공
```

### 도메인 단위 정책 수립
```
"회원 정책 전체를 수립해주세요"
→ expert-printing-policy 에이전트가 회원 도메인의
  가입/등급/그룹/B2B/탈퇴 등 전체 정책안을 구조화하여 생성
```

### 빠른 기능 분류 확인
```
"파일 업로드는 shopby에서 되나요?"
→ CUSTOM 분류 + shopby에 파일 업로드 시스템 없음 안내 + 추천 아키텍처 제시
```

## 정책 결정 우선순위 (Top 5)

1. **인쇄 옵션 종속 구조 설계** - 전체 아키텍처에 영향 (CUSTOM)
2. **편집기 솔루션 선정** - 자체개발 vs 외부 솔루션 (CUSTOM)
3. **인쇄 주문 상태 정의** - shopby 주문상태 확장 방식 (SKIN)
4. **가격 매트릭스 구조** - 옵션 조합별 가격 관리 방식 (CUSTOM)
5. **B2B 후불결제 정책** - 기업고객 결제/정산 방식 (CUSTOM)
