## 1. 프로젝트 배경

### 1.1 후니프린팅 리뉴얼의 목적

후니프린팅(buysangsang.com)은 온라인 인쇄 전문 업체로, 기존 자체 구축 사이트에서 **NHN Commerce shopby Enterprise** 플랫폼으로 이전하는 리뉴얼 프로젝트를 진행 중이다. 리뉴얼의 핵심 목표는 다음과 같다:

- **플랫폼 안정성 확보**: shopby Enterprise의 검증된 이커머스 인프라 활용
- **인쇄 특화 기능 강화**: 종속 옵션 엔진, 파일 업로드/검수, 가격 매트릭스 등 인쇄 전용 모듈 자체 개발
- **운영 효율화**: shopby 관리자 기능을 활용한 주문/회원/상품 관리 체계 구축
- **사용자 경험 개선**: 경쟁사(레드프린팅, 와우프레스 등) 수준의 UI/UX 품질 달성

### 1.2 왜 기획서가 화면설계 전에 필요한가

SPEC-SCREEN-001(88개 화면 마스터 플랜)이 이미 존재하지만, 화면설계만으로는 **운영 정책, 비즈니스 로직, 기능 요구사항**이 100% 정의되지 않는다. 기획서 없이 화면설계를 진행하면 다음 문제가 발생한다:

1. **미결정 사항 40건**: 88개 기능 중 40개가 key_decision이 존재하나 모두 미결정 상태
2. **운영정책 25건 미확정**: 배송비/결제/쿠폰/리뷰 정책이 모두 미결정
3. **CUSTOM 개발 9건 상세 미정의**: 인쇄 특화 대형 모듈의 기능 상세가 부재
4. **화면설계 입력 정보 부족**: 정책이 결정되지 않으면 UI에 표시할 값, 유효성 검사 규칙, 에러 메시지 등을 확정할 수 없음

SPEC-PLAN-001은 이 모든 미결정 사항을 취합하고 추천안을 제시하여, 화면설계에 필요한 입력 정보를 100% 확보하는 역할을 한다.

### 1.3 SPEC-SCREEN-001과의 관계

SPEC-PLAN-001은 SPEC-SCREEN-001과 **보완 관계**에 있으며, 서로 다른 영역을 담당한다:

| 영역 | SPEC-SCREEN-001 | SPEC-PLAN-001 |
|------|-----------------|---------------|
| 화면 설계 | 88개 화면 레이아웃/UI 가이드 | 참조만 (중복 없음) |
| 기능 요구사항 | 개략적 | **EARS 형식 상세 정의** |
| 운영정책 결정 | 권장 기본값 제시 | **25개 정책별 결정 가이드** |
| 요소 체크리스트 | 미포함 | **88개 기능별 필수 요소 목록** |
| 비즈니스 로직 | 미포함 | **CUSTOM 개발 상세 정의** |
| 수용 기준 | 미포함 | **Given/When/Then 형식** |

## 7. 아키텍처 결정사항

### 7.1 Hybrid 전략 유지

```
Tier 1: 쇼핑몰 NATIVE/SKIN (53개 화면)

  - PageShell/ResponsiveGrid 레이아웃
  - shopby Provider + API 래퍼
  - CSS 오버라이드는 @MX:WARN으로 관리

Tier 2: 인쇄 특화 CUSTOM (25개 화면)
  - 완전 독립 모듈
  - Tailwind CSS + Radix UI
  - 커스텀 API (NestJS BFF + AWS S3 + PitStop)
  - shopby 의존성 제로

Tier 3: 관리자 백오피스 (35개 화면)
  - Tailwind CSS + shadcn/ui
  - AdminLayout 독자 체계 (SPEC-SKIN-005)
  - shopby Server API 직접 호출
```

**Hybrid 전략 선정 근거**: 가중 평가 결과 Hybrid(7.55) > Skin 유지(7.05) > Headless 전환(6.45)

### 7.2 PC 우선 설계 (인쇄 특화 화면)

인쇄 관련 기능은 **Desktop-First** 설계를 적용한다:

- **PC 우선 대상**: 상품옵션 선택, 파일 업로드, 편집 미리보기, 가격 시뮬레이터, 관리자 전체
- **이유**: 인쇄 작업은 대형 모니터에서 수행, 파일 업로드가 PC 중심
- **모바일**: 기본 정보 조회만 지원, 파일 업로드는 제한적

일반 쇼핑몰 기능은 **Mobile-First** 유지:

- **모바일 우선 대상**: 로그인, 회원가입, 마이페이지, 장바구니, 결제
- **이유**: 일반 사용자의 모바일 접속률이 높음

### 7.3 단일 페이지 옵션 폼

상품 옵션 선택은 **Step Wizard 방식을 사용하지 않는다**. 피그마 option_NEW 패턴의 **단일 페이지 스크롤 폼** 방식을 채택:

- 모든 옵션을 한 페이지에서 선택
- 종속 옵션은 상위 선택 시 하위 영역이 동적으로 노출
- 가격은 옵션 변경 시 실시간 업데이트
- 하단 고정 요약 + 주문 버튼

### 8.5 공통 패턴 (업계 표준)

경쟁사 분석에서 도출한 **인쇄업계 공통 패턴**:

1. **옵션 구조**: 위에서 아래 순차 선택 세로형 (규격 -> 인쇄도수 -> 용지 -> 코팅 -> 후가공 -> 수량)
2. **가격 표시**: 옵션 변경 시 실시간 계산, VAT 별도/포함 명시
3. **결제 방식**: 선결제 후제작 시스템
4. **파일 포맷**: PDF(CMYK, 300dpi+) 표준, 도련 3mm, 서체 아웃라인 필수
5. **결제 수단**: 신용카드, 계좌이체, 무통장, 간편결제(네이버페이/카카오페이)
6. **주문 상태**: 접수 -> 파일확인 -> 제작 -> 출고 -> 배송 공통 흐름

## 9. 기획서가 해결해야 할 핵심 과제

### 과제 1: 40개 미결정 기능 사항에 대한 추천안 제시

88개 기능 중 40개가 key_decision을 보유하고 있으나 전부 미결정 상태이다. 기획서는 각 결정 사항에 대해:

- 경쟁사 벤치마크 기반 추천안 제시
- 선택지별 장단점 비교
- shopby 설정 가능 여부 명시
- 추가 개발 필요 시 규모 산정

**우선 결정 대상 (P1 기능 중 key_decision 보유)**:
- 로그인 식별자 (이메일 vs 아이디)
- SNS 로그인 채널 범위
- 비회원 주문 허용 여부
- 회원등급 체계 (단계/기준/혜택)
- 프린팅머니 적립/충전 정책
- 파일 업로드 UI/UX 프로세스
- 배송비 정책 (무료배송 기준, 도서산간)
- 간편결제 도입 범위
- 인쇄 주문상태 체계

### 과제 2: 88개 기능별 필수 요소 체크리스트 완성

각 기능마다 화면설계에 필요한 입력 정보를 100% 취합:

- **입력 필드**: 필수/선택 구분, 유효성 검사 규칙
- **표시 정보**: 기본값, 단위, 포맷
- **인터랙션**: 에러 메시지, 성공 메시지, 로딩 상태
- **권한**: 로그인 필수 여부, 회원등급 제한
- **의존 관계**: 타 기능과의 데이터 흐름

### 과제 3: 화면설계를 위한 입력 정보 100% 취합

최종 목표는 **SPEC-SCREEN-001의 88개 화면을 구현할 때 추가 질문 없이 개발에 착수할 수 있는 수준**의 정보 완성도를 달성하는 것이다.

완성도 측정 기준:
