# SPEC-PLAN-001 시스템 연동 맵

> shopby 플랫폼과 CUSTOM 모듈 간의 데이터 경계, 이벤트 트리거, API 계약을 정의합니다.
> 개발 착수 전 아키텍처 명확화를 위한 핵심 문서입니다.

---

## 1. shopby API <-> CUSTOM BFF 경계 정의

| 데이터 | shopby 관리 | CUSTOM 관리 | 동기화 방식 | 연결 키 |
|--------|------------|------------|------------|---------|
| 회원 정보 | memberNo, email, grade | 참조만 (읽기 전용) | shopby API 조회 | `memberNo` |
| 상품 기본 | 상품명, 카테고리, 이미지, 기본가격 | 인쇄옵션, 가격매트릭스 | productNo로 연결 | `productNo` |
| 주문 기본 | 주문번호, 결제정보, 배송정보 | 인쇄상태, 파일검수 | orderNo/orderItemNo로 연결 | `orderNo`, `orderItemNo` |
| 적립금(프린팅머니) | 잔액 관리 | 충전/사용 TX 기록 | shopby 적립금 API + BFF TX 기록 | `memberNo` |
| 쿠폰 | 발급/사용/만료 전담 | 없음 | shopby 전담 | `couponNo` |
| 리뷰 | 리뷰 작성/노출 관리 | 보상 지급 트리거 | shopby 이벤트 감지 -> BFF 적립금 지급 | `reviewNo`, `orderItemNo` |
| 인쇄 파일 | 없음 | S3 저장 + PitStop 검수 전담 | CUSTOM 전담 | `fileId`, `orderItemNo` |
| 인쇄 옵션 | 없음 | 종속옵션 엔진 전담 | CUSTOM 전담 | `productNo`, `optionId` |
| 가격 매트릭스 | 없음 | 8종 가격표 전담 | CUSTOM 전담 | `productNo`, `matrixId` |
| 인쇄 주문 상태 | 주문상태 수신 (동기화) | 인쇄 공정 상태 관리 | CUSTOM -> shopby 주문상태 변경 API | `orderNo`, `customStatusCode` |

### 경계 원칙

- **shopby 우선**: 회원/쿠폰/리뷰 기본 데이터는 shopby가 SSOT(Single Source of Truth)
- **CUSTOM 전담**: 인쇄 파일/옵션/가격/공정 상태는 BFF가 SSOT
- **양방향 동기화**: 주문 상태와 적립금은 양측 모두 관여하며, 연결 키를 통해 일관성 유지

## 2. 이벤트 트리거 맵

| 이벤트 | 발생지 | 수신지 | 액션 | 관련 기능 |
|--------|--------|--------|------|----------|
| 결제 완료 | shopby | BFF | 인쇄 주문 초기화 + 파일 검수 시작 | 주문관리, 파일업로드 |
| 파일 검수 완료 | BFF (PitStop) | Client | 검수 결과 표시 (Pass/Fail/Warning) | 파일 업로드/검수 모듈 |
| 파일 재업로드 | Client | BFF | 재검수 트리거 (PitStop 재실행) | 파일 업로드/검수 모듈 |
| 인쇄 상태 변경 | BFF (Admin) | shopby + Client | shopby 주문상태 동기화 + 알림톡 발송 | 인쇄 주문상태 트래킹 |
| 배송 완료 | shopby | BFF | 리뷰 유도 쿠폰 배치 트리거 | 쿠폰 (#21), 리뷰 |
| 리뷰 작성 | shopby | BFF | 적립금 보상 지급 (텍스트 1,000원 / 포토 2,000원) | 리뷰 보상 (#14) |
| 리뷰 삭제 | shopby | BFF | 쿠폰 자동 회수 (사용분 제외) | 리뷰 회수 (#24) |
| 프린팅머니 충전 | BFF (PG) | shopby | shopby 적립금 증액 API 호출 | 프린팅머니 충전 모듈 |
| 회원가입 완료 | shopby | BFF | 신규 회원 혜택 지급 (쿠폰 5,000원 + 적립금) | 쿠폰 (#10), 회원 |
| 주문 취소 | shopby | BFF | 인쇄 파일 S3 삭제 + 커스텀 상태 '취소' 전환 | 주문관리, 파일 모듈 |
| 수동카드결제 | BFF (Admin) | PG -> shopby | 오프라인 결제 처리 (관리자 수동 결제 등록) | 관리자 수동결제 |

### 이벤트 처리 원칙

- **shopby 발생 이벤트**: shopby Webhook 또는 주기적 API 폴링으로 감지
- **BFF 발생 이벤트**: BFF 내부 이벤트 버스 또는 직접 API 호출로 처리
- **Client 발생 이벤트**: REST API 요청을 통해 BFF로 전달

## 3. API 계약 경계

| 영역 | shopby Open API | NestJS BFF API | 비고 |
|------|----------------|----------------|------|
| 인증 | `/auth/*` (shopby 전담) | - | shopby 인증 체계 사용 |
| 회원 | `/members/*` (shopby 전담) | - | 회원 CRUD shopby 전담 |
| 상품 기본 | `/products/*` (shopby 전담) | - | 상품명/카테고리/이미지 등 |
| 상품 인쇄옵션 | - | `/custom/options/*` (BFF 전담) | 종속옵션 엔진, 옵션 체인 |
| 상품 가격 | - | `/custom/pricing/*` (BFF 전담) | 8종 가격매트릭스 계산 |
| 주문 기본 | `/orders/*` (shopby 전담) | - | 주문 생성/조회/취소 |
| 주문 인쇄상태 | - | `/custom/orders/*/status` (BFF 전담) | 인쇄 공정 8단계 상태 |
| 파일 | - | `/custom/files/*` (BFF 전담) | S3 업로드, PitStop 검수 |
| 쿠폰 | `/coupons/*` (shopby 전담) | - | 쿠폰 발급/사용/조회 |
| 적립금 잔액 | `/members/*/accumulation` (shopby 전담) | - | 적립금 잔액 조회 |
| 적립금 충전 | - | `/custom/printing-money/*` (BFF -> shopby API 호출) | PG 결제 후 shopby 적립금 증액 |
| 리뷰 | `/reviews/*` (shopby 전담) | - | 리뷰 작성/조회/삭제 |
| 리뷰 보상 | - | `/custom/rewards/*` (BFF 전담) | 적립금 자동 지급/회수 |
| 알림 | - | `/custom/notifications/*` (BFF 전담) | 카카오 알림톡, SMS |
| 관리자 기본 | `/admin/*` (shopby Server API) | - | shopby 관리자 기능 |
| 관리자 인쇄 | - | `/custom/admin/*` (BFF 전담) | 인쇄 상태 변경, 파일 확인 |
| 통계 기본 | `/admin/statistics/*` (shopby) | - | 매출, 주문, 회원 통계 |
| 통계 인쇄 | - | `/custom/admin/statistics/*` (BFF 전담) | 인쇄/굿즈/패키지별 통계 |

### API 설계 원칙

- **shopby Open API**: 쇼핑몰 Front 영역에서 호출 (회원/상품/주문/쿠폰/리뷰)
- **shopby Server API**: 관리자 영역에서 호출 (인증 토큰 필요)
- **BFF API**: `/custom/*` 네임스페이스로 통일, 인쇄 특화 기능 전담
- **BFF -> shopby 호출**: 적립금 충전, 주문상태 동기화 등 BFF에서 shopby API를 서버 사이드로 호출

## 4. 데이터 동기화 규칙

### 4.1 실시간 동기화

즉각적인 데이터 일관성이 필요한 영역

| 대상 | 방향 | 트리거 | 허용 지연 |
|------|------|--------|----------|
| 주문 상태 변경 | CUSTOM -> shopby | 관리자 인쇄 상태 변경 시 즉시 | < 3초 |
| 적립금 변동 | BFF -> shopby | 프린팅머니 충전 완료 시 즉시 | < 5초 |
| 파일 검수 결과 | BFF -> Client | PitStop 검수 완료 시 즉시 | < 10초 |

### 4.2 배치 동기화

주기적으로 처리해도 무방한 영역

| 대상 | 방향 | 주기 | 처리 방식 |
|------|------|------|----------|
| 회원 등급 변경 | shopby -> BFF 참조 | 일 1회 (새벽) | shopby 등급 산정 결과 반영 |
| 쿠폰 만료 처리 | shopby 전담 | shopby 자체 배치 | shopby 내부 처리 |
| 리뷰 유도 쿠폰 발행 | BFF -> shopby | 배송완료 14일 후 | 배치 스케줄러 (#21) |
| 재구매/VIP 쿠폰 발행 | BFF -> shopby | 월 1회 | 구매 이력 분석 후 발행 (#18, #19) |

### 4.3 이벤트 기반 동기화

특정 이벤트 발생 시에만 동기화

| 대상 | 트리거 이벤트 | 방향 | 처리 |
|------|-------------|------|------|
| 결제 완료 -> 인쇄 주문 초기화 | shopby 결제 Webhook | shopby -> BFF | 인쇄 주문 레코드 생성 |
| 리뷰 작성 -> 적립금 보상 | shopby 리뷰 이벤트 | shopby -> BFF -> shopby | 보상 계산 후 적립금 증액 |
| 배송 완료 -> 리뷰 유도 | shopby 배송 이벤트 | shopby -> BFF | 쿠폰 발행 배치 큐에 등록 |
| 주문 취소 -> 파일 삭제 | shopby 취소 이벤트 | shopby -> BFF | S3 파일 삭제 + 상태 전환 |

### 동기화 원칙

- **멱등성 보장**: 동일 이벤트 중복 수신 시에도 동일한 결과 보장
- **순서 보장**: 주문 상태 변경은 상태 전이 규칙에 따라 순서 검증
- **실패 재시도**: 동기화 실패 시 최대 3회 재시도, 이후 Dead Letter Queue로 이관

## 5. 장애 대응

### 5.1 shopby API 장애 시

| 영향 범위 | CUSTOM 모듈 독립 동작 가능 범위 | 제한 사항 |
|----------|-------------------------------|----------|
| 인쇄 파일 업로드 | 정상 동작 (S3 + PitStop 독립) | 주문 연결 불가 |
| 인쇄 옵션 조회 | 정상 동작 (BFF DB 독립) | 상품 기본정보 표시 불가 |
| 가격 계산 | 정상 동작 (BFF 매트릭스 독립) | 최종 주문 생성 불가 |
| 인쇄 상태 관리 | 정상 동작 (BFF 상태 관리) | shopby 주문상태 동기화 지연 |
| 프린팅머니 충전 | PG 결제만 가능 | shopby 적립금 반영 지연 -> 큐 적재 |
| 리뷰 보상 | 보상 큐 적재 | 적립금 지급 지연 |

**대응 전략**: shopby API 장애 감지 시 Circuit Breaker 패턴 적용, 동기화 대상 이벤트는 큐에 적재 후 복구 시 일괄 처리

### 5.2 BFF 장애 시

| 영향 범위 | shopby 기본 기능 유지 | 중단 기능 |
|----------|---------------------|----------|
| 회원/로그인 | 정상 | - |
| 상품 기본 조회 | 정상 | 인쇄옵션/가격 조회 불가 |
| 주문 기본 | 정상 | 인쇄 파일 업로드/검수 불가 |
| 쿠폰/리뷰 | 정상 | 리뷰 보상 자동 지급 중단 |
| 관리자 기본 | 정상 | 인쇄 상태 변경 불가 |

**대응 전략**: BFF 장애 시 shopby 기본 쇼핑몰 기능은 정상 유지. 인쇄 특화 기능에 대해 "일시적으로 이용할 수 없습니다" 안내 표시

### 5.3 PitStop 장애 시

| 영향 범위 | 대응 방안 |
|----------|----------|
| 파일 검수 불가 | 수동 검수 모드로 전환 |
| 신규 업로드 | 파일 업로드는 허용, 검수 결과 '대기' 표시 |
| 관리자 확인 | 관리자가 직접 파일 다운로드 후 수동 확인 |
| 복구 후 | 대기 상태 파일 일괄 재검수 배치 실행 |

**대응 전략**: PitStop 장애 감지 시 자동으로 수동 검수 모드 전환, 관리자에게 알림톡 발송

### 5.4 S3 장애 시

| 영향 범위 | 대응 방안 |
|----------|----------|
| 파일 업로드 | 일시 중단, 사용자에게 재시도 안내 |
| 파일 다운로드 | 캐시된 파일 제공 (CDN 유효 시) |
| 파일 삭제 | 삭제 큐에 적재, 복구 후 일괄 처리 |
| 복구 후 | 큐 대기 파일 순차 업로드 처리 |

**대응 전략**: S3 장애 시 파일 업로드 UI에 "파일 업로드가 일시적으로 지연됩니다" 안내. 로컬 임시 저장 후 복구 시 자동 재업로드

### 장애 대응 공통 원칙

- **Health Check**: 각 외부 서비스에 대해 30초 주기 Health Check 수행
- **Circuit Breaker**: 연속 5회 실패 시 Circuit Open, 30초 후 Half-Open 시도
- **Fallback**: 장애 시 Graceful Degradation으로 핵심 쇼핑 기능 유지
- **알림**: 장애 발생 즉시 관리자 Slack/알림톡 발송
- **복구 자동화**: 장애 복구 감지 시 적재된 큐 자동 처리
