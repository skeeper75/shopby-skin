# SPEC-SKIN-002: 마이페이지 - 구현 계획

> **SPEC ID**: SPEC-SKIN-002
> **문서 유형**: 구현 계획 (plan.md)

## 1. 마일스톤

### Primary Goal: 주문조회 + 회원관리
- 마이페이지 메인 레이아웃 (사이드바 + 콘텐츠)
- 주문조회 목록 + 상세 (타임라인 UI 포함)
- 회원정보수정 / 비밀번호변경

### Secondary Goal: 인쇄 전문 기능
- 옵션보관함 (API + UI)
- 프린팅머니 충전 (PG 연동)
- 증빙서류 발급 (4상태 관리)

### Final Goal: 커뮤니티 기능
- 쿠폰관리, 리뷰 (쿠폰 자동발행/회수), Q&A, 1:1문의
- 체험단 활동내역

## 2. 기술적 접근
- Aurora MyPage 구조 재활용, 사이드바 네비게이션 추가
- 커스텀 API 필요 항목: 옵션보관함, 프린팅머니 충전, 증빙서류, 체험단
- TanStack Query로 서버 상태 관리 (주문 목록 캐싱)

## 3. 리스크

| 리스크 | 영향도 | 대응 |
|--------|--------|------|
| 커스텀 API 서버 부재 | High | Shopby API 최대 활용 후 부족분 판별 |
| PG 연동 (프린팅머니) | High | 이니시스 SDK 선행 검토 |
| 리뷰 쿠폰 자동발행/회수 | Medium | Shopby 쿠폰 API 활용 가능 여부 확인 |
