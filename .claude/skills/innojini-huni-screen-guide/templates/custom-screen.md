## 6. 상태 관리 패턴

> CUSTOM 권장: **독립 Context 또는 useState 조합**

**단순 화면 (S/M)**:
```
useState 조합
  ├── [data, setData] - API 응답 데이터
  ├── [loading, setLoading] - 로딩 상태
  └── [error, setError] - 에러 상태
```

**복잡 화면 (L/XL)**:
```
독립 Context Provider
  └── useReducer
        ├── state.step: 현재 단계 (멀티스텝)
        ├── state.data: 수집된 데이터
        ├── state.files: 업로드 파일 목록
        └── state.validation: 유효성 검증 결과
```

## 7. 인터랙션 상태

| 상태 | UI 설명 | 트리거 |
|------|---------|--------|
| loading | 스켈레톤 또는 스피너 | 초기 로딩, API 호출 |

## 8. 에러 처리

| 에러 케이스 | 사용자 피드백 | 복구 방법 |
|------------|-------------|----------|
