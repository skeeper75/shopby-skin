## 5. shopby API 연동

> SKIN 화면은 `fetchHttpRequest()` (src/utils/api.js)를 통해 shopby API를 직접 호출합니다.

**API 호출 패턴**:
```javascript
// src/utils/api.js의 fetchHttpRequest 사용
const response = await fetchHttpRequest({
  url: '/api/endpoint',
  method: 'GET',
  params: { /* 요청 파라미터 */ }
});
```

| API | 메서드 | 경로 | 요청 파라미터 | 응답 주요 필드 | 인증 |
|-----|--------|------|-------------|--------------|------|
| [여기에 작성] | GET/POST | /api/... | [파라미터] | [필드] | public/member |

## 6. 상태 관리 패턴

> SKIN 권장: **shopby API + useReducer 로컬 상태**

```
useReducer (로컬 상태)
  ├── state.data: API 응답 데이터
  ├── state.loading: 로딩 상태
  ├── state.error: 에러 상태
  └── state.filters: 필터/검색 상태

dispatch({ type: 'FETCH_START' })
dispatch({ type: 'FETCH_SUCCESS', payload: data })
dispatch({ type: 'FETCH_ERROR', error: err })
```

## 8. 에러 처리

| 에러 케이스 | 사용자 피드백 | 복구 방법 |
|------------|-------------|----------|
| API 응답 실패 | 에러 메시지 표시 + "다시 시도" 버튼 | 재요청 |
