## 3. 컴포넌트 트리

```
AdminLayout
  └── Sidebar (nav)
  └── Content
        ├── PageHeader (title + breadcrumb)
        ├── FilterSection (Card, collapsible)
        │     ├── [여기에 작성: 검색 조건 필드들]
        │     └── ButtonGroup (검색 + 초기화)
        └── ResultSection
              ├── StatusTabBar (tabs + count badges)
              ├── ActionBar (전체선택 + bulk action buttons)
              ├── DataTable (checkbox + sortable + row actions)
              │     └── DataTableRow → onClick → DetailPanel
              └── Pagination (pageSize: 30/100 + 엑셀 export)

DetailPanel (Sheet/Drawer):
  ├── [여기에 작성: 상세 정보 섹션들]
  └── ActionButtons (수정/삭제/상태변경)
```

## 5. API 연동

> Admin SKIN: shopby Server API 활용
> Admin CUSTOM: 자체 API 설계

**목록 조회**:

| API | 메서드 | 경로 | 요청 파라미터 | 응답 주요 필드 | 인증 |
|-----|--------|------|-------------|--------------|------|
| 목록 조회 | GET | [여기에 작성] | page, size, status, keyword, dateRange | items[], totalCount | admin |
| 상세 조회 | GET | [여기에 작성]/:id | - | 상세 필드들 | admin |
| 수정 | PUT | [여기에 작성]/:id | 수정 데이터 | success | admin |
| 삭제 | DELETE | [여기에 작성]/:id | - | success | admin |
| 일괄 상태변경 | POST | [여기에 작성]/batch | ids[], targetStatus | results[] | admin |

## 6. 상태 관리 패턴

> Admin 권장: **useState (로컬) + URL 쿼리 파라미터 (필터 상태 영속화)**

```
URL 쿼리 파라미터 (필터 상태)
  ├── ?page=1&size=30
  ├── ?status=접수
  ├── ?keyword=검색어
  └── ?dateFrom=2026-01-01&dateTo=2026-03-19

useState (로컬 상태)
  ├── data: DataTable 데이터
  ├── selectedIds: 체크박스 선택 목록
  ├── detailOpen: Sheet 열림 상태
  └── detailId: 상세 조회 대상 ID
```

## 7. 인터랙션 상태

| 상태 | UI 설명 | 트리거 |
|------|---------|--------|
| loading | DataTable 스켈레톤 | 초기 로딩, 필터 변경, 페이지 이동 |
| empty | "검색 결과가 없습니다" + 필터 초기화 링크 | 결과 0건 |
| error | 에러 토스트 + 재시도 버튼 | API 실패 |
| success | DataTable 정상 렌더링 | 데이터 로딩 완료 |
| detail-open | Sheet(Drawer) 우측 슬라이드 | 행 클릭 |
| bulk-selecting | ActionBar 활성화 + 선택 카운트 표시 | 체크박스 선택 |
| status-changing | 확인 Dialog + 진행 스피너 | 일괄 상태 변경 |
| [여기에 작성] | [추가 상태] | [트리거] |
