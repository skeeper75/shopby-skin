- ID: SCR-A7-ABOUT
- 화면명: 회사 소개

- 규모: M (히어로 + 통계 배너 + 연혁 타임라인 + 오시는 길 링크)

    ├── TimelineHistory

### 4. Props / States

| 항목 | 타입 | 설명 |
|------|------|------|
| `companyStats` | `StatItem[]` | 연매출/고객사/직원수 수치 |

| `heroImageUrl` | `string` | 히어로 배경 이미지 경로 |

### 5. API 매핑

| 동작 | 방식 | 비고 |
|------|------|------|
| 콘텐츠 로드 | 정적 (하드코딩 또는 CMS) | Shopby API 미사용 |

1. 페이지 마운트 → 정적 데이터 렌더링 (API 없음)

| 상태 | 설명 |
|------|------|
| Initial | 정적 콘텐츠 즉시 렌더링 |
| Animated | StatsBanner 카운터 애니메이션 (뷰포트 진입 시) |
| Hover | TimelineItem 호버 시 강조 |
| CTA Hover | 찾아오시는 길 버튼 호버 상태 |

### 8. 에러 처리

| 에러 | 처리 방식 |
|------|---------|
| 이미지 로드 실패 | alt 텍스트 표시 + 배경색 폴백 |

### 9. 접근성

- CompanyHero 배경 이미지: `role="img"` + `aria-label="회사 소개 대표 이미지"`
- StatsBanner: `aria-label` 각 항목에 단위 포함 (예: "연매출 50억원 이상")
- TimelineHistory: `<ol>` 구조 사용, 연도별 `<li>`
- DirectionsCTA: 명시적 링크 텍스트 ("찾아오시는 길 보기")

### 1. 화면 개요

- ID: SCR-A7-TERMS
- 화면명: 이용약관
- 분류: NATIVE (GET /terms 활용)
- 우선순위: 3 (법적 필수 문서, 신규 구현 비중 낮음)
- 규모: S (TOC 네비게이션 + 법률 문서 본문)

### 2. 와이어프레임

│  2. 서비스 이용               │
│  3. 개인정보 처리              │
│  ...                         │
├─────────────────────────────┤
│ 제1조 (목적)                  │ ← LegalDocument

### 3. 컴포넌트 트리

```
PageShell (maxWidth="xl")
└── TermsPage
    ├── PageHeader (제목: "이용약관")
    ├── TOCNav (데스크탑: sticky 사이드바, 모바일: 접이식)
    │   └── TOCItem × N (섹션 앵커 링크)
    └── LegalDocument
        └── LegalSection × N (조항별)
```

### 4. Props / States

| 항목 | 타입 | 설명 |
|------|------|------|
| `termsContent` | `TermsData` | API 응답 약관 본문 |
| `activeSection` | `string` | 현재 스크롤 위치 섹션 ID |
| `isLoading` | `boolean` | 로딩 상태 |

### 5. API 매핑

| 동작 | 메서드 | 엔드포인트 | 파라미터 |
|------|--------|-----------|---------|
| 약관 조회 | GET | `/terms` | `termsType=USE` |

### 6. 데이터 플로우

1. 페이지 마운트 → GET /terms?termsType=USE
2. 스크롤 → activeSection 업데이트 → TOCNav 하이라이트
3. TOCItem 클릭 → 해당 섹션으로 스크롤

### 7. 인터랙션 상태 (규모 S: 3가지)

| 상태 | 설명 |
|------|------|
| Loading | 스켈레톤 UI |
| Loaded | 약관 본문 표시, TOC 활성화 |
| Empty | "약관 정보를 불러올 수 없습니다" |

### 8. 에러 처리

| 에러 | 처리 방식 |
|------|---------|
| API 실패 | 에러 메시지 + 재시도 버튼 |

### 9. 접근성

- TOCNav: `<nav aria-label="약관 목차">`
- LegalDocument: 각 조항 `<section>` + `id` 앵커
- TOCItem: `aria-current="true"` (현재 섹션)

| 동작 | 방식 | 비고 |
|------|------|------|
| 지도 표시 | 카카오맵 iframe 임베드 | API Key 환경변수 필요 |

### 6. 데이터 플로우

1. 페이지 마운트 → KakaoMap iframe 로드
2. 지도 로드 완료 → mapLoaded=true
3. "카카오맵으로 열기" 클릭 → 카카오맵 앱/웹 딥링크

| 상태 | 설명 |
|------|------|
| MapLoading | 지도 영역 스켈레톤 |
| MapLoaded | 지도 + 교통편 안내 표시 |
| MapError | "지도를 불러올 수 없습니다" + 주소 텍스트 표시 |

### 8. 에러 처리

| 에러 | 처리 방식 |
|------|---------|
| 카카오맵 로드 실패 | 주소 텍스트 + 외부 링크 버튼만 표시 |
| API Key 오류 | 동일 (폴백 처리) |

### 9. 접근성

- KakaoMap iframe: `title="후니프린팅 위치 지도"`
- ExternalMapLink: `aria-label="카카오맵에서 길찾기 (새 탭에서 열림)"` + `target="_blank" rel="noopener"`
- TransportInfo: 각 섹션 `<section>` + heading 구조

### 6. 데이터 플로우

1. 목록 페이지 마운트 → 가이드 카드 로드
2. 검색어 입력 → 클라이언트 사이드 필터링
3. GuideCard 클릭 → 상세 페이지 이동
4. 상세 이미지 탭/클릭 → ImageZoom 활성화
5. GuideNavigation → 이전/다음 가이드 이동

### 7. 인터랙션 상태 (규모 M: 4가지 + 2 인터랙션)

| 상태 | 설명 |
|------|------|
| Loading | 카드 스켈레톤 UI |
| Loaded | 가이드 카드 그리드 표시 |
| DetailView | 상세 페이지 (이미지 + 본문) |
| ZoomedImage | 이미지 확대 오버레이 |

| 인터랙션 | 설명 |
|---------|------|
| 이미지 확대 | 탭/클릭 → ImageZoom 오버레이. 핀치줌(모바일) / 스크롤줌(데스크탑) |
| 가이드 검색 | 입력 즉시 필터링 (debounce 300ms) |

### 8. 에러 처리

| 에러 | 처리 방식 |
|------|---------|
| 이미지 로드 실패 | 대체 이미지 표시 |
| 콘텐츠 로드 실패 | 에러 메시지 + 재시도 버튼 |

### 9. 접근성

- GuideCardGrid: `role="list"`, GuideCard: `role="listitem"`
- GuideCard: 클릭 가능 전체 영역 `<a>` 태그 사용
- ImageZoom: `role="dialog"` + `aria-modal="true"`, ESC로 닫기
- GuideNavigation: `aria-label="이전/다음 가이드"` + 비활성 시 `aria-disabled="true"`
