## SCR-A4-NOTICE

**공지사항 | NATIVE | 우선순위 2 | 규모 S**

### 1. 화면 개요

- ID: SCR-A4-NOTICE
- 화면명: 공지사항
- 분류: NATIVE (Shopby GET /boards/notices 활용)
- 우선순위: 2 (운영 공지, 이벤트 안내, 서비스 변경 사항 전달)
- 규모: S (카테고리 필터 + 목록 + 페이지네이션)

### 2. 와이어프레임

**모바일 (375px)**
```

│ │                2025.03.15│ │

```

### 3. 컴포넌트 트리

```
PageShell
└── NoticeListPage
    ├── PageHeader (제목: "공지사항")
    ├── CategoryFilter
    │   └── FilterTab × 4 (전체 / 공지 / 이벤트 / 상품)
    ├── NoticeList
    │   └── NoticeItem × N
    │       ├── CategoryBadge (이벤트 / 공지 / 상품)
    │       ├── NoticeTitle
    │       └── NoticeDate
    └── Pagination
```

### 4. Props / States

| 항목 | 타입 | 설명 |
|------|------|------|
| `activeCategory` | `string` | 선택된 카테고리 필터 (기본: "전체") |
| `currentPage` | `number` | 현재 페이지 번호 |
| `noticeList` | `Notice[]` | 공지 목록 배열 |
| `totalCount` | `number` | 전체 공지 수 |
| `isLoading` | `boolean` | 목록 로딩 상태 |

### 5. API 매핑

| 동작 | 메서드 | 엔드포인트 | 파라미터 |
|------|--------|-----------|---------|
| 목록 조회 | GET | `/boards/notices` | `boardType`, `pageNumber`, `pageSize=20` |
| 카테고리 필터 | GET | `/boards/notices` | `boardType=EVENT|NOTICE|PRODUCT` |

### 6. 데이터 플로우

1. 페이지 마운트 → GET /boards/notices (전체, page=1)
2. CategoryFilter 탭 클릭 → boardType 파라미터 변경 → 재요청
3. 페이지네이션 클릭 → pageNumber 변경 → 재요청
4. NoticeItem 클릭 → 공지 상세 페이지로 이동

### 7. 인터랙션 상태 (규모 S: 3가지)

| 상태 | 설명 |
|------|------|
| Loading | 스켈레톤 UI (NoticeItem × 5) |
| Loaded | 목록 표시, 카테고리 필터 활성화 |
| Empty | "등록된 공지사항이 없습니다" 안내 메시지 |

### 8. 에러 처리

| 에러 | 처리 방식 |
|------|---------|
| API 실패 | "공지사항을 불러오지 못했습니다. 잠시 후 다시 시도해주세요" + 재시도 버튼 |
| 네트워크 오류 | 토스트 메시지 표시 후 이전 데이터 유지 |

### 9. 접근성

- CategoryFilter 탭: `role="tablist"` / `role="tab"` / `aria-selected`
- NoticeList: `role="list"` / `role="listitem"`
- CategoryBadge: `aria-label="카테고리: {분류명}"`
- Pagination: `aria-label="페이지 탐색"`, 현재 페이지 `aria-current="page"`

## SCR-A4-FAQ

**FAQ | NATIVE | 우선순위 2 | 규모 S**

### 1. 화면 개요

- ID: SCR-A4-FAQ
- 화면명: FAQ (자주 묻는 질문)
- 분류: NATIVE (Shopby GET /boards/faqs 활용)
- 우선순위: 2 (주문/배송/결제 관련 반복 문의 자가 해결 지원)
- 규모: S (카테고리 탭 + 검색바 + 아코디언 목록)
- 참고: SPEC-LAYOUT-002 fix 적용 — 아코디언 + 카테고리 탭 조합

### 2. 와이어프레임

### 3. 컴포넌트 트리

```
PageShell
└── FAQPage
    ├── PageHeader (제목: "FAQ")
    ├── SearchBar (placeholder: "FAQ를 검색해보세요")
    ├── CategoryTabs
    │   └── CategoryTab × N (전체 / 주문결제 / 배송 / 취소교환환불 / 기타)
    └── FAQAccordion
        └── FAQItem × N
            ├── FAQQuestion (Q. 텍스트 + 토글 아이콘)
            └── FAQAnswer (A. 텍스트, 조건부 렌더링)
```

### 4. Props / States

| 항목 | 타입 | 설명 |
|------|------|------|
| `activeCategory` | `string` | 선택된 카테고리 (기본: "전체") |
| `searchKeyword` | `string` | 검색어 입력값 |
| `openItemId` | `string \| null` | 현재 펼쳐진 아코디언 항목 ID |
| `faqList` | `FAQ[]` | FAQ 목록 |
| `isLoading` | `boolean` | 로딩 상태 |

### 5. API 매핑

| 동작 | 메서드 | 엔드포인트 | 파라미터 |
|------|--------|-----------|---------|
| 전체 목록 | GET | `/boards/faqs` | `pageSize=50` (전체 로드) |
| 카테고리 필터 | GET | `/boards/faqs` | `boardType=ORDER|DELIVERY|CANCEL|ETC` |
| 검색 | GET | `/boards/faqs` | `keyword={검색어}` |

### 6. 데이터 플로우

1. 페이지 마운트 → GET /boards/faqs 전체 로드 (페이지네이션 없이 전체)
2. 검색어 입력 (디바운스 300ms) → keyword 파라미터로 재요청
3. CategoryTabs 클릭 → boardType 변경 → 재요청 (검색어 초기화)
4. FAQItem 클릭 → openItemId 토글 (하나만 열리는 exclusive 방식)

### 7. 인터랙션 상태 (규모 S: 3가지)

| 상태 | 설명 |
|------|------|
| Loading | 스켈레톤 아코디언 × 5 |
| Loaded | FAQ 목록 표시, 검색/필터 활성화 |
| Empty (검색 결과 없음) | "'{키워드}'에 대한 FAQ가 없습니다" + 1:1 문의 유도 링크 |

### 8. 에러 처리

| 에러 | 처리 방식 |
|------|---------|
| API 실패 | "FAQ를 불러오지 못했습니다" 인라인 에러 + 재시도 버튼 |
| 검색 실패 | 이전 목록 유지, 토스트 경고 |

### 9. 접근성

- SearchBar: `role="search"`, `aria-label="FAQ 검색"`
- CategoryTabs: `role="tablist"` / `role="tab"` / `aria-selected`
- FAQAccordion: `role="list"`
- FAQItem 버튼: `aria-expanded={true|false}`, `aria-controls="faq-answer-{id}"`
- FAQAnswer: `id="faq-answer-{id}"`, `role="region"`

|------|--------|-----------|---------|

| 문의 등록 | POST | `/boards/product-inquiries` | `productNo`, `title`, `content`, `secret` |
| 내 문의 조회 | GET | `/boards/product-inquiries/me` | - |

3. WriteForm 제출 → POST /boards/product-inquiries → 성공 시 목록 새로고침
4. 비밀글 항목 클릭 → 본인 또는 관리자만 내용 확인 가능 (권한 체크)

### 7. 인터랙션 상태 (규모 S: 3가지)

| 상태 | 설명 |
|------|------|
| Loading | 스켈레톤 QAItem × 5 |
| Loaded | 목록 표시, 필터/작성 활성화 |
| Empty | "등록된 문의가 없습니다. 첫 번째 문의를 남겨보세요!" |

### 8. 에러 처리

| 에러 | 처리 방식 |
|------|---------|
| 문의 등록 실패 | 폼 내 인라인 에러 메시지, 입력 내용 보존 |
| 권한 없는 비밀글 접근 | "비밀글입니다. 작성자만 확인 가능합니다" |
| 미로그인 문의 시도 | 로그인 페이지로 리다이렉트 |

### 9. 접근성

- WriteButton: `aria-haspopup="dialog"`
- WriteForm 모달: `role="dialog"`, `aria-labelledby="write-form-title"`
- SecretCheckbox: `aria-label="비밀글로 작성"`
- StatusFilter: `role="group"`, `aria-label="답변 상태 필터"`
